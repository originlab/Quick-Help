FAQ-701 Ich kann das Projekt aufgrund eines nicht sichtbaren offenen bzw. minimierten Dialogs nicht speichern Wie löse ich das Problem?

saveOPJ-when-hidden-dialog

Letztes Update: 07.09.2021

Auf dieses Problem treffen Sie vielleicht unbewusst. Sie öffnen einen Dialog mit dem geöffneten OPJ, führen eine Operation durch und "verbergen" ihn dann irgendwie, vielleicht indem Sie ihn minimieren oder aus dem Computerbildschirm herausziehen. Wenn Sie das Arbeitsprojekt speichern, wird die Fehlermeldung "Bitte schließen Sie alle geöffneten Dialoge, bevor Sie Origin schließen" angezeigt. Zu diesem Zeitpunkt können Sie den Dialog nicht finden und ihn daher nicht schließen. Um dieses Problem zu beheben, versuchen Sie die folgenden zwei Lösungen:

Lösung 1:

Ändern Sie die Bildschirmauflösung drastisch, um zu prüfen, ob ein minimierter Dialog angezeigt wird.

Wenn Lösung 1 nicht funktioniert, versuchen Sie Lösung 2.


Lösung 2:

Führen Sie die Datei CloseOriginDialogs.exe aus (Wenn Sie die 64-Bit-Version von Origin installiert haben, führen Sie vorzugsweise die Datei CloseOriginDialogs_64.exe aus), wodurch alle möglichen angezeigten Dialoge geschlossen werden.


Lösung 1:

In Origin 2022 oder einer neueren Version können Sie die Systemvariable SDO = 1 setzen. Sie ermöglicht das direkte Speichern des Projektes, auch wenn es einen unsichtbaren geöffneten Dialog oder minimierten Dialog gibt. Die Fehlermeldung "Bitte alle geöffneten Dialoge vor dem Schließen von Origin schließen." wird nicht angezeigt.

Schlüsselwörter:minimierter Dialog, OPJ speichern
