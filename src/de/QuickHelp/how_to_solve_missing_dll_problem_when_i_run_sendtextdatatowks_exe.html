FAQ-802 Wenn ich ein COM-Beispiel ausführe, das von VB erstellt wurde, sehe ich eine Fehlermeldung zu einer fehlenden DLL. Wie behebe ich dieses Problem?

SendTextDataToWks-fail

Letztes Update: 31.07.2018

Wenn Sie einige unserer integrierten, mit VB erstellten COM-Beispiele ausführen wie SendTextDataToWks.exe im <Origin-Verzeichnis>\Samples\COM Server and Client\VB\SendTextDataToWks, kann es sein, dass Sie eine Fehlermeldung erhalten, die besagt:

Die Klasse wird nicht registriert. Die folgende Datei muss auf Ihrem Rechner installiert sein. MSSTDFMT.DLL.

Um dieses Problem zu umgehen:

  • Wenn Sie ein 64-Bit-Betriebssystem haben:
    1. Laden Sie MSSTDFMT.DLL herunter. Speichern Sie die Datei im Ordner C:\Windows\sysWOW64.
    2. Bleiben Sie im Ordner sysWoW64 und suchen die Datei cmd.exe.
    3. Klicken Sie mit der rechten Maustaste auf cmd.exe und wählen Sie Als Administrator ausführen. Das Befehlsfenster wird angezeigt mit
      C:\Windows\system32>_
    4. Geben Sie Regsvr32 C:\Windows\SysWOW64\MSSTDFMT.DLL ein.
    5. Jetzt haben Sie im Befehlsfenstereintrag:
      C:\Windows\system32>Regsvr32 C:\Windows\SysWOW64\MSSTDFMT.DLL
      Drücken Sie Enter.
  • Wenn Sie ein 32-Bit-Betriebssystem haben:
    1. Laden Sie MSSTDFMT.DLL herunter. Speichern Sie die Datei im Ordner C:\Windows\system32.
    2. Bleiben Sie im Ordner system32 und suchen die Datei cmd.exe.
    3. Klicken Sie mit der rechten Maustaste auf cmd.exe und wählen Sie Als Administrator ausführen. Das Befehlsfenster wird angezeigt mit
      C:\Windows\system32>
    4. Geben Sie Regsvr32 C:\Windows\system32\MSSTDFMT.DLL ein.
    5. Jetzt haben Sie im Befehlsfenstereintrag:
      C:\Windows\system32>Regsvr32 C:\Windows\system32\MSSTDFMT.DLL
      Drücken Sie Enter.
