FAQ-926 Kann ich steuern, welches Fenster im Dateiexplorer Symbolansichten und Vorschaufelder anzeigt?

Control-Origin-Window-Display-File-Explorer-Preview

Letztes Update: 15.07.2018

Wenn Sie eine OPJU-Datei mit Origin 2018b oder höher speichern, wird ein Projektfenster für das Symbol des Windows-Dateiexplorers verwendet (wählen Sie im Dateiexplorer keine Datei aus, klicken Sie mit der rechten Maustaste und wählen Sie eine der Symbolansichten, z. B. Extragroße Symbole). Zusätzlich wird das gleiche Fenster oben in der Liste der Fenster in der Vorschau gezeigt, wenn das Vorschaufeld des Dateiexplorers aktiviert ist (ALT + P).

In den einfachsten Fällen wird das aktive Diagrammfenster verwendet, um das Symbol der Projektdatei zu erstellen. In komplexen Projektdateien oder in Situationen, in denen Sie einen bestimmten Fenstertyp benötigen, der als Projektsymbol dient, können Sie etwas Kontrolle ausüben, indem Sie den Wert der "@ Systemvariable" @SPM festlegen, wobei die folgende Prioritätsfolge der Fenster zu beachten ist:

Matrix = 1 Worksheet = 2, HolderImage = 3 //book with MatrixSheet in Image View MatImage = 4  //matrix in Image View HolderGraph = 5 GraphNotOpen = 6 //in another folder, hidden or iconized Graph = 7

... wobei sich "Holder" auf eine Grafik oder eine Matrix bezieht, die als Blatt zu einer Arbeitsmappe hinzugefügt wurden. Falls es sich um eine Matrix handelt muss der Bildmodus (Ansicht: Bildmodus) verwendet werden.

Die Variable @SPM bestimmt, ob ein gegebener Fenstertyp auf "Must Use if Active" ("Muss verwendet werden, wenn aktiv") erhöht wird. Standard ist @SPM = 3, kann aber im Skriptfenster (Fenster: Skriptfenster) auf einen der obenstehenden Werte geändert werden. Der Wert von @SPM wird in der Windows Registry gespeichert und bleibt über verschiedene Origin bestehen, bis er wieder geändert wird.

Wenn Sie zum Beispiel @SPM=5 festlegen, werden nur Diagramme verwendet, entweder alleinstehende Diagrammfenster oder "HolderGraphs" (Diagramme, die als Blätter zu Arbeitsmappen hinzugefügt wurden). Wenn das aktive Fenster kein Diagramm ist, wird ein anderes sichtbares Diagrammfenster (nicht symbolisiert) verwendet, wenn es verfügbar ist. Wenn es mehrere solche Diagrammfenster im aktuellen Ordner gibt, wird das zuletzt aktive verwendet.

Wenn Sie sich Gedanken zur Wirkung der Vorschau auf die Dateigröße machen, können Sie das Speichern der Projekt-Explorer-Vorschauen in Dateien mit Hilfe der Systemvariablen @PESS verhindern.

Schlüsselwörter:aktives Diagramm, eingebettetes Diagramm, Diagrammblatt, Matrixblatt, Projekt Explorer, Vorschauen
