FAQ-462 Was ist der Unterschied zwischen Datenmatrix und Bildmatrix?

Data-and-Image

Letztes Update: 10.04.2020

Ein Matrixfenster kann mehrere Blätter enthalten, und jedes Blatt kann mehrere Matrixobjekte besitzen. Jedes Matrixobjekt kann wiederum ein Bild oder Daten enthalten.

Wenn eine Bilddatei in ein Matrixobjekt importiert wird, legt Origin das Matrixobjekt standardmäßig auf den Typ Bild fest. Dies wird durch das Bildsymbol rechts oben im Matrixfenster angezeigt.

Wenn ein Matrixobjekt durch das Importieren von Daten über die Formel Werte setzen mit numerischen Werten gefüllt wird oder wenn Anwender ein Bild in Daten umwandeln, dann ist der Matrixobjekttyp Daten. Das wird rechts oben im Matrixfenster mit dem Datensymbol angezeigt.

Anwender können zwischen den Fensteransichten Bildmodus und Datenmodus mit Hilfe des Menüs Ansicht hin- und herschalten je nach dem, ob die Matrix ein Bild oder Daten enthält.

Sie können Matrixobjekte als Miniaturbilder oder -- mit einem Schieber -- als eine Reihe von Bildern direkt in der Matrix anzeigen. Weitere Informationen finden Sie unter Miniaturbilder und Schieber.

Wenn die Matrix ein Bild enthält

Die Ansicht Datenmodus zeigt in jeder Zelle eine Zahl an oder eine Reihe von Zahlen, die jedem Pixel des Bildes entsprechen. Daher enthält zum Beispiel jede Zelle, wenn ein RGB-Bild importiert wurde, drei Zahlen wie 70 123 25. Wenn ein graustufiges Bild importiert wurde, enthält jede Zelle eine einzelne Zahl wie 25186 oder 210.

Die Ansicht Bildmodus zeigt das Bild einfach so an, wie es in einer externen Anwendung angezeigt würde.

Mit Bildmatrizen können einfache Bildverarbeitungs- und Transformationsoperationen durchgeführt werden, die im Menü Bild verfügbar sind. Die meisten numerischen Operationen wie 2D-FFT sind jedoch nicht verfügbar, bis das Bild in Daten umgewandelt wurde. Der Menüpunkt Bild: Konvertierung: In Daten konvertieren kann zum Umwandeln von Bild in Daten verwendet werden.

Wenn die Matrix Daten enthält

Die Ansicht Datenmodus zeigt die numerischen Wert in jeder Zelle an.

Die Ansicht Bildmodus zeigt die Darstellung eines 256-stufigen Graustufenbildes der Daten in der Matrix an. Dem Minimum der Datenwerte wird die Farbe Schwarz zugewiesen, dem Maximum der Datenwerte die Farbe Weiß und allen anderen Werten werden mit Hilfe einer linearen Werteskala die Grautöne dazwischen zugeordnet. Beachten Sie, dass die Datenwerte aus jeder Art von Zahlenbereich sein können, wie z.B. Bruchzahlen aus den Ergebnissen einer Berechnung, und nicht auf positive ganze Zahlen beschränkt sind.

Die Minimum- und Maximumwerte, die für die graustufige Abbildung verwendet werden, können mit Hilfe der Systemvariablen z1 und z2 geändert werden. Das Skriptfenster könnte also geöffnet und Befehle wie unten eingegeben werden, um die graustufige Abbildung zu ändern.

z1=10; z2=35;

Durch Ändern der z1- und z2-Werte können Anwender die Datenmatrix mit dem gewünschten Kontrast anzeigen. Wenn also zum Beispiel ein graustufiges 16-Bit-TIFF-Bild mit sehr kleinen Intensitätswerten in jedem Pixel in ein Matrix importiert wird, erscheint es möglicherweise als schwarz/leer, da der standardmäßige Typ der Matrix auf Bild eingestellt ist. Konvertiert der Anwender aber das Bild in den Datentyp, kann der Anwender z1 und z2 auf die gewünschten Werte einstellen, um die verfügbaren Daten detaillierter anzuzeigen.

Anzeigepalette ändern

Wenn eine Matrix Daten oder ein graustufiges 8-Bit-Bild enthält und im Bildmodus angezeigt wird, können Sie eine unterschiedliche Anzeigepalette anwenden. Wählen Sie Matrix: Palette auf Farbabbildung anwenden und verwenden Sie die Schaltfläche zum Durchsuchen im Dialogfeld, um eine andere Palettendatei auszuwählen.
