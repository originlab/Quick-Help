FAQ-1146 Origin fügt immer ein MathType-Objekt in ein Diagramm ein, obwohl ich AxMath auswähle, wenn ich die Schaltfläche Objekt einfügen verwende. Was soll ich tun?

Fail-to-Insert-AxMath-Object-MathType

Letztes Update: 16.08.2022

Wenn Sie sowohl MathType 7.x als auch AxMath auf dem gleichen Computer installiert haben, werden Sie merken, dass das Hilfsmittel "OLE-Objekt einfügen" in einer anderen Anwendung wie Origin oder Word durch MathType besetzt wird. Wenn Sie zum Beispiel die Schaltfläche Objekt einfügen in Origin verwenden, um das OLE-Objekt in ein Diagrammfenster einzufügen, und Sie wählen AxMath im Dialog Objekt einfügen, fügt Origin immer das MathType-Objekt ein.

Das Problem wird dadurch verursacht, dass MathType 7 und höher den Key AutoConvertTo während der Installation im Register setzen.

Um dieses Problem zu beheben:

  1. Klicken Sie auf die Startschaltfläche von Windows. Geben Sie im Textfeld Ausführen Folgendes ein: 
    regedit

    Der Registry Editor wird geöffnet.

  2. Navigieren Sie im Registry Editor zum AxMaths Key 
    Computer\HKEY_CLASSES_ROOT\WOW6432Node\CLSID\{B18C2BCC-4E79-436A-A2A5-A7F8D25A9A28}

    Darunter befindet sich der Key AutoConvertTo. Sie sehen, dass sein Value auf MathType gesetzt wurde.

  3. Löschen Sie den Key AutoConvertTo und versuchen Sie erneut, das Objekt einzufügen. Das Problem sollte damit behoben sein.

Schlüsselwörter:Microsoft, AutoConvertTo, OLE, Objekt einfügen, MathType, AxMath
