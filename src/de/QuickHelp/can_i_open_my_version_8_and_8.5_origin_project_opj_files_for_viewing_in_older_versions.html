Mit älteren Dateien in neueren Versionen von Origin arbeiten

Ältere Origin-Dateien, einschließlich Projektdateien, Diagrammvorlagen, Arbeitsmappe und Analysevorlagen sowie andere Origin-Dateitypen, sind alle mit neueren Versionen von Origin kompatibel.

Wissenswerte Fakten:

  • Vor Origin 8 konnten Anwender Origins Standardfarbpalette von 24 Farben bearbeiten

und 16 benutzerdefinierten Farben hinzufügen. Dies hatte Probleme mit gemeinsam genutzten Projektdateien zur Folge, da verschiedene Installationen potenziell verschiedene Farbpaletten verwenden konnten. Die Standardpalette kann man inzwischen nicht mehr bearbeiten. In späteren Versionen ist eine erweiterte Farbunterstützung hinzugefügt worden. Benutzerdefinierte Farben werden in der Datei selbst gespeichert und erlauben auf diese Weise ein besseres Teilen der benutzerdefinierten Farben.

  • Vor Version 4 speicherte Origin Projekte mit einer .ORG-Erweiterung. Diese Dateityp

wurde später durch die Projektdatei (OPJ) ersetzt. Um .ORG-Dateien in den neuesten Versionen von Origin zu öffnen, wählen Sie Datei: Öffnen und navigieren Sie zum Speicherort Ihrer Datei. Die Auswahlliste Dateityp enthält nicht "Alle Dateien" oder ".ORG". Um Ihre Datei zu sehen, müssen Sie "*.*" oder "*.ORG" im Feld Dateiname eingeben und Enter drücken. Wählen Sie Ihre Datei und klicken auf Öffnen. Sobald sie geöffnet ist, empfehlen wir das erneute Speichern der Datei als ein Origin-Projekt (OPJ).

Mit neueren Dateien in älteren Versionen von Origin arbeiten

Ab Origin 2018 wurde eine neue Reihe von Origin-Dateitypen eingeführt, deren Dateierweiterungen ein "u" am das Ende anhängen, wie z. B. OPJU für Projektdateien, OGGU für Diagrammfenster, OTPW für Diagrammvorlagen etc. Diese neuen Dateitypen können nicht in Origin 2017 oder einer älteren Version geöffnet werden.

Es wird jedoch nicht empfohlen, auch nicht für die älteren Dateitypen (OPJ, OGG, OTP etc.), Origin-Dateien mit einer älteren Version von Origin zu bearbeiten und neu zu speichern, die mit einer späteren Origin-Version erstellt wurden. Sie können neuere OPJ-Dateien in älteren Versionen von Origin öffnen, wenn Sie einfach numerische Daten und Diagramme anzeigen möchten. Sie könnten allerdings auf folgende Schwierigkeit treffen:

Leere Fenster
Folgendes wird in älteren Versionen nicht unterstützt. Daher kann es sein, dass leere Fenster in der älteren Version geöffnet werden.
  • Berichtsblätter mit eingebetteten Tabellen und Diagrammen werden ab Origin 8.0 unterstützt.
  • 3D-OpenGL-Diagramme werden ab Version 9.0 unterstützt.
  • Neue Diagrammtypen wie XYZ-Kontur und r(X) Theta(Y) Z Polar Kontur werden ab Origin 8.0 unterstützt. Heatmap wird ab Version 2015 unterstützt. Um eine Liste der neuen Diagrammtypen nach Version zu sehen, besuchen Sie bitte diese Seite, Abschnitt Diagrammtypen.
Fehlermeldungen
Beim Öffnen einer OPJ- oder OGG-Datei in einer älteren Version wie 7.5. Fehlermeldungen können im Skriptfenster gezeigt werden. Solange die Abbildungen korrekt aussehen, können Sie sie ignorieren:
Projekt enthält nicht erkannte Grafikobjekte...

<*><*><*><*><*><*> Name storage error.

<*><*><*><*><*><*> Syntax storage error.

Calling member of unattached wrapped class
Schloss ist beschädigt
Wenn die Projektdatei der neueren Version Analysefunktionen enthält, die nicht von der älteren Version unterstützt werden, und der Modus der Neuberechnung auf Auto oder Manuell gesetzt ist, wird das Schloss in der älteren Version beschädigt und die automatische Neuberechnung geht verloren.
Arbeitsmappe mit mehreren Blättern kann in Version 7.x nicht korrekt angezeigt werden
Wenn eine Arbeitsmappe mehrere Blätter enthält, ist in älteren Versionen von Origin wie Origin 7.5 nur das letzte Blatt sichtbar. Wenn Sie also Daten in allen Blättern ansehen möchten, müssen Sie die Blätter beim Speichern in Origin 8.x in mehrere Arbeitsmappen separieren. Dazu können Sie einfach auf den Blattnamen klicken und es aus der Mappe herausziehen. Wenn Sie viele Mappen haben, können Sie das Menü Worksheet: Arbeitsmappen teilen verwenden.
Headerzeilen der Metadaten und Parameter werden nicht in Version 7.x angezeigt
Die folgenden Funktionen der Arbeitsmappe werden ab Origin 8.0 unterstützt. Sie können nicht angezeigt werden, wenn sie in Version 7.x geöffnet werden.
  • Arbeitsblattverknüpfungen und eingebettete Objekte (Diagramme, Bilder etc.)
  • Parameterheaderzeilen (einschließlich Benutzerdefiniert)
  • Formatierungsoptionen in Arbeitsblattzellen
  • Ergebnis durch Methoden zum Extrahieren von Arbeitsblattzellen anstatt alter Standardzeichenketten zum Extrahieren wie z. B. col(B)[i]>0.
Tabellenobjekte in einem Diagramm werden nicht in Version 7.x angezeigt
Tabellenobjekte in einem Diagramm werden in Versionen wie Origin 7.5 oder älter nicht angezeigt, obwohl die Daten in diesen verborgenen Arbeitsblättern verfügbar sind, vorausgesetzt, dass es statische Daten sind und keine Links.
Numerische Ergebnisse sind im Ergebnisfenster von Version 7.x nicht sichtbar
Das Format des Ergebnisfensters wurde in Version 8.0 geändert. Alle numerischen Ergebnisse, die im Ergebnisfenster in neueren Versionen gespeichert wurden, sind in älteren Versionen nicht sichtbar. Sie müssen die numerischen Ergebnisse in einem Notizfenster oder in einer separaten Arbeitsmappe als numerische Werte speichern, um sie in älteren Versionen zu betrachten.
Paketdatei OPX wird nicht in Version 7.x erkannt
Der erweiterte Name für Paket lautet ab Origin 8.0 OPX im Unterschied zu OPK in Version 7.x. OPK-Dateien können noch immer in Origin 8 verwendet werden, aber OPX-Dateien werden nicht in Version7.x erkannt.
Daten bestehen aus vielen langen Zeichenketten, die in Versionen älter als 8.1 möglicherweise nicht angezeigt wurden
Eine Funktion, die die Dateigröße signifikant reduziert, deren Daten aus vielen langen Zeichenketten bestehen, wurde ab Origin 8.1 hinzugefügt. Diese ab Origin 8.1 gespeicherten Dateien zeigen womöglich nicht alle Daten an, wenn sie in älteren Versionen geöffnet werden. Um ältere Versionen zu unterstützen (zu Lasten einer gestiegenen Dateigröße), setzen Sie die Systemvariable @LSSP = 100.
