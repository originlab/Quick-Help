FAQ-849 Kann ich die Zellennotation in meinen Arbeitsmappen ausschalten bzw. selektiv steuern?

Turn-off-spreadsheet-cell-notation

Letztes Update: 08.01.2019

 

Mit Origin 2017 wurde eine vereinfachte "Zellennotation" (SCN) eingeführt, die in den Spaltenformeln in Werte setzen und F(x)= verwendet wird. Diese Notation vereinfacht das Schreiben von Spaltenformeln und unterstützt Referenzen auf andere Blätter und Mappen, aber es gibt sicherlich Gelegenheiten, in denen Sie die ältere Notation "col()/wcol()" bevorzugen (z. B. wenn Sie Skripte verwenden, die Spaltenkurznamen umbenennen). Diese FAQ fasst zusammen, wie Sie die Verwendung der Zellennotation in Ihren Arbeitsmappen und Projekten steuern.

Vereinfachte Zellennotation (SCN) für eine einzelne Arbeitsmappe ein- bzw. ausschalten

Seit ihrer Einführung wurde SCN standardmäßig für neue Arbeitsmappen aktiviert. Die Versionen Origin 2017 bis Origin 2019 zeigten dieses Symbol in der oberen linken Ecke des Arbeitsblatts an, um darauf hinzuweisen, dass die vereinfachte Zellennotation SCN aktiviert war. Seit Origin 2019b wird das Symbol per Standard nicht mehr angezeigt, um neuen Funktionen mehr Platz zu bieten. Die SCN bleibt jedoch wie bei den Vorgängerversionen für neue Arbeitsmappen aktiviert.


Um die Zellennotation via die Benutzeroberfläche auszuschalten ...

Klicken Sie mit der rechten Maustaste auf die Titelleiste der Arbeitsmappe und wählen Sie Einstellungen. Deaktivieren Sie auf der Registerkarte Einstellungen das Kontrollkästchen Zellennotation für Tabellenkalkulationsblatt (in der Mitte des Dialogs).


Um sie via LabTalk zu steuern, verwenden Sie ...

page.xlcolname = 0; // 0 = off, 1 = on.

Mit der LabTalk-Systemvariable @RCN die Zellennotation steuern

Origin 2017, @RCN = 7 (Standard): Die vereinfachte Zellennotation ist per Standard für neue Mappen eingeschaltet, die über die Symbolleistenschaltfläche Neue Mappe erstellt wurden, oder neue Mappen, die beim Dateiimport erstellt wurden, wenn die vereinfachte Zellennotation in der aktiven Arbeitsmappe eingeschaltet ist. Zusätzlich ist bei neuen Arbeitsmappe, die von den meisten auf X-Funktionen basierenden Analysedialogen erstellt wurde, die vereinfachte Zellennotation eingeschaltet, unabhängig davon, ob sie in der aktiven Arbeitsmappe aktiviert ist (die Arbeitsmappe mit den Daten, die Sie analysieren).

Origin 2019, @RCN = 15 (Standard): Zusätzlich zu diesen für Origin 2017 beschriebenen Fällen ist in Origin 2019 und neueren Versionen die SCN für Fenster eingeschaltet, die mit win -t oder newbook erstellt wurden. Dieses neue Standardverhalten funktioniert möglicherweise nicht bei Ihnen -- zum Beispiel wenn Sie Skripte haben, die den Spaltenkurznamen ändern (siehe beispielsweise FAQ-872).

Wenn Sie aus diesem Grund die vereinfachte Zellennotation beim Erstellen neuer Fenster mit win -t oder newbook nicht aktiviert haben möchten, dann sollten Sie den Wert von @RCN=7 im Dialog Systemvariablen festlegen. Jedes Mal, wenn Origin ausgeführt wird, wird der Wert von @RCN abgelesen und Origin behandelt Fenster, die mit win -t oder newbook erstellt wurde, wie in Origin 2017.


Bit-Werte von @RCN:

  • 0 = Die Zellennotation ist für alle neuen Mappen ausgeschaltet.
  • 1 = Die Zellennotation ist für neue Mappen, die über die Symbolleistenschaltfläche Neue Arbeitsmappe erstellt wurden, eingeschaltet.
  • 2 = Das Verhalten der Zellennotation folgt beim Importieren der aktiven Mappe; wenn die Zellennotation in der aktiven Mappe ausgeschaltet ist, ist sie in allen neuen Mappen ausgeschaltet. Wenn Sie in der aktiven Mappe eingeschaltet ist, ist sie in allen Mappen eingeschaltet.
  • 4 = Die meisten X-Funktionen zur Analyse haben ein Kontrollkästchen Automatische Tabellenzellennotation bei neuer Mappe, das im Modus Baumansicht angezeigt wird. Wenn dieses Kontrollkästchen und dieser Bit aktiviert sind, wird für jede neue Arbeitsmappe, die mittels Ausgabe einer X-Funktion erzeugt wird, die SCN aktiviert sein (Automatisch Ein), unabhängig von dem Status der Zellennotation der aktiven Mappe (gilt nicht für wsplit, wsplit_book, merge_book, wmove_sheet).
  • 8 = in 2019 hinzugefügt. Die Zellennotation ist für alle Arbeitsmappen, die mit dem LabTalk-Befehl win -t oder der X-Funktion newbook erstellt wurden, eingeschaltet.
Hinweis: Bit-Werte sind Zusätze (z. B. bedeutet der Standardwert von @RCN = 7, dass die Zellennotation automatisch eingeschaltet ist, damit die Analysehilfsmittel übereinstimmen (+4), der Import entspricht der Einstellung der aktiven Mappe (+2) und die Zellennotation ist für alle neuen Arbeitsblätter, die über die Symbolleiste (+1) erstellt werden, aktiviert). Aber sie ist ausgeschaltet für alle Arbeitsmappen, die mit dem LabTak-Befehl win -t oder der X-Funktion newbook erstellt wurden.

Zellennotation global ausschalten

Wenn Sie den Wert von @RCN bearbeiten, steuern Sie die Verwendung der Zellennotation für neue Arbeitsmappen. Es gibt jedoch eine Methode zum Ausschalten der Zellennotation für jede vorhandene Mappe im Projekt und, optional, zum Ausschalten der Zellennotation für alle neue Mappen.

Ausschalten der Zellennotation für jede im Projekt vorhandene Mappe:
Öffnen Sie Ihr Skript- oder Befehlsfenster, geben Sie Folgendes ein und drücken Sie Enter:

doc -e WW {page.xlcolname=0}

Beachten Sie, dass Sie die Zellennotation mit Hilfe von page.xlcolname=1 einschalten können. Siehe die LabTalk-Objekteigenschaft page.xlcolname.

Ausschalten der Zellennotation für alle im Projekt vorhandenen Mappen UND für alle neuen, in Projekten erstellten Mappen:
Öffnen Sie Ihr Skript- oder Befehlsfenster, geben Sie Folgendes ein und drücken Sie Enter:

ClrX

Das Ausführen des Makros ClrX hat zwei Folgen:

  • Das Skript doc -e WW {page.xlcolname=0} wird ausgeführt, das oben beschrieben wird. Die Zellennotation wird für jede Arbeitsmappe im Projekt deaktiviert.
  • Die LabTalk-Systemvariable @RCN wird zum Dialog Systemvariablen festlegen hinzugefügt und der Wert auf 0 gesetzt. Die Zellennotation wird für alle neuen Arbeitsmappen deaktiviert. Dies wird auf alle Origin-Sitzungen angewendet (es sei denn/bis Sie @RCN löschen oder seinen Wert im Dialog Systemvariablen festlegen ändern).

Beachten Sie, dass Sie die Effekte der Ausführung dieses Makros rückgängig machen können durch:

  1. Ausführen des Befehls doc -e WW {page.xlcolname=1}.
  2. Öffnen des Dialogs Systemvariablen und entweder Löschen von @RCN oder Festlegen ihres Werts auf 1.
