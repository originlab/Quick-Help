FAQ-1128 Was soll ich tun, wenn Origin sich beim Arbeiten mit sehr großen Datensätzen stark verlangsamt?

How-to-remove-sparklines-from-slow-Origin-project

Letztes Update: 11.07.2022

Falls Sie in einer Projektdatei einen sehr großen Datensatz haben (viele Arbeitsmappen oder ein Arbeitsblatt enthält viele Spalten/Zeilen), und die Datensatzspalten Sparklines enthalten (angezeigt oder verborgen), führt jede arbeitsblattbasierte Operation zu einer Verlangsamung oder in Extremfällen sogar zu keiner Reaktion.

Sie können die folgenden Lösungen testen, um das System zu beschleunigen.

Inhalt

  • 1 Origin 2022b und neuer
  • 2 Sparklines für aktuelle Datensätze ausschalten
  • 3 Sparkline für Operationsausgaben ausschalten
  • 4 Weiteres

Origin 2022b und neuer

Seit Origin 2022b haben Sie zwei einfache Optionen zum Entfernen von Sparklines:

  • Zum Steuern der Anzeige (z. B. Erstellen von neuen Sparklines verhindern plus vorhandene Sparklines beim Laden des Projekts verbergen) verwenden Sie die LabTalk-Systemvariable @SPK.
  • Um Sparklines aus dem aktuellen Projekt zu löschen, verwenden Sie den LabTalk-Befehl delete. Öffnen Sie das Skriptfenster (Fenster: Skriptfenster), geben Sie folgenden Befehl ein und drücken Sie Enter:
del -spk


Benutzer der älteren Versionen von Origin können eine der folgenden Lösungen verwenden.

Sparklines für aktuelle Datensätze ausschalten

Führen Sie die folgenden Skripte aus, um alle Sparklinediagramme zu löschen im Projekt zu löschen.

StringArray sa; doc -e P { if (%H == "sparkline*") sa.Add(%H); } int nn = sa.GetSize(); for (int ii=1; ii<=nn; ii++) { win -c %(sa.GetAt(ii)$); }

Sparkline für Operationsausgaben ausschalten

Falls das Bearbeitungs-/Analysehilfsmittel, das Sie für das aktuelle Arbeitsblatt bzw. die Arbeitsmappe verwenden, eine große Anzahl neuer Arbeitsblätter/-mappen erstellt, z. B. Arbeitsblatt aufteilen, verlangsamt sich die Systemgeschwindigkeit, da die Sparklines standardmäßig aktiviert sind.

Um das System zu beschleunigen, deaktivieren Sie bitte die Sparklines im Operationsdialog.
