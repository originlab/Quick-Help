FAQ-915 Warum wird mir der Fehler "license daemon: system error code: No such file or directory" beim Start des FLEXlm Lizenzservers auf Linux gemeldet?

No-Such-File-or-Directory-Err-Linux-Concurrent

Letztes Update: 23.04.2023

Wenn das Starten des Origin-Lizenzservers auf einem Linux-Betriebssystem fehlschlägt und Sie die folgende Fehlermeldung in der Log-Datei lmgrd erhalten:

(lmgrd) license daemon: system error code: No such file or directory

Oder die folgenden Informationen in der lmstat-Ausgabe bekommen:

Vendor daemon status (on server_name):
 orglab: The desired vendor daemon is down (-97,121)

Der Grund hierfür ist, dass der Daemon lmgrd nicht den Daemon orglab findet.

Um dieses Problem zu beheben:

  • Prüfen Sie bitte zuerst, ob die Datei orglab.exe existiert oder nicht. Wenn der Orglab-Daemon nicht existiert oder beschädigt ist, laden Sie ihn bitte von unserer Webseite herunter.
  • Stellen Sie sicher, dass orglab.exe zur Ausführung zugelassen ist. Falls nicht, können Sie ein chmod 755 auf orglab.exe ausführen und lmgrd.exe dann erneut starten.
  • Stellen Sie sicher, dass Ihre Servermaschine die erforderlichen Bibliotheken instaliert hat, um die 32-Bit-Binarys auszuführen. Sie können sie folgendermaßen installieren:
    sudo apt update sudo dpkg --add-architecture i386 sudo apt update sudo apt install libc6:i386 libstdc++6:i386 libncurses5:i386 zlib1g:i386 sudo ln -s /lib/i386-linux-gnu/ld-linux.so.2 ld-lsb.so.3
  • Wenn Sie die obenstehenden Prüfungen durchgeüfhrt haben, das Problem aber weiterhin besteht, tun Sie bitte Folgendes:
    1. Halten Sie den FLEXLm-Dienst an, wenn dieser bereits gestartet wurde.
    2. Bearbeiten Sie die Lizenzdatei, um folgende Änderung zu machen:
      VENDOR orglab <vollständiger Pfad zum orglab daemon binary>


      Zum Beispiel:

      VENDOR orglab /opt/OriginLicenseServer
    3. Starten Sie den FLEXlm-Dienst neu.
