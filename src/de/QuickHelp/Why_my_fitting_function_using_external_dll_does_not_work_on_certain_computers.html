FAQ-1077 Warum funktioniert meine Anpassungsfunktion mit externen DLL nicht auf bestimmten Computern?

dll-compatibility

Letztes Update: 19.11.2020

Origin hat nur vcredist_x64_2008, 2010, 2012 und 2015-2019 vorinstalliert. Wenn Sie also eine andere Version von Visual Studio verwenden, zum Beispiel 2013, um die DLL zu erzeugen, kann es sein, dass sie womöglich nicht richtig auf anderen Rechnern funktioniert, wenn vcredist_x64_2013 dort nicht installiert ist.

Um sie in diesen Fällen kompatibel zu machen, muss der Entwickler die Runtime Library auf die Option Multi-threaded (/MT) auf der Seite Configuration Properties: C/C++: Code Generation in den Projekteigenschaften setzen.
