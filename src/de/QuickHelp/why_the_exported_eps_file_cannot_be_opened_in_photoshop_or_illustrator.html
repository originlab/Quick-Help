FAQ-448 Wenn ich Grafiken mit eingebetteten Schriftarten als PDFs in den Remote-Desktop exportiere, fehlen die eingebetteten Schriftarten. Was soll ich tun?

Remote-Export-Embedding-Fonts-as-PDF

Letztes Update: 24.01.2021

Beim Exportieren von Diagrammen mit eingebetteten Schriften wie PDFs in Remote Desktop bemerken Sie vielleicht, das die eingebetteten Schriften in den exportierten PDF-Dateien fehlen.

Dies liegt daran, dass Origin so eingestellt ist, dass es im Allgemeinen keine Schrifteinbettungen bei exportierten PDFs erlaubt, wenn Zugriff über Remote Desktop stattfindet. Sie können die Einbettung der Schriften forcieren (wodurch der Text möglicherweise schlecht gezeichnet wird während des PDF-Exports), indem Sie eine Systemvariable festlegen: @EMRD=0 durch Eingabe von "@EMRD=0;" im Befehlsfenster oder Skriptfenster, um es auszuführen. Unten sehen Sie ein Beispiel, bei dem wir Arial Garamond Pro als eingebetteten Schriftart genutzt haben, während @EMRD=0 festgelegt war. In der exportierten PDF-Datei können Sie sehen:

Wenn dies für Sie funktioniert, können Sie es als ständige Einstellung festlegen. Dazu gehen Sie zum Menü Einstellungen und wählen Systemvariablen. Geben Sie im Dialog EMRD in die Spalte Variable ein und legen Sie in der Spalte Wert 0 fest. Die Variable ist immer aktiviert, es sei denn, Sie löschen sie auf dem Dialog Systemvariablen festlegen oder setzen Sie im Dialog auf 2.
