FAQ-267 Wie beschleunige ich die Anpassungsroutine der benutzerdefinierten Funktion mit Integration?

Call-GSL-to-SpeedUp-IntegrationFit

Letztes Update: 04.02.2015

Wenn Ihre benutzerdefinierte Origin C-Anpassungsfunktionen - Integration mit Hilfe der NAG Library - nicht schnell genug ist, können Sie die GSL Library aufrufen, um die Anpassung mit Integration durchzuführen.

Bitte lesen Sie dieses Tutorial zum Definieren von Origin C-Anpassungsfunktionen.

Bitte lesen Sie Externe DLL verwenden zum Erstellen einer externen DLL-Anpassungsfunktion.


In dem Ordner <Origin Installation Directory>/Samples/Curve Fitting/ befindet sich die .opx-Datei FittingWithExternalDLL.opx.

  1. Ziehen Sie sie per Drag&Drop in Origin, um sie zu installieren.
  2. Zu Ihrem Anwenderdateiordner wird der Ordner Fitting with External DLL hinzugefügt, einschließlich der Beispielcodes, Hilfedokumentation etc..
  3. Die dritte Beispielanpassungsfunktion, die in FitFuncs.c auskommentiert wird, leitet Sie beim Aufrufen der GSL-Bibliothek an, um die Integrationsanpassung mit Hilfe einer externen DLL durchzuführen.

 

Schlüsselwörter: Geschwindigkeit, Integration, DLL, GSL
