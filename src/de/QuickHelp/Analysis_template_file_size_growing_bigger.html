FAQ-976 Warum wird meine Analysevorlagendatei immer größer?

Analysis-template-file-size-growing-bigger

Letztes Update: 07.12.2018

Aufgrund eines Problems in Origin 2018, Origin 2018b und Origin 2019 (das Problem betrifft ausschließlich diese Versionen) könnte es vorkommen, dass Ihre Analysevorlagendateien (.OGW oder .OGWU) jedes Mal größer werden, wenn Sie sie zur Stapelverarbeitung verwenden.

Um dieses Problem zu beheben, setzen Sie die Systemvariable @BST=0, entweder über den Dialog Systemvariablen festlegen, das Befehlsfenster oder das Skriptfenster.

Um die Größe Ihrer Analysevorlagendatei auf ihre ursprüngliche Größe zu reduzieren, befolgen sie diese Schritte:

  1. Öffnen Sie die Analysevorlagendatei (.OGW oder .OGWU) direkt im Menü Datei: Öffnen.
  2. Aktivieren Sie das Arbeitsblatt, das die importierten Daten enthält.
  3. Wählen Sie im Menü Fenster die Option Arbeitsblatt löschen und klicken Sie auf OK im Popupdialog Achtung!!! .
  4. Führen Sie im Befehlsfenster oder Skriptfenster den Code del -r 16; aus.
  5. Wählen Sie im Menü Datei die Option Arbeitsmappe als Analysevorlage speichern und speichern Sie die Vorlage erneut mit der Datei.

Nachdem Sie @BST=0 gesetzt und die Analysevorlagendatei auf ihre ursprüngliche Größe reduziert haben, wird sie nicht wieder wachsen, es sei denn Sie ändern sie selbst.
