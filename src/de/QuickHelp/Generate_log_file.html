FAQ-1105 Wie erzeuge ich eine Protokolldatei?

generate-log-file

Letztes Update: 26.05.2021

Origin erzeugt eine Protokolldatei, die bei der Diagnose der Software und bei Lizenzierungsproblemen hilfreich sein kann. Wenn Sie von einem Kundendienstmitarbeiter oder Mitarbeiter des technischen Supports von OriginLab gebeten werden, eine Protokolldatei zu erzeugen, können Sie die folgende Vorgehensweise verwenden:

  1. Starten Sie Origin.
  2. Geben Sie im Skriptfenster (Fenster: Skriptfenster), folgenden Befehl ein und drücken Sie dann Enter: 
    3. @log=1
  3. Schließen Sie Origin und starten Sie die Software erneut, um die Protokolldatei zu erzeugen.
  4. Wiederholen Sie die zuvor beobachteten Schritte, um eine Fehlermeldung zu erstellen.
  5. Das Protokollieren wird automatisch nach 3 Sitzungen deaktiviert. Navigieren Sie zu der Protokolldatei -- mit dem Namen ORIGIN_LOG_###.TXT und in Ihrem Anwenderdateiordner gespeichert (Hilfe: Ordner öffnen: Anwenderdateien).
  6. Hängen Sie die Protokolldatei an das existierende Support Ticket an. Wenn Sie kein Support Ticket haben, erstellen Sie hier eines.

In Origin 2019 und älteren Versionen

  1. Öffnen Sie das Skriptfenster (Fenster: Skriptfenster) und geben Sie Folgendes ein (drücken Sie zum Ausführen Enter):
    debug_log 2
    Das Protokollieren wird gestartet, und der Speicherort der Protokolldatei wird im Skriptfenster ausgegeben.
  2. Zum Ausschalten des Protokollierens führen Sie folgendes Skript aus:
    3. debug_log 0
  3. Um den Protokollstatus sowie Name und Speicherort der Debug-Protokolldatei zu sehen, führen Sie Folgendes aus:
    4. debug_log
  4. Senden Sie uns die Protokolldatei, wie unter Schritt #6 beschrieben.

Schlüsselwörter:Lizenzfehler, beschränkt auf 3 Minuten, abgelaufen
