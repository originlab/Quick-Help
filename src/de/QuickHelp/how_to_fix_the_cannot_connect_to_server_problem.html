FAQ-670 Wenn ich Origin auf einem Clientrechner starte und den korrekten Hostnamen des FLEXlm-Servers eingebe, wird diese Information nicht akzeptiert und der Dialog der Lizenzinformationen erneut geöffnet. Wie behebe ich dieses Problem?

troubleshoot-server-connection-problem

Letztes Update: 04.02.2015

  1. Stellen Sie sicher, dass der FLEXlm-Lizenzdienst von Origin ordnungsgemäß läuft. Um dies zu tun:
    a. Führen Sie auf dem FLEXlm-Lizenzdienstcomputer LMTOOLS aus.
    b. Klicken Sie auf der Registerkarte Serverstatus auf "Perform Status Enquiry" (Statusanfrage durchführen).
    Sie sollten Lizenzserver, Vendor Deamon und Usage Output sehen, die alle erfolgreich ausgeführt werden. Sollten sie das nicht, kontaktieren Sie Ihren Netzwerksystemadministrator oder den technischen Support von Origin.
  2. Stellen Sie sicher, dass der Computer, auf dem das Origin-Netzwerkelement installiert ist, eine Verbindung zum FLEXlm-Servercomputer hat. Dazu öffnen Sie den Windows Explorer und prüfen, ob Sie zu dem Computer navigieren können, der den Origin-FLEXlm-Lizenzdienst ausführt.
  3. Sollte das Problem weiterhin bestehen, überprüfen Sie, ob der Computer, auf dem der FLEXlm-Lizenzserver installiert ist, eine "Firewall" hat, die ohne "Ausnahmen" aktiviert ist.
    Windows XP, 7 und 8 (möglicherweise auch weitere Windows-Betriebssysteme) haben ihre eigene "Firewall" für Internetverbindungen. Wenn diese ohne Ausnahmen aktiviert ist, wird die Verbindung von Origin, das auf anderen Computern installiert ist, mit dem FLEXlm-Dienst verhindert. Um dies zu prüfen, öffnen Sie den Dialog der Windows-Firewall auf dem FLEXlm-Servercomputer: Klicken Sie auf den Link ”Ein Programm oder Feature durch die Windows-Firewall zulassen” auf der linken Seite und wählen Sie dann LMGRD.EXE und ORGLAB.EXE in der Liste aus. Nachdem Sie diese Programme als ”Ausnahmen” hinzugefügt haben, sollten die Firewall sie durchlassen.
  4. Wenn das Problem weiterhin besteht, wenden Sie sich an Ihren Netzwerksystemadministrator und finden Sie heraus, ob es eine Firewall (eine andere als die unter 3 oben erwähnt) zwischen dem Computer des Netzwerkelements und dem FLEXlm-Lizenzdienstcomputer gibt. Wenn es eine Firewall gibt, müssen Sie Ihren Systemadministrator bitten, einen Port in der Firewall zu öffnen und Sie über die Portnummer zu informieren.
    Sobald Sie die Portnummer kennen, müssen Sie die Lizenzdatei des FLEXlm-Lizenzservers bearbeiten, um diesen Port hinzuzufügen. Weitere Einzelheiten lesen Sie bitte hier nach.
  5. Wenn die Maßnahmen 1 - 4 das Problem nicht lösen können, sollten Sie feststellen, ob eine andere Software auf dem gleichen Computer, auf dem Sie auch Origin ausführen, FLEXlm-Verwaltung verwendet und ob diese andere Software eine Umgebungsvariable LM_LICENSE_FILE erstellt hat. Um das Vorhandensein dieser Umgebungsvariablen festzustellen, öffnen Sie Systemsteuerung | System | Einstellungen ändern auf dem Client-Computer. Wechseln Sie zur Registerkarte Erweitert und klicken Sie dann auf die Schaltfläche Umgebungsvariablen. Sehen Sie nach, ob eine Variable LM_LICENSE_FILE aufgeführt wird. Falls ja, sollten Sie das Problem mit einem Origin-Patch lösen.
