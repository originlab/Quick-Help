FAQ-677 Wie zeichne ich mit Hilfe meiner eigenen Diagrammvorlage mit LabTalk-Skript?

Letztes Update: 05.07.2017

Graph-Template-LabTalk

Sobald Sie das Diagramm Ihren Wünschen gemäß bearbeitet haben, sollten Sie es als eine Diagrammvorlage speichern, die Sie für zukünftige Diagramme verwenden können. Sie können Ihre Daten mit Hilfe der Menüoption Zeichnen: Benutzerdefiniert oder über Skript mit Hilfe des Befehls worksheet zeichnen.

Abhängig davon, ob Als klonbare Vorlage markieren beim Speichern aktiviert ist oder nicht, unterscheidet sich die LabTalk-Syntax zum Zeichnen mit der Vorlage:

  • Die Vorlage ist nicht als klonbare Vorlage markiert. Die allgemeine Syntax zum Zeichnen mit einer Vorlage ist: 
    worksheet -p n templatename

    Der Wert von n bezieht sich auf einen besonderen Diagrammtyp, der dem Diagrammtyp entsprechen sollte, von dem Ihre Vorlage abgeleitet wurde. Alle möglichen Werte für die Zeichnungsoption können in den IDs für die Diagrammtypen nachgesehen werden.

    Markieren Sie zum Beispiel die Daten und führen Sie dann aus:

    worksheet -p 200 MyTemplate;

    Sie zeichnen die aktuelle Auswahl aus dem Arbeitsblatt in Ihre Vorlage, die auf einem Liniendiagrammtyp (200) basierte.

  • Die Vorlage ist als klonbare Vorlage markiert. Die allgemeine Syntax zum Zeichnen mit einer Vorlage ist: 
    worksheet -pa? templatename

    Für klonbare Vorlage müssen vorher keine Daten markiert werden. Aktivieren Sie nur das Arbeitsblatt und führen Sie Folgendes aus:

    worksheet -pa? MyMulti_Panel.otp

    Die klonbare Vorlage weiß, welche Daten zu zeichnen sind, und zeichnet die Daten im Arbeitsblatt.
