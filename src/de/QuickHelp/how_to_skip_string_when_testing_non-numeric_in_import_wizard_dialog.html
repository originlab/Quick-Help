FAQ-425 Wie überspringe ich Zeichenketten beim Testen von nichtnumerischen Daten im Dialog des Importassistenten?

Letztes Update: 04.02.2015

Skip-String-in-Import-Wizard


Wenn Sie ASCII-Daten mit dem Importassistenten importieren, gibt es eine Einstellung, mit der Sie prüfen können, wenn sich nichtnumerische Daten in einem numerischen Feld befinden. Wenn nichtnumerische Daten mit numerischen Daten vermischt werden, können Sie dieses Bedienelement verwenden, um zu entscheiden, wie fortgefahren werden soll.

Origin hat eine Variable, die manuell zugewiesen werden muss, um eine Zeichenkette aus einem nichtnumerischen Test auszuschließen. Beispielsweise haben Ihre Daten vielleicht NAN, wo Werte gefehlt haben.

Durch Verwenden dieser Daten als ein Beispiel wird der Test auf nichtnumerische Daten gestartet, wenn er sowohl RUN#2 als auch NAN findet. Wenn unsere Absicht darin besteht, NAN nur als einen weiteren Datenwert zu behandeln, muss die Variable vor dem Import zugewiesen werden.

RUN#1 Production Scrap
1 18,4 0,42
2 NAN NAN
3 7,6 0,01
4 9,2 0,63
5 NAN NAN
6 17,2 0,05
RUN#2 Production Scrap
1 4,4 0,46
2 11 0,25
3 NAN NAN
4 12 0,18
5 5,4 0,98
6 11,8 0,87

Bevor Sie die Datei importieren, führen Sie

__SNUM$="NAN"

im Befehlsfenster aus. NAN wird nicht nicht als Text betrachtet, der neue Daten startet. Danach können Sie die Bedienelemente für Nichtnumerische Daten in einem numerischen Feld auf der Seite Datenauswahl des Importassistenten verwenden, um zu entscheiden, was unternommen werden soll, wenn nichtnumerische Daten - wie RUN#2 - auftreten. In diesem Fall, d.h. ohne den speziellen Ausschluss, würde die Verwendung von Neues Arbeitsblatt öffnen fünf Arbeitsblätter ergeben, während es nur zwei wären mit dem speziellen Ausschluss.

Schlüsselwörter:importieren, Textdaten, nichtnumerische Daten, Zeichenkette überspringen, Text überspringen, Importassistenten
