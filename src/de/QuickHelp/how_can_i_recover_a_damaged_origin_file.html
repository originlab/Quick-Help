FAQ-464 Wie kann ich eine beschädigte Origin-Datei wiederherstellen?

Recover-Damaged-OPJ

Letztes Update: 10.10.2020

Wenn Sie eine Origin-Projektdatei speichern, speichert Origin zuerst eine Kopie der Datei, wie sie beim letzten Speichern bestand, im Backup. Wenn Sie bemerken, dass Ihre Projektdatei beschädigt ist, können Sie vielleicht die zuvor gespeicherte Version der Datei durch Öffnen einer Sicherungsdatei (Backup) wiederherstellen:

  • In Origin 2018b und höher hat jedes Projekt seine eigene Sicherungsdatei. Diese Dateien werden im Ordner \Backup (z. B. unter C:\Users\user\Documents\OriginLab\Backup) gespeichert und erhalten Namen wie MyProject-1234567.opju).
  • In älteren Versionen von Origin befand sich eine einzelne Sicherungsdatei in Ihrem Anwenderdateiordner (z. B. unter C:\Users\user\Documents\OriginLab\2017\User Files) und hatte den Namen Backup.OPJ.

1. Suchen und Öffnen Ihrer Sicherungsprojektdatei

  • In Origin 2018b und höher wählen Sie Hilfe: Ordner öffnen: Project Backup oder Einstellungen: Optionen, klicken auf die Registerkarte Systempfad, kopieren Sie den Pfad \Backup und fügen Sie den Pfad in die Adressleiste des Windows-Dateiexplorers ein.
  • In Origin 2018 und älter finden Sie ein einzelnes Backup.OPJ im Anwenderdateiordner. Öffnen Sie Einstellungen: Optionen, klicken Sie auf die Registerkarte Systempfad und kopieren Sie den Pfad Anwenderdateiordner und fügen Sie diesen Pfad in die Adressleiste im Windows-Dateiexplorer ein.


Wenn Sie die Sicherungsdatei gefunden haben, öffnen Sie sie in Origin. Wenn dies die Datei ist, die Sie suchen:

  • Origin 2018b und höher: Speichern Sie die Datei in einem neuen Speicherort.
  • Origin 2018 und älter: Speichern Sie die Datei in einem neuen Speicherort mit einem neuen Namen (d. h. einem anderen Namen als "Backup.OPJ").

2. Wiederherstellen von Datensätzen aus einem beschädigten Projekt

Falls Methode 1) nicht funktioniert hat, versuchen Sie Folgendes: Öffnen Sie Ihre beschädigte Projektdatei, öffnen Sie anschließend das Skriptfenster (Fenster: Skriptfenster) und geben folgenden Befehl ein:

list s

Drücken Sie Enter. Wenn die Datei nicht zu beschädigt, wird eine Liste der Datensätze ausgegeben. Diese Liste wird ungefähr so wie die Folgende aussehen:

 1  MyData_Time 2*        MyData_Sample1 3*     MyData_Sample2 4   Data2_A 5*    Data2_B 6*    Data2_C

Es werden zwei verschiedene Arbeitsblätter aufgelistet: MyData und Data2. Um diese Datensätze wiederherzustellen, verwenden Sie den Befehl Bearbeiten und erstellen nacheinander alle Arbeitsblätter erneut. Führen Sie zum Beispiel aus:

Edit MyData_Sample1;

Drücken Sie Enter. Dies öffnet das Arbeitsblatt "MyData" einschließlich der drei zugehörigen Datensätze.

Seit Origin 2021 können Sie den folgenden Befehl verwenden, um alle Arbeitsblätter auf einmal wiederherzustellen:

run -oc EditAllSeries;

Leider können keine Diagrammfenster wiederhergestellt werden.

Nach dem "Bearbeiten" der Arbeitsblätter kopieren Sie die Daten von der ersten Instanz von Origin (die beschädigte Datei) und fügen sie in einer zweiten Instanz von Origin (eine nicht beschädigte Datei) wieder ein. Alternativ können Sie die Arbeitsblattdaten der beschädigten Datei exportieren (wählen Sie Datei: Export: ASCII aus dem Hauptmenü) und anschließend die ASCII-Daten in eine neue Origin-Projektdatei importieren.
