1.196 FAQ-914 双击复制到其他软件中的图形，为什么无法重新打开 Origin？

reopen-Origin-by-graph-copied

Last Update: 4/24/2019

如果你从 Origin 2018 或更高的版本中，复制图形粘贴到 Word 或 PowerPoint 等软件，然后将这些软件的文件分享给使用 Origin 2018 之前的版本的用户。当他们双击从 Origin 复制的图形时，无法重新打开 Origin 并显示以下错误消息。

这是因为，在Origin 2018 以及更高的版本中，我们使用了新的文件类型 .OGGU 代替了原来图形文件的 .OGG 。但在旧的版本中，Origin 并不支持这种新的文件格式。因此，图形无法在 Origin 中打开。 如果想了解更多关于新的文件类型，你可以参考这页。

在Origin 2018 以及更高的版本中，你可以使用系统变量 @OPJ 来控制，在默认情况下是否以新的文件类型保存项目文件以及复制图形。

@OPJ=0 以新文件类型保存项目和复制图形。
@OPJ=1 以旧文件类型保存项目，但以新文件类型复制图形
@OPJ=2 以新文件类型保存项目，但以旧文件类型复制图形。
@OPJ=3 以旧文件类型保存项目和复制图形。


如何设置系统变量：

  1. 在菜单上选项 Preferences: System Variables 。
  2. 在 Set System Variables 对话框中，你可以在 Variable 下添加新项 OPJ，然后在 Value 下输入 2 或 3 。 点击OK 保存设置
  3. 然后当你再复制图形页面到其他软件时，就会使用旧的文件类型 OGG。
