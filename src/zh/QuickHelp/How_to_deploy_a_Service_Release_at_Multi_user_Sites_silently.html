6.76 FAQ-1083 如何向多个用户推送并静默安装补丁包？

Deploy-SR-Silently

Last Update: 4/22/2021

注意：Origin 提供 组文件夹 工具来发布文件，可以设立组长（Group Leader），由组长为组成员发布补丁。请参阅这个页面获取更详细信息。

如果您是一个多用户 Origin 的管理者，想要向用户端推送并静默安装同一个版本的补丁包，

  1. 下载补丁，并保存在目标电脑上的任意位置或网络驱动器。
  2. 若要静默安装补丁，可以运行以下脚本，即通过 /S 选项，让补丁在目标电脑上静默运行： 
    <path>\<patch file> /S

    例如，如果补丁保存于 c:\773d-725c-sr0tosr1.exe，您可以运行以下脚本进行安装： 

    c:\773d-725c-sr0tosr1.exe /S

    只要 Origin 已经在目标电脑至少运行过一次，补丁程序将会定位到 Origin 文件夹来打补丁。

  3. 补丁程序完成后，名为 VisualPatchLogSr1.txt 的文件将保存到 Origin 的 \Updates\ 文件夹中。该文件列出了所有补丁文件。
  4. 如果 Origin 还没有在目标电脑运行过，您必须提供初始化文件用于指定 Origin 程序的文件夹位置，即假定您是在 Origin 的 \Updates\ 文件夹以外的其他路径运行补丁。初始化文件必须包含下面的内容： 
    [Variables]
%AppFolder%=<path to Origin program folder>

    假如： 

    [Variables]
%AppFolder%=C:\Program Files\OriginLab\Origin 85\

    在静默安装补丁时，您需要指定这个初始化文件，脚本的语法如下： 

    <path>\<patch file> "/S:<path and initialization file name>"

    例如： 

    c:\773d-725c-sr0tosr1.exe "/S:c:\silentPatch.ini"

    如果不同电脑的 Program Files 位置不同，那么您可以在初始化文件里使用 %ProgramFilesFolder% 符号来指定，如下所示： 

    [Variables]
%AppFolder%=%ProgramFilesFolder%\OriginLab\Origin 85\
