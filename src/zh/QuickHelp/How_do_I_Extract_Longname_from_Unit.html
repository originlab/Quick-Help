2.9 FAQ-416 导入 ASCII 数据时如何分离同一标题行中的长名称和单位？

extract-longname-from-units

Last Update: 4/24/2019


某些情况下变量名和单位位于同一标题行，利用 Origin 中自动检测分隔符在导入数据时可以对同一标题行中的变量名和单位进行分离。

Origin 2018及以后版本

  1. 点击 Data: Import from File: Single ASCII 或 Data: Import from File: Multiple ASCII 菜单. 选择要导入的 ASCII 文件并勾选 Show Options Dialog。 在打开的 impASC 对话框中对要导入的数据进行设置。
  2. 在 Header Lines 分支下的 Composite Header Line No. 中选择 1 - 9 或在框中输入数字来指定要分离的标题行的位置。
  3. 从 Composite Pattern 中选择一种分隔方式，如果选择 <auto>, Origin 将根据已知的分隔符 代码自动检测识别分隔符。

注意： Origin 中的导入向导（Import Wizard）也有类似的功能，具体内容请看下节。

Origin 2015 到 Origin 2017

点击 Data: Import from File: Single ASCII(或 Data: Import from File: Multiple ASCII )菜单使用 Import single ASCII (或 Import Multiple ASCII ) 导入数据。 选择要导入的 ASCII 文件并打开 impASC 对话框，勾选 Import Option: Header Lines 下的 Extract Long Name and Units from Same Line 点击 OK 关闭对话框完成导入， 变量名将自动写入 Long Name 标题行而单位写入 Units 标题行中。

在 导入向导 (Import Wizard)中也可进行类似的操作，可以在 Import Wizard 对话框中勾选标题行 下的 Extract Long Name and Units from Same Line 。
