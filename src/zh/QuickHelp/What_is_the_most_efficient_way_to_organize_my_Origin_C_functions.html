4.37 FAQ-397 什么是保存 Origin C 函数最有效的方法？

Organize-OriginC-Func

Last Update: 7/11/2018

当您开发自己的 Origin C 函数的时候，您肯定会希望这些函数在以后的 Origin 版本中也能使用。所以最好的方式是将这个函数存放于一个子文件夹，而此子文件夹不在 Origin 安装路径下也不在当前用户文件夹下。例如，您可将这些函数存放于电脑的“文档”中的一个子文件夹，并将此文件夹命名为 "My Origin C Functions"。

当您需要使用这些 Origin C 文件的时候，您必须将这些文件添加到 Origin C 工作区，并将其编译和链接。

  • 在 Code Builder 的左侧面板的 User 节点上点击右键并选择 Add Files...，或者选择 File: Open 菜单。注意，请务必勾选在 Open 对话框中的 "Add to Workspace"。
  • 点击 Build 按钮完成编译和链接。
  • 您也可以将 User 节点里的所有 Origin C 文件存为一个 Origin C 工作区文件(*. ocw)。如果在将来您需要再次使用这个文件，您可以通过选择 File: Open Workspace 来打开它。


如果您希望在任意的 Origin 进程都能使用您的 Origin C 函数，您可以在 Code Builder 的左侧面板的 Origin C Workspace 节点上点击右键并选择 Build on Startup。注意，进行此操作前请首先确认您的 Origin C 文件已被添加到工作区。

请参阅此教程了解更多关于保存和访问 Origin C 函数的详细信息。
