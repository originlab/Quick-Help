6.105 FAQ-1160 如果我在启动服务器时看到"tcp_s is bad"错误，我该怎么办？

Tcp-s-Is-Bad

Last Update: 11/16/2023

如果 Origin 许可证服务器无法启动并且您在调试日志中看到以下错误：

tcp_s is bad!!! Exiting

这是因为您在许可证文件中为供应商守护程序"orglab"指定了一个端口号，但该端口已被另一个进程使用。

为了解决这个问题，

如果 Origin 不在防火墙的另一端，则无需为"orglab"指定端口号。要么，确保您使用空闲、未占用的端口。 请参阅 这个页面 了解更多关于如何允许 Origin 通过防火墙访问的信息。

Keywords:FLEXlm 服务器，无法联系，端口号，orglab 退出，状态为 28，EXITING DUE TO SIGNAL 28，并发许可证，lic
