6.72 FAQ-1003 为什么 Origin 无法连接到网络以检查更新等等?

Download-updates-data-connectors

Last Update: 4/14/2023

如果你在以下任何方面遇到问题：

  • 获得帮助文件/检查更新。
  • App Center 以及下载或更新 App。
  • 更新 Learning Center。
  • 使用数据连接器从网络（Origin 2019b 或更高版本）导入数据。


这可能与网络代理服务器的使用有关。 许多组织使用代理服务器来绕过某些受控门户路由通过网络。 默认情况下，Origin可能不遵循代理服务器设置，因此可能无法执行各种功能所需的与网络相关的活动。


从 Origin 2019 开始，您可以更改 Origin 的代理服务器设置以查看是否可以解决问题：

  1. 选择设置：选项菜单（在 Origin 2019 中选择工具：选项菜单）打开选项对话框。
  2. 在选项对话框中，导航到其他选项卡。
  3. 在代理控制组中，有三个选项：
    • 直接连接绕过任何代理服务器-Origin 尝试直接连接到目标。
    • 使用系统配置使用常用网络浏览器或系统上许多其他面向网络的软件所使用的相同代理服务器设置。
    • 使用 HTTP 代理服务器允许指定一个不可更改的代理服务器 IP 地址和端口。
      如果使用要求身份验证的代理服务器，请使用以下语法： 
      [username]:[password]@[server name|server ip]:[port]

      如果用户名或密码包含"百分比编码保留字符"，则将它们替换为百分比编码（注意 ：Origin 2021b支持百分比编码）。 例如，如果username = user@a.com, password = pwd, server name = proxy 和 port = 3129，你可以写 

      user%40a.com:pwd@proxy:3129


强烈建议你首先选择使用系统配置选项，以查看是否可以解决问题（必须重新启动 Origin 才能使新设置生效）。 如果这样不能解决问题，则要在使用 HTTP 代理服务器中输入一个不可变更的代理服务器地址和端口（此后也要重新启动）。 你应该向 IT 部门咨询需要使用的正确地址和端口。

如果在 Origin 中进行这些更改仍不能解决问题，则你的 Windows 或防火墙可能阻止了 Origin。 在这种情况下，应查阅相关防火墙的文档，以获取能使 Origin 通过的方法。
