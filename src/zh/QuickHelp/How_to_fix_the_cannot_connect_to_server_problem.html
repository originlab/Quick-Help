6.41 FAQ-670 我在客户端的机子上打开 Origin 并输入了正确的 FLEXlm 服务器的 HostName，但是 License Information 还是反复弹出。怎么办？

troubleshoot-server-connection-problem

Last Update: 4/27/2020

如果你所使用的 Origin 是网络版（序列号的后7位是 79xxxxx），在输入了正确的 FLEXlm 服务器的 HostName 之后， License Information 对话框又重新打开，请排查以下几个可能的原因:

  1. 检查并确保 Origin 的 FLEXlm 许可证服务器正常运行:
    a. 在 FLEXlm 许可证服务器端，打开 LMTOOLS。
    b. 选择 Server Status 选项卡，点击 Perform Status Enquiry 按钮。
    下方的文本框将会输出一些查询结果，包括服务器名称和 vendor deamon 的信息，以及目前许可证的使用状态。如果你看到的是这样的结果，证明服务器是正常运行的。如果你看到了错误信息，请联系你的网络管理员或者 Origin 技术人员解决。
  2. 确保客户端可以连接到 FLEXlm 服务器端。要检查网络的连接情况，在客户端的电脑上打开 Windows 资源管理器，检测是否能够打开 Origin FLEXlm 服务器端的电脑。
  3. 所有的许可证都已经被使用
    这种情况下，License Information 对话框会提示你，所有的许可证都已经被用掉了。请联系你的 Origin 管理者解决这个问题。
  4. 你所购买的是一个 N 年期的许可证，许可证已经过期了
    这种情况下，License Information 对话框会提示你，FLEXnet 服务器的许可证已经过期。请联系你的 Origin 管理者解决这个问题。
  5. 如果上述检测都没有问题，但 Origin 还是连不上服务器，请检查服务器端是否打开了防火墙，并且 Origin FLEXlm 服务器程序没有被列入”允许通过的程序“名单。
    Windows Vista, 7, 8, 8.1 和 10 都有自带的防火墙，如果防火墙被打开并且 Origin FLEXlm 没有被设置为”允许通过的程序“时，客户端的 Origin 到 FLEXlm 服务器的连接就会被防火墙阻断。要检查防火墙是否被打开，打开 Origin FLEXlm 服务器端电脑上的 Windows Firewall 对话框（Control Panel|System and Security）。点击左边的 Customize Settings 连接，防火墙的状态应该是 ”On“，并且确保 "Block all incoming connections, including those in the list of allowed programs" 没有被勾上。回到 Windows Firewall 的主页面，点击左边的链接 “Allow a program and features through Windows Firewall”，在右边使用 Allow another program 按钮把 LMGRD.EXE 和 ORGLAB.EXE 这两个应用程序（在 FLEXnet 的 \Servers 子文件夹下）加入 “允许通过防火墙的应用” 列表。这样客户端和服务器的通信才不会被防火墙阻断。
  6. 如果问题仍然没有解决，联系你的网络管理员，查看在服务器端和客户端之间是否设置了防火墙（跟第3步中的本机防火墙不同），阻断了二者的通信。如果确实存在防火墙，请让网管在防火墙上为 Origin FLEXlm 设置一个端口号，并将这个端口号告诉你。
    网管设置好端口号之后，你需要在 FLEXlm 服务器上修改许可证，把这个端口号加进去。详细步骤看这里。
  7. 客户端安装的 Origin 的版本或者序列号与服务器端的许可证文件中的版本和序列号不符。
    请联系你的 Origin 管理者解决这个问题。
  8. 如果上述的所有方法都不能解决问题，请检查在这台服务器上是否有其他软件也使用了这个 FLEXlm 来管理许可证，以及其他软件是否创建了一个 LM_LICENSE_FILE 环境变量。要查看是否有这个环境变量，打开客户端的 Windows 的 Control Panel | System | Change Settings。 选择 Advanced 选项卡并点击 Environment Variables 按钮。检查列表中是否有 LM_LICENSE_FILE 变量。如果有，请将其路径改成 Origin 的路径。
