1.200 FAQ-928 为什么在日文/中文 Origin 2017 中的特殊字符在 Origin 2018 或更高版本中会变成乱码呢？

Special-Characters-Gibberish-Japanese

Last Update: 7/25/2018


在日文或中文操作系统中，使用 Origin 2017 创建一个 OPJ 文件，如果其中包含如 Mu(μ), degree(°) 等的特殊字符，再在Origin 2018 打开此文件，特殊字符会变成乱码。

这是因为这些特殊字符是通过 Symbol Map 插入到 Origin 2017 中的，他们是扩展的 ASCII 字符。但在使用日文/中文代码页(code page)的情况下，Origin 2018 Unicode 机制无法正确处理这些字符。

即使用户在使用 Origin 2017，如果将字体从 Arial 改为日语/中文字体，也会因为日文/中文代码页而出现乱码的问题。

因此，我们强烈建议 Origin 2018 及更高版本的用户，使用 Unicode 字符，以避免这种乱码现象。


将特殊字符转换为 Unicode 字符
  • 选择菜单 Window: Command Window，打开 Command Window。
  • 在 Command Window，输入以下脚本，然后按 ENTER 键运行脚本。
@CPGL=1252
  • 在 Origin 2018 或更高版本中加载 OPJ。其中的特殊字符将自动转换为 Unicode 字符。
  • 将OPJ另保存为 OPJU文件
  • 在 Command Window，输入以下脚本，按 ENTER 键以恢复默认设置。
@CPGL=-1
