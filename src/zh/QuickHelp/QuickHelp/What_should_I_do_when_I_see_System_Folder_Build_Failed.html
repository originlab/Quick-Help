6.17 FAQ-19 遇到 "System Folder Build Failed" 或者 "X-Function failed to execute" 这样的错误信息，该怎么解决？

System-Folder-Build-Failed

Last Update: 4/24/2019

如果在打开 Origin 的时候看到 "System Folder Build Failed" 或者 "X-Function failed to execute" 这样的错误信息，尝试如下方法解决：

  1. 选择菜单 Window: Script Window。
  2. 在 Script Window 中输入以下命令，按 Enter 键运行，删除 OCTemp 文件夹。这样，OriginC 就会使用全新的源文件进行编译。 
    del -occlean
  3. 在 Script Window 中输入以下命令，清除注册表中的 User Files Folder 路径。在下次打开 Origin 的时候你可以重新设置用户文件夹。 
    del –path
  4. 重启 Origin。在弹出的 User Files folder 对话框中选择一个全新的路径作为用户文件夹。
注:

在重设了用户文件夹之后，可以根据以下步骤把原来的用户文件从旧的文件夹转移到新的文件夹中：

  1. 选择菜单 Preferences: Transfer User Files...，打开 Transfer User Files 对话框。
  2. 从 Source Location 下拉列表中选择 Specified Folder，勾选 Recursive。把想要转移的文件从左边列表添加到右边，然后点击 Transfer 按钮。


如果上述方法不能解决问题，请打开 Origin 的安装向导并选择 Repair 来修复 Origin（你可以从Windows 的开始菜单找到并打开 Origin "Add or Remove Files" 菜单来打开安装向导）。安装程序运行完毕后，重新启动 Origin，看看问题是否解决了。

如果 Repair 仍不能解决问题，请联系 OriginLab 技术支持部门。
