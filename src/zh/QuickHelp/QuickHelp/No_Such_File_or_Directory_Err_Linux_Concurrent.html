6.70 FAQ-915 当我试图在 Linux 系统下启动 FLEXlm 服务器时，看到这个错误信息 "license daemon: system error code: No such file or directory"。怎么解决？

No-Such-File-or-Directory-Err-Linux-Concurrent

Last Update: 10/25/2023

如果您在 Linux 系统下架设并启动 Origin 的许可证服务器时失败，并在 lmgrd 日志文件里看到以下的错误信息：

(lmgrd) license daemon: system error code: No such file or directory

或者，在 lmstat 的输出中看到以下错误信息

Vendor daemon status (on server_name):
 orglab: The desired vendor daemon is down (-97,121)

这是因为 lmgrd daemon 找不到 orglab daemon 造成的。

要解决这个问题，

  • 请先检查 orglab 文件是否存在。 如果 orglab daemon 文件不存在或已经损坏，请从我们的网站重新下载。
  • 确保 orglab 有执行权限。如果没有，您可以对 orglab 运行 chmod 755，然后重新启动 lmgrd。
  • 确保您的服务器计算机已安装了运行 32 位二进制文件所需的库。您可以运行下面的命令来安装它们： 
    sudo apt update
sudo dpkg --add-architecture i386
sudo apt update
sudo apt install libc6:i386 libstdc++6:i386 libncurses5:i386 zlib1g:i386
sudo ln -s /lib/i386-linux-gnu/ld-linux.so.2 /lib/ld-lsb.so.3
  • 如果您已经试过了上述几个方法但是问题仍然没有解决，请尝试如下解决方案：
    1. 如果 FLEXLm 服务器还在运行中，先暂停它。
    2. 对许可证文件进行如下修改 
      VENDOR orglab <full path to orglab daemon binary>


      例如， 

      VENDOR orglab /opt/OriginLicenseServer
    3. 保存修改。重新启动 FLEXlm 服务器。
