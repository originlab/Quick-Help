用新版 Origin 打开旧版 Origin 创建的项目文件

旧版 Origin 文件（包括项目文件，图形模板，工作簿和分析模板以及其他 Origin 文件类型）都能在较新版本的 Origin 中打开和使用。

需要知道的一些小问题：

  • 在 Origin 8 之前，用户可以编辑 Origin 系统自带的 24 色调色板并添加16种用户定义的颜色。但这会导致共享项目文件出现问题，因为在不同的机器上安装的 Origin 可能使用不一样的调色板。在以后的版本中，增强了对颜色的支持，自定义颜色保存在文件本身中，可以更好地共享自定义颜色。
  • 在 Origin 4 之前，项目文件的格式是 ORG，后来被更新版本项目文件格式 OPJ 取代。要在新版 Origin 中打开 ORG 格式的项目文件，首先选择菜单 File: Open 然后找到文件的位置，在文件类型下拉列表中如果没有 "all files" 或 ".ORG" 选项，那么在 File name 输入框内手动输入

"*.*" 或 "*.ORG" ，然后按 Enter 键。选中文件，单击 Open 打开文件，之后建议你将此文件保存为 OPJ 格式。

在旧版 Origin 中打开新版 Origin 创建的文件

自 Origin 2018 开始，有了新的文件格式，即在OPJ后添加了 "U"，如项目文件格式是 OPJU，图形窗口文件格式是 OGGU，图形模板文件格式是OTPW等。此文件格式不能在 Origin 2017 及以前版本中打开。

然而，对于旧文件格式（OPJ，OGG，OTP 等），我们也不建议使用旧版本的 Origin 来使用和保存那些使用更高版本的 Origin 创建的文件。如果你只想查看数据和图，也可以在旧版本中打开较新版的 OPJ 文件，但你可能会遇到以下情况：

空窗口
以下内容在旧版本Origin中不受支持，因此，这些内容可能会在旧版本中打开时只会看到一个空窗口。
  • Origin 8.0 之后支持的嵌入式表格，图表以及结果表。
  • Origin 9.0 之后支持的 3D OpenGL 图。
  • Origin 8.0 之后的新图形如 XYZ Contour 和 R(X) Theta(Y) Z Polar Contour ,Origin 2015 版本之后的图形如 Heatmap 。要了解各图表类型所支持的版本请参阅 此处 - 图形类型。
错误信息
在旧版本 Origin （如 Origin 7.5）中打开 OPJ 或 OGG 文件时。在脚本窗口中可能会显示下面错误信息。只要数据看起来没什么问题，你可以忽略错误信息：
项目包含未知的图形对象...
<*><*><*><*><*><*> 文件名存储错误。
<*><*><*><*><*><*> 语法存储错误。
Calling member of unattached wrapped class
绿锁 (重计算模式)被破坏
如果新版本的项目文件包含任何旧版本不支持的分析功能，并且重计算模式设置为“自动”或“手动”，那么在旧版本中对应的绿锁会被破坏，自动重计算的结果将丢失。
包含多个工作表的工作簿在 Origin 7.x 中不能正确显示
当工作簿包含多个工作表，在 Origin 7.5 或更早版本中只能显示最后一张工作表。 因此，如果要查看工作簿中所有的数据，则需要在 Origin 8.x 或更高版本中将工作表分成多个工作簿来保存。要执行此操作，你只需单击工作表名称并将其拖出工作簿。如果你有许多工作簿，则可以使用菜单 Worksheet:Split Workbooks 来完成此操作。
在 Origin 7.x 中不显示元数据和参数列标签行
以下在 Origin 8.0 及更高版本中所支持的功能在 Origin 7.x 版本不能被支持
  • 工作表链接和嵌入式对象（图表，图像等）。
  • 参数列标签行（包括用户自定义的）。
  • 工作表单元格的不同格式。
  • 通过 Extracted Worksheet Data 方法提取工作表数据得到的结果 (条件设定不再是旧版中默认的像 col(B)[i]>0 这样的字符串形式时)。
在 Origin 7.x 中不能显示图形中的表格对象
在 Origin 7.5 及更早版本中不能显示图形中嵌入的表格，但这些隐藏的数据仍是可用的，前提是它是具体的数据而不是链接。
在 Origin 7.x 中不能显示结果日志的数值结果
Results Log 的格式自 Origin 8.0 后已更改，因此在较旧版本（如 Origin 7.5）中将不会显示结果日志 (Result Log) 中任何数值结果。如果需要你需要将数值结果放在 Notes窗口或作为数值存放在另一工作簿中，以便在旧版本中查看。
Origin 7.x 版本不能识别 OPX 格式的打包文件
Origin 8.0 以及之后版本的文件打包格式是 OPX，与 Origin 7.x 版本中的 OPK 不同。OPK 文件仍可在 Origin 8 中使用，但在 Origin 7.x 版本无法识别 OPX 文件。
由许多长字符串组成的数据，可能不会在 Origin 8.1 之前版本中完全显示
在 Origin 8.1 版本之后新增了一项功能，该功能显著减少了具很多长字符串的数据的文件大小。在 Origin 8.1 之后版本中保存的此类文件在早期版本中打开时，可能不会显示所有数据。要支持旧版本（以增加文件大小为代价），请设置系统变量@LSSP = 100。
