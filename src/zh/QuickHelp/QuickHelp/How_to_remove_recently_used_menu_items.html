5.53 FAQ-873 如何移除最近使用的菜单项？

Remove-MRU

Last Update: 10/11/2022

如果最近使用了菜单项，则此项将列在菜单列表的底部。它最多可以列出10个项目。此外，如果使用了分析菜单, 则最后使用的对话框主题 (dialog theme) 将作为子项加载到该菜单下，从而可以轻松访问上次使用的设置。如果保存或安装分析对话框主题，它也将作为子项加载。

从 Origin 2017 SR1 版本开始，软件提供了一个系统变量 @MRU 来控制是否在菜单中显示最近使用的项以及最近最近加载的对话框主题 (dialog theme)。此变量共有三个值：

1 = 允许在绘图菜单中显示最近使用的项
2 = 允许在分析菜单中显示最近使用的项
4 = 允许将最后加载的主题显示在菜单中

@MRU 是多个控制值的组合变量。其默认值为 7，表示的是允许在菜单中显示最近使用的项 (@MRU=1 允许绘图菜单显示最近项，以及 @MRU=2 允许分析菜单显示最近项)，同时允许显示最近加载的主题 (@MRU=4)。

问题

在一些系统里，最近使用的菜单项数量的累积会导致打开菜单（分析和统计这两个菜单最容易发生）产生一些迟缓。如果这个问题给你造成了影响，你可以全局性地禁用最近使用的菜单项。

全局性地禁用最近使用的菜单项:

  1. 你可以临时禁用最近使用的菜单项，首先是打开脚本窗口（窗口: 脚本窗口），然后在提示符后输入以下命令并按回车键： 
    @mru = 0
  2. 注意这是临时性的（只在当时会话），一旦重启Origin，最近使用的菜单项又会显示。
  3. 你可以永久性禁用这个特性，首先打开设置系统变量对话框（设置: 系统变量），然后输入mru = 0。
  4. 如果要重新使用最近使用的菜单项这个特性，在设置系统变量对话框里把mru删掉。又或者可以设置相应的位值来显示对应菜单的最近使用的菜单项，可用的位值可参考上面的介绍。
