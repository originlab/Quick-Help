2.112 FAQ-1020 连接和导入数据之间的区别是什么

Data-Connector-Difference-Import-From-File

Last Update: 4/14/2021

连接数据, 或者说 数据连接器, 是在OriginPro 2019b 中新引入的导入框架。两者都将可以将数据导入到工作簿中，但是我们更为建议使用新的连接数据框架。因为数据连接器已有许多改进并将在未来的版本继续进行优化，而原有的导入工具已停止优化。

从用户的角度来看，连接数据和原来的导入数据之间的主要区别是：

  1. 数据连接器会锁定导入的数据，这样可以防止意外更改实际数据。 从2021b的版本开始，您可以点击数据连接器的图标，然后从弹出的菜单中选择解锁导入的数据，这样数据就变为可编辑的了。您也可以通过以下方式控制锁定行为： 修改系统变量 @IPDC 。
  2. 连接器将保持从文件或云到源数据之间的连接。
  3. 使用连接器导入的数据通常不保存在项目中，从而使项目文件变小。 在 2019b，这是默认的行为。在 2020 如果文件小于 500kB 则数据将默认保存在项目中，而且您可以设置 LabTalk 系统变量 @DNS 以及 @FSE 控制默认行为。 任何情况下, 您只需单击数据连接器图标 并消去保存时清除导入数据旁边的复选标记, 即可选择保存导入的数据。
  4. 对于文本文件，使用 CSV 连接器进行导入可以极大地改善用户体验。因为对于大多数 ASCII 文本文件，它无需您指定文件结构且该过程是完全自动的。将其与导入对话框进行比较，在导入对话框中，您需要指定文件结构的详细信息以及诸如日期时间格式之类的东西，最后将设置另存为主题以备将来使用。
