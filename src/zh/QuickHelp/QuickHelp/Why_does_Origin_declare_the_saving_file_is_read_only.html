5.66 FAQ-932 为什么 Origin 提示正在保存的文件为只读文件？

Declare-saving-file-read-only

Last Update: 4/24/2019

有时候，当在 Origin 2018b 或更高的版本中保存文件 (*.opju) 时，Origin 可能会提示正在保存的文件为只读。当出现这种问题时，可能需要检查 Origin 的相关工作文件夹，如 User Files Folder(UFF), Autosave, Unsaved, TempSave, 以及 Backup 文件夹是否位于网盘上。即使已经将用户文件文件夹 (User Files Folder) 的位置更改为本地路径，保存该文件的只读警告仍可能会弹出。

在这种情况下，请尝试将所有文件夹 (UFF, Autosave, Unsaved, TempSave, and Backup folder) 从网络移动到本地驱动器。

更改 UFF, Autosave, Unsaved 和 Backup 文件夹的存储位置
  1. 选择 Preference: Options 菜单以打开 Options 对话框。
  2. 切换到 System Path 对话框，分别选择 User Files Folder, Autosave, Project Backup 和 Unsaved Files 文件夹，然后使用 Change... 按钮将这些路径更改到本地磁盘中。
修改 TempSave 文件夹的存储位置

软件中没有提供更改 TemSave 文件夹的操作界面，所以需要打开 Registry Editor 来：
如果存在 TemSave 的键值，则进行修改，
否则
添加一个键值 TemSave，并对其赋值来定义路径。

可以按照此常见问题解答中的步骤来修改注册表中的键值，或者添加一个字符串来给 Origin 的 TemSave 文件夹定义路径。

注意： 对于 Origin 2018 及之前的版本，只需要将 AutoSave folder 和 User Files folder 从网盘移动到本地磁盘即可避免文件只读的警告。
