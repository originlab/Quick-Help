6.15 FAQ-17 双击 OPJ/OPJU 文件没办法在 Origin 中打开，怎么办？

Double-Click-Open-Failed

Last Update: 7/4/2018

这个问题有可能会出现在 Windows Vista 及以后发布的 Windows 操作系统中，但并非所有电脑都会遇到这个问题，这也不是 Origin 所独有的问题。通常，你会看到类似于 There is a problem sending the command to the program 这样的错误信息。目前我们尚无 100% 有效的解决方法。如果你遇到此类问题，我们建议尝试以下方法。

方法 1

设 Origin 为打开 .opj/opju 文件时的默认应用程序

  1. 右键单击 Origin 图标并选择菜单 Run as administrator，以管理员身份运行一次 Origin，然后关闭它。
  2. 右键单击任一 OPJ/OPJU 文件并选择菜单 Open With:Choose Programs。
  3. 从应用程序列表中选择 Origin 并且勾选 "Always use the selected program for this file"。
  4. 我们从 9.0 版本开始提供64位的 Origin。如果你的系统是64位的并且安装了 32 和 64 位的 Origin。根据部分用户的反馈，如果 OPJ/OPJU 的默认打开方式原本是设置为 Origin##_64.exe，那么更改为 Origin##.exe 可以解决这个问题。反之亦然。

方法 2

清除注册表中的 DDE 设置

  1. 以管理员身份运行 origin，选择菜单 Window:Script Window，打开 Script window。
  2. 运行命令 
    doc -ddde 1
    这里的 ddde 表示禁用 DDE。
  3. 此时会弹出一个窗口提示你关闭 Origin。关闭 Origin，然后以管理员身份运行一次 Origin。再关闭它。

方法 3

检查 Origin.exe 文件的兼容性设置是否被修改。

  1. 关闭 Origin。在 Windows 资源管理器中找到 Origin EXE 文件。
  2. 右键单击 Origin exe 文件并选择菜单 Properties。
  3. 在 Compatibility 选项卡上，检查并确保这页选项卡上所有的复选框都没有勾上。点击 OK。
  4. 右键单击 Origin EXE 并选择菜单 Run as administrator，以管理员身份运行一次 Origin。

对于 Origin 9.0 及以上版本的用户，如果你的操作系统是64位并且安装了32和64位的 Origin，要检查 32 和 64 位两个 exe 文件的兼容性设置（既 Origin##.exe 和 Origin##_64.exe)。


如果你在同一台电脑上安装了不同的 Origin 版本，例如（在 Windows Vista, 7, 8, 8.1 或 10 上）安装了 Origin 9.0 和 Origin 7.5，在双击 OPJ 文件的时候打开的是 Origin 7.5 而不是 Origin 9.0，你可以尝试以下解决方法：

  1. 关闭 Origin 7.5 和 Origin 9.0。
  2. 在 Origin 的安装目录下找到 Origin 9.0 EXE 文件。
  3. 右键单击 EXE 文件并选择菜单 Run as administrator，以管理员身份运行一次 Origin 9.0。
