2.5 FAQ-412 如何自定义拖放导入各种文件类型？

Drag-and-Drop-Import

Last Update: 4/17/2024


当将文件拖放到 Origin 一个新工作簿时，将使用以下规则处理数据文件：

  1. 导入对话框的设置 保存在工作表中，则使用此设置导入文件。
  2. 工作表中没有导入对话框的设置，Origin 会在数据文件夹中查找匹配的 Origin 导入过滤器 (*.OIF)，如果有，则会使用此过滤器导入文件。
  3. 如果数据文件夹中不存在过滤器，Origin 会查看 User Files 文件夹下的 Filters 子文件夹。若存在用户过滤器，并且 过滤器管理中的支持拖放 筛选框已选中，会使用此过滤器导入文件。
  4. 如果 User Files 文件夹下的 Filters 子文件夹没有匹配的过滤器，Origin 会检查是否有已安装的 数据连接器 能匹配该文件类型。有的话，连接器会被用来导入该文件。
  5. 如果没有匹配的连接器，Origin 会在 EXE 路径下的 Filters 子文件夹里查找过滤器。 如果有，并且 过滤器管理 中的系统 支持拖放 筛选框已选中，则会使用系统过滤器。
  6. 如果上述步骤失败，Origin 会提示用户使用 导入向导 导入数据。

如果您对 Origin 导入过滤器(*.oif)感到满意，但又喜欢使用数据连接器的一些优点，您可以通过选择 数据: 连接到文件: 导入过滤器将两者结合起来。对于拖放操作，请尝试将 *.oif 文件放入数据文件夹。

如果你更喜欢以往的拖放处理而不需要数据连接器，请修改系统变量 @DDC 使数据连接器剔除你想要的文件类型。比如，设置 @DDC=2 来禁止 Excel 连接器在拖放导入 .xls(x) 文件时启用。请参阅 系统变量介绍页面 有关设置系统变量和 @DDC 的完整选项说明。
