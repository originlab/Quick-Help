5.59 FAQ-917 从网络位置移动用户文件文件夹后 Origin 2018 不再正常工作

Origin-2018-Moving-User-Files-Folder-from-Network-Location

Last Update: 4/24/2019

注意： 本条问题帮助仅针对 Origin 2018 和 2018 SR1 (不含之前的版本以及 2018b 之后的版本)。

如果将 Origin User Files 文件夹从网络位置移动到电脑上的本地磁盘后发现 Origin 运行存在问题。例如，Origin 在尝试保存文件时可能会收到报错提示该项目文件是只读的。 则可能还需要更改某些 Origin 2018 使用的文件夹。

Origin 2018 除了 User Files 文件夹之外还创建了 4 个文件夹。这些文件夹是与 User Files 文件夹同级的文件夹，名称分别是 Autosave, Unsaved, Backup, 和 TempSave。如果 Origin 操作异常，可能是由于这些文件夹中的一个或多个依然在网盘中，需要将其全部移动到本地磁盘路径中，最好是放到与新的 User Files 文件夹同级的文件夹下。

修改文件夹的位置

当修改了 User Files 文件夹位置后，请执行下面所简述的操作。

Autosave 文件夹

  1. 在 Origin 中，从 Help 菜单下，选择 Open Folder:User Files Folder。
  2. 在 Windows 文件资源管理器窗口中，定位到文件夹层次结构的上一层。
  3. 查看是否有一个名为 Autosave 的文件夹已经存在于 User Files 文件夹的同级位置？
  4. 如果没有，那么在此位置上新建一个名为 Autosave 的空文件。
  5. 回到 Origin 中，从 Preference 菜单中选择 Options。
  6. 在 Options 对话框中，选择到 System Folders 选项卡。
  7. 选中 Autosave 条目，然后点击 Change... 按钮。
  8. 在 Browse Folder 对话框中，定位到在第 4 步中新建的 Autosave 文件夹，然后点击 OK 按钮。
  9. 回到 Options 对话框，检查列表中 Autosave 后面的路径是否正确，然后点击 OK 按钮。
  10. 关闭 Origin。

Unsaved, Backup, 和 TempSave 文件夹

Part I

  1. 在 Windows 文件资源管理器中，定位到本地磁盘中新建的 User Files 文件夹。
  2. 定位到文件夹层次结构中的上一层
  3. 查看 Unsaved, Backup, 和 TempSave 文件夹是否已经存在于 User Files 文件夹的同级文件夹中？
  4. 如果并没有都存在于此位置，则新建与缺少的文件夹同名的空白文件夹。记下每个文件夹的完整路径。

Part II

  1. 打开 Registry Editor
    对于 Windows 10 系统，在 Search 框中输入 regedit 并按下 Enter 键。
    对于之前版本的 Windows 系统，点击 Start 按钮并在运行框中输入 regedit 然后按下 Enter 键。
  2. 在注册表编辑器中定位到如下路径： Computer\HKEY_CURRENT_USER\Software\OriginLab\Origin 9.5\International
  3. 在 Registry Editor 对话框的右侧面板中，将会看到 Autosave 条目。你讲要在此位置为 Unsaved, Backup, 和 TempSave 创建类似的条目。(如果已经存在，则直接修改)。
  4. 若要创建这些条目，则在右侧面板中右击并选择 New:String Value。
  5. 新建的值将被命名为 New Value #1。双击名称，并将其改为 Unsaved, Backup, 或 TempSave (这一步很重要)。
  6. 右击这个词条，然后选择 Modify...。
  7. 在 New String 对话框中，输入对应文件夹的完整路径。请在路径的结尾添加一个反斜杠（\）。
  8. 重复步骤 4-7 为 Unsaved, Backup, 和 TempSave 文件夹创建条目。
  9. 关闭 Registry Editor。

最后检验

启动 Origin 软件并检查之前的错误问题是否依然存在。如果依然存在，请联系 OriginLab 技术支持来获取额外的帮助和指导。
