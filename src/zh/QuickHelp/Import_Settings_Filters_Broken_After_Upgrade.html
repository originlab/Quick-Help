2.90 FAQ-919 升级 Origin 2018 之后 ASCII 导入筛选器或者导入选项不能保存到工作表

import-settings-filter-broken-after-upgrade

Last Update: 7/15/2018

虽然Origin 的 ASCII 导入代码在 Origin 2018 SR0 (b9.5.0.193) 中进行了修改。 但用户使用早期软件版本自定义的导入筛选器文件 (*.OIF) 和工作簿文件 (*.OGW) 导入文件时应当不会出现问题，依然能正常使用。

但如果确实碰到了不能正常使用的问题，即使用导入存有导入设置的筛选器文件 (.OIF) 或者工作簿文件 (.OGW) 进行数据导入时不能正常工作的，则需要将这些文件转换以供 Origin 2018 使用。


将用 Origin 2017 或者更早版本创建的 .OGW 文件转换到 Origin 2018 SR0 及之后的版本

  1. 在 Origin 2018 中打开旧版本创建的 .OGW 文件。
  2. 打开脚本窗口 (Window: Script Window) 并确认以下脚本的返回结果为 2。
    @IA = <Enter>
  3. 重新导入 ASCII 文件。
  4. 重新将工作簿保存为 (File: Save Window As) .OGW 的文件类型。


将用 Origin 2017 或者更早版本创建的 .OIF 文件转换到 Origin 2018 SR0 及之后的版本

  1. 打开脚本窗口 (Window: Script Window) 并确认以下脚本的返回结果为 2。
    @IA = <Enter>
  2. 拖放使用旧版本的 .OIF 导入的 ASCII 文件。
  3. 在工作簿的标题栏上右击并选择 Show Organizer。
  4. 点击以展开 Imported Files 节点，然后右击导入的文件并选择 Make Filter。
  5. 在 Origin 2018 或更高的版本中保存筛选器。
