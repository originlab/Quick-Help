5.62 FAQ-923 编辑 Word 中的 Origin 图形时弹出错误提示: "The program used to create this object is Origin95"

Error-editing-graph-in-Word-object-is-Origin95

Last Update: 4/24/2019

当使用 Edit: Copy Page 并将图形粘贴到 Word 中，此后可以在 Word 文档中双击图形进行编辑。所使用的是对象链接与嵌入 (OLE) 机制，Word 将这个对象标记为 Origin 图形，然后在 Origin 中载入该图形，因此可以在 Origin 环境中编辑这个图形，然后返回文档 (Exit and Return to Document)。然后会发现 Word 文档中的图形也会相应的更新。

Origin 2018 中引入了新的 Unicode-兼容文件类型，并将其包含在了 Origin 项目 (OPJU) 和 Origin 图形 (OGGU) 中。如果使用的时 Origin 2018 或更高版本的软件在 Word 文档中嵌入 Origin 图形，然后使用 Origin 2017 及其之前的软件来打开文档中的图形，则无法正常打开，同时会弹出如下错误提示： "The program used to create this object is Origin95. That program is either not installed on your computer or it is not responding ..."

然而，除了将图形作为图片插入到 Word 之外，Origin 2018 没有别的方法可以解决这个问题，而从 Origin 2018b 开始，用户可以通过设置 系统变量 @OPJ 来在复制图形页面时持续使用以前的 OGG 文件类型。

若要配置 Origin 2018b 及之后的版本在复制页面时将旧版本的 OGG 文件格式复制到剪切板，则：

  1. 选择 Preferences: System Variables。
  2. 在 Set System Variables 对话框中，输入 Variable = OPJ (前面没有 "@"), 然后设置 Value = 2 或 = 3, 然后点击 OK。

那么此时使用 Edit: Copy Page 菜单命令将会默认使用以前的 OGG 文件格式，允许用户可以使用旧版本的软件打开 Word 文档中的图形。 (注意，图形的类型必须要是旧版本软件所支持的类型).
