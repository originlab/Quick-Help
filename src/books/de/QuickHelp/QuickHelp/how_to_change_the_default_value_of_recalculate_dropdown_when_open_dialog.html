FAQ-288 Wie kann der Standardwert des Auswahlmodus für Neuberechnung beim Öffnen eines Dialogs geändert werden?

recalculate-dropdown-default

Letztes Update: 01.06.2022

Analyse

Wenn Sie eine Analyse für einige Spalten oder ein Arbeitsblatt durchführen, ist der Modus Neu berechnen beim Öffnen eines Dialogs standardmäßig auf Manuell gesetzt. Das Symbol wird als grünes Schloss im Ergebnisblatt angezeigt. Sollte es eine Änderung in den Quelldaten oder Parametereinstellungen geben, müssen Sie manuell auf das Schlosssymbol klicken und die Option Neu berechnen auswählen, um die Ergebnisse neu zu berechnen.

Um den Neuberechnungsmodus global auf Auto zu setzen, geben Sie die Systemvariable @DAM=1 (Standardwert ist 2) im Skriptfenster oder Befehlsfenster ein, bevor eine Analyse durchgeführt wurde. Standardmäßig wird nun beim Öffnen eines Dialogs die Option Neu berechnen gezeigt und nicht Auto.

Hinweis: Die Systemvariable @DAM beeinflusst sowohl die Hilfsmittel, die auf X-Funktionen basieren, als auch die Hilfsmittel, die auf Operationen basieren. Das Hilfsmittel Daten aus Arbeitsblatt extrahieren wird von dieser Systemvariablen nicht gesteuert, und der Standardwert bleibt bei Kein.

 

Spaltenformel

Wenn Sie eine Spaltenberechnung mit Hilfe der Zelle F(x) durchführen, wird das Schloss für die Neuberechnung als ein grünes Schlosssymbol neben dem Spaltenkopf angezeigt. Die Systemvariable zum Steuern der Neuberechnung der Spaltenfunktion wird durch @AUFL bestimmt. Standardmäßig ist @AUFL auf 0 gesetzt. Das bedeutet, dass erst geprüft wird, ob es eine Formel mit dem Schloss der Neuberechnung in der Spalte gibt. Falls ja, wird der Neuberechnungsmodus beibehalten. Ansonsten wird Auto als Neuberechnungsmodus gesetzt.

Um den Standardmodus der Neuberechnung global auf Auto oder Manuell zu setzen, setzen Sie @AUFL = 1 bzw. @AUFL = 2 im Skriptfenster oder Befehlsfenster, bevor Sie die Formel eingeben.

Setzen Sie @AUFL = 3 auf Kein, so dass kein Neuberechnungsschloss hinzugefügt wird. Wenn sich die Eingabe ändert, wird die Ausgabe nicht automatisch aktualisiert. Der Vorteil ist, dass Sie die Werte in der Ausgabespalte frei bearbeiten können.

Sie können den Wert von @DAM und @AUFL in Origin permanent ändern. Weitere Einzelheiten finden Sie in dieser FAQ.
