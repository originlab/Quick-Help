FAQ-917 Origin 2018 funktioniert nicht mehr richtig nach dem Verschieben des Anwenderdateiordners vom Speicherort im Netzwerk

Origin-2018-Moving-User-Files-Folder-from-Network-Location

Letztes Update: 14.07.2018

Hinweis: Diese FAQ gilt nur für Origin 2018 und 2018SR1 (nicht 2018b oder frühere/spätere Versionen).

Wenn Sie Probleme mit Origin haben, nachdem Sie den Anwenderdateiordner aus einem Netzwerklaufwerk in einen lokalen Speicherort auf Ihrem PC verschoben haben, müssen Sie vielleicht einige Änderungen an bestimmten Ordner machen, die Origin 2018 verwendet. Sie beobachten zum Beispiel, dass Origin eine modifizierte Projektdatei meldet, die nur lesbar ist, wenn Sie versuchen, sie zu speichern.

Origin 2018 erstellt 4 Ordner zusätzlich zu Ihrem Anwenderdateiordner. Diese Ordner sind Geschwisterordner zu Ihrem Anwenderdateiordner und heißen Autosave, Unsaved, Backup und TempSave. Wenn Origin Probleme macht, kann das daran liegen, dass einer oder mehrere dieser Ordner sich noch auf einem Netzwerk gespeichert ist und in ein lokales Laufwerk verschoben werden muss, vorzugsweise als Geschwister des neuen Ordners Anwenderdateien.

Speicherort des Ordners ändern

Nachdem Sie den Speicherort des Anwenderdateiordners geändert haben, führen Sie die unten beschriebenen Schritte aus.

Der Ordner Autosave

  1. Wählen Sie in Origin im Menü Hilfe die Option Ordner öffnen: Anwenderdateiordner.
  2. Navigieren Sie im Fenster des Windows-Dateiexplorers eine Ebene nach oben in der Verzeichnishierarchie.
  3. Existiert bereits ein Ordner mit dem Namen Autosave als Geschwister des Anwenderdateiordners?
  4. Falls nicht, erstellen Sie einen leeren Ordner mit dem Namen Autosave in diesem Speicherort.
  5. Wählen Sie zurück in Origin im Menü Hilfsmittel den Befehl Optionen.
  6. Wählen Sie im Dialog Optionen die Registerkarte Systempfad.
  7. Wählen Sie den Eintrag Automatisch speichern und klicken Sie auf die Schaltfläche Ändern....
  8. Navigieren Sie im Dialog Ordner durchsuchen zum Ordner Autosave, den Sie in Schritt Step 4 oben erstellt haben, und klicken Sie auf OK.
  9. Prüfen Sie zurück im Dialog Optionen, dass die Listung für Automatisch speichern korrekt ist, und klicken Sie auf OK.
  10. Schließen Sie Origin.

Die Ordner Unsaved, Backup und TempSave

Teil I

  1. Navigieren Sie im Windows-Dateiexplorer zum Speicherort Ihres neu erstellten lokalen Anwenderdateiordners.
  2. Navigieren Sie eine Ebene nach oben in der Verzeichnishierarchie.
  3. Existiert bereits einer der Ordner Unsaved, Backup und TempSave als Geschwisterordner des Ordners Anwenderdateien?
  4. Falls nicht, erstellen Sie leere Ordner für die fehlenden und vergeben Sie die Namen Unsaved, Backup bzw. TempSave. Notieren Sie sich die vollständigen Pfade zu jedem der Ordner.

Teil II

  1. Öffnen Sie den Registrierungs-Editor
    Geben Sie für Windows 10 regedit in das Feld Suchen ein und drücken Sie Enter.
    In vorherigen Versionen von Windows klicken Sie auf die Startschaltfläche und geben im Suchfeld regedit ein. Drücken Sie Enter.
  2. Navigieren Sie zum folgenden Speicherort im Registry Editor: Computer\HKEY_CURRENT_USER\Software\OriginLab\Origin 9.5\International
  3. Im Bedienfeld auf der rechten Seite des Dialogs Registrierungs-Editor können Sie einen Eintrag für Autosave sehen. Sie werden entsprechende Einträge für Unsaved, Backup und TempSave erstellen (oder sie modifizieren, falls sie bereits vorhanden sind).
  4. Um diese Einträge zu erstellen, klicken Sie mit der rechten Maustaste in das rechte Bedienfeld und wählen Sie Neu: Zeichenfolge.
  5. Der neue Wert heißt New Value #1. Klicken Sie doppelt auf ihn, um seinen Namen in Unsaved, Backup oder TempSave zu ändern (Fall ist wichtig).
  6. Klicken Sie mit der rechten Maustaste auf den Eintrag und wählen Sie Ändern....
  7. Geben Sie im Dialog Neue Zeichenfolge den VOLLSTÄNDIGEN Pfad zum zugehörigen lokalen Ordner ein. Fügen Sie bitte ein \ (Backslash) am Ende des Pfads hinzu.
  8. Wiederholen Sie die Schritte 4-7 für die Ordner Unsaved, Backup und TempSave.
  9. Schließen Sie den Registrierungs-Editor.

Abschluss

Starten Sie Origin und prüfen Sie, ob das fehlerhafte Verhalten behoben ist. Sollte dies nicht der Fall sein, können Sie sich an den Technischen Support von OriginLab wenden, um weitere Unterstützung und Anleitung zu erhalten.
