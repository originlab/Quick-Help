FAQ-1127 Das Öffnen einer mit OneDrive synchronisierten Datei verursacht Fehler: Die Datei ist beschädigt

OneDrive-file-corrupted-project-backup

Letztes Update: 08.02.2022

Beim Öffnen von Origin-Dateien aus einem Ordner, der mit einem Microsoft-OneDrive-Konto synchronisiert ist, sehen einige Benutzer eine Fehlermeldung, die anzeigt, dass die Datei beschädigt ist, aber dass eine ältere Version der Datei noch verfügbar ist. Der Benutzer wird aufgefordert, im Menü Hilfe: Ordner öffnen: Projekt-Backup zu wählen, um die Backup-Datei aufzurufen.

Benutzer haben Folgendes berichtet:

  • Nachdem Sie bei der Fehlermeldung auf OK geklickt haben, wird die Datei normal geöffnet und intakt angezeigt.
  • Wenn der Benutzer jedoch versucht, die Datei zu speichern, speichert Origin standardmäßig im Ordner \Backup. Dies macht es notwendig, zum OneDrive-Ordner zu navigieren und die Datei dort zu speichern.

Diese Fehlermeldung wird nicht angezeigt, wenn das Benutzerkonto über die vollständigen Administrationsrechte verfügt. Wir sind noch dabei, die Einzelheiten zu untersuchen. Laut Berichten von Benutzern scheint der Wechsel von einem normalen Benutzerkonto zu einem Administratorkonto mit erweiterten Berechtigungen das Problem zu lösen.
