FAQ-256 Wie schreibe ich lineare Nebenbedingungen in verschiedene Anpassungsfälle?

Linear-Constraints-Cases

Letztes Update: 08.08.2018

In verschiedenen Fällen unterscheiden sich die Schreibweisen von linearen Nebenbedingungen.

Anpassen von Wiederholungen

Verwenden Sie parameter name +__n als Schreibweise, wobei n die (n-1)te Wiederholung bezeichnet. Beachten Sie, dass zwei Unterstriche verwendet werden.

Angenommen, y0 ist ein Parameter und es gibt eine Kopie. Die verfügbare Schreibweise wäre dann:

y0 y0__2
Hinweis: Die Schreibweise ist anders als in Origin 7.5.

Peaks analysieren

Die Nebenbedingung verwendet die Schreibweise parameter name+__n, wobei n den n-ten Peak bezeichnet. Beachten Sie, dass zwei Unterstriche verwendet werden.

Angenommen, es gibt zwei Peaks und zwei Variablen, a und b. Dann wäre die verfügbare Schreibweise:

a__1 b__1 a__2 b__2

Ein typisches Beispiel für eine Nebenbedingung ist b__1 = b__2.

Existieren geteilte Parameter verwenden Sie bitte den ersten Parameter in der gleichen Gruppe zum Schreiben von Nebenbedingungen. Wenn zum Beispiel die Parameter a__2, a__3 geteilt werden, wird die Nebenbedingung a__1 = a__2 verwendet und nicht a__1 = a__3.

Hinweis:
  1. Die Schreibweise unterscheidet sich von der in PFM in Origin 7.5 verwendeten.
  2. Der Unterschied zwischen Nebenbedingungen im Hilfsmittel Peaks analysieren und den Bedingungen von Wiederholungen liegt in der ersten Gruppe der Parameter. Wiederholungen verwenden im Gegensatz zum Hilfsmittel Peaks analysieren keine Unterstriche und numerische Postfixe für die erste Gruppe der Parameter.

Globale Anpassung

Verwenden Sie die Schreibweise parameter name+_n, wobei n den n-ten Datensatz bezeichnet.

Angenommen, a ist ein Parameter und es gibt zwei Datensätze. Die verfügbare Schreibweise wäre dann:

a a_2
Hinweis: Die Schreibweise mit Postfix "_n" ist nur bei der globalen Anpassung gültig. Die unabhängige Anpassung passt jeden Datensatz getrennt an, so dass "_n" bedeutungslos ist.
