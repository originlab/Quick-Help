Arbeitsblattdaten können in eine Matrix konvertiert werden. Es gibt zwei Hauptmethoden für die Konvertierung von Arbeitsblatt in Matrix in Origin: Direkte Konvertierung und XYZ-Gridding. Die direkte Konvertierung transformiert das Arbeitsblatt Worksheet in eine Matrix (Daten werden in der Matrix wie in dem ursprünglichen Arbeitsblatt angeordnet). Gridding-Methoden konvertieren XYZ-Daten mit einem ausgewählten Algorithmus in eine Matrix.

Eine Origin-Matrix ist eine rechteckige Anordnung von Z-Werten, deren Spalten linear nach X-Werten und deren Zeilen linear nach Y-Werten abgebildet werden. Wenn die XYZ-Daten in dem ursprünglichen Worksheet äquidistant sind, können wir die Daten direkt in eine Matrix konvertieren, da jede XY-Position einen repräsentativen Z-Wert hat. Wenn die Daten zufällig verteilt sind, sollten die 2D-Gridding-Methoden verwendet werden, um eine Reihe von zufälligen (X-, Y-, Z-) Punkten für die Schätzung der Werte von Z in einem rechteckigen Gitternetz für jedes (X, Y-) Paar zu verwenden.

Bei der direkten Konvertierung können die Werte der X- und Y-Abbildung in die Konvertierung eingebunden werden, müssen aber innerhalb einer anpassbaren Toleranz linear sein.

Bei Gridding-Methoden für Zufallsdaten kann das Problem mathematisch so beschrieben werden, dass eine Reihe gegebener, nicht äquidistant verteilter Punkte (xi, yi, zi) eine Funktion F(x, y) ergeben, die die Daten durch F(xi, yi)=zi interpoliert. In der X-Y-Ebene müssen die Datenpunkte eindeutig sein. Die erstellte Oberfläche ist kontinuierlich und hat kontinuierliche erste Ableitungen.

Beim Interpolieren tragen die Datenpunkte nahe der zu schätzenden Gitternetzpunkte mehr bei (mit höherer Gewichtung). Hinsichtlich des Bereichs, den eine Schätzung abdeckt, gibt es zwei Strategien, eine globale und eine lokale. Die globalen Algorithmen berechnen hierbei alle Datenpunkte, um die jeweiligen Gitter zu schätzen. Die lokalen Strategie verwendet nur die Informationen innerhalb eines bestimmten Bereichs. Es kann dabei zu einem Unterschied bei den jeweiligen Gridding-Ergebnissen kommen. Der globale Algorithmus ist rechenintensiver.

Hinweis 1: Kollineare Daten in einem Satz sind nicht erlaubt, da sie keine Oberfläche definieren. Diese Punkte müssen also entfernt werden. Mehrere z-Werte können bei einem bestimmten x,y-Punkt auch mehrere Optionen zum Bestimmen eines zusammengesetzten z-Wertes haben.

Origin verfügt über die X-Funktion XYZ2Mat für die Methode XYZ-Gridding. Ebenso verfügt Origin über die X-Funktion XYZ2MatLog, die logarithmische Werte für X und Y berechnet, bevor das XYZ-Gridding für die Daten durchgeführt wird (Log(X), Log(Y), Z).
