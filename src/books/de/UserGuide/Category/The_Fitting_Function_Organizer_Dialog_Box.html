Fitfunktionen verwalten

FitFunc-Organizer


Es gibt zwei Hauptbereiche im Dialog Fitfunktionen verwalten: den oberen Bedienbereich und den unteren Vorschaubereich. Sie können mit den Schaltflächen Vorschau zeigen/verbergen , den unteren Bereich anzeigen oder verbergen.

  • Der obere Bereich ist in drei Teile unterteilt: Der linke Teil führt die Funktionskategorien auf, in der Mitte befindet sich der Hauptbereich zum Bearbeiten der Funktionen und ganz rechts sind die Bedienschaltflächen zu finden.
  • Der untere Bereich weist drei Registerkarten auf: Die Registerkarten Gleichung und Beispielskurve zeigen Ihnen die Gleichungen und Beispielkurven (nur verfügbar für standardmäßige Funktionen). Die Registerkarte Hinweise bietet hilfreiche Hinweise für die Verwendung des aktiven Bedienelements.

Weitere Informationen zu standardmäßigen und benutzerdefinierten Anpassungsfunktionen finden Sie unter diesen Themen:

Bedienelemente der Funktionskategorie

Die linke Seite des oberen Bereichs führt Funktionen nach Kategorie auf. Erweitern Sie einen Zweig, um die verbundenen Funktionen anzuzeigen und einzelne Funktionen zum Modifizieren, Duplizieren oder Löschen auszuwählen. Benutzerdefinierte Kategorien und Funktionen werden in den Standardfunktionen unten angezeigt.

Parametereinstellungen

Klicken Sie auf die Schaltfläche Parametereinstellungen rechts vom Bearbeitungsfeld oder bearbeiten Sie das Feld direkt, um Parametereinstellungen festzulegen (z.B. Grenzen, Initialisierungswerte, Status Fest/Variieren, Einzelheiten hierzu unter Grenzen und Nebenbedingungen).

Der Dialog Parametereinstellungen kann Parameter auch mit Hilfe einer Initialisierungsformel initialisieren (Werte der Spaltenstatistik etc.), indem Sie erst das Kontrollkästchen in der Spalte Initialisierungsformel aktivieren und dann auf die dreieckige Schaltfläche in jeder Parameterzelle klicken, um die gewünschte Initialisierungsformel den jeweiligen Parameter auszuwählen.
Automatische Suche aktivieren

Verwenden Sie dieses Kontrollkästchen, um den Parameterinitialisierungscode, der im Bearbeitungsfeld Parameterinitialisierung festgelegt ist, zu aktivieren/deaktivieren. Beachten Sie, dass - sollte dieses Kontrollkästchen aktiviert sein - die Initialisierungswerte für die Parameter entsprechend dem Parameterinitialisierungscode gesetzt werden, auch wenn Sie die Initialisierungswerte in den Parametereinstellungen festgelegt haben sollten.
Initialisierung der festen Parameter zulassen

Aktivieren Sie dieses Kontrollkästchen, um den Parameterinitialisierungscode zu aktivieren, auch wenn die Parameter festgelegt sind.
LabTalk verwenden/Origin C verwenden

Wenn Sie eine neue Funktion erstellen, ist die Option Origin C verwenden standardmäßig aktiviert. Ist dieses Kontrollkästchen aktiviert, muss der Parameterinitialisierungscode mit Origin C definiert sein. Diese Option optimiert die Parameterinitialisierung.

Ist die Option LabTalk verwenden deaktiviert, wird der Parameterinitialisierungscode als LabTalk-Skript interpretiert.

Aktivieren von Bedingungen

Verwenden Sie dieses Kontrollkästchen, um die Nebenbedingungen, die im Bearbeitungsfeld Bedingungen festgelegt sind, zu aktivieren/deaktivieren. Nur verfügbar, wenn das Funktionsmodell auf Explizit gesetzt ist.
Skript vor dem Fit LabTalk-Skripts, die vor Beginn der Anpassung ausgeführt werden. Beachten Sie, dass das Skript in diesem Textfeld vor den Skripts im Dialog Parameterinitialisierung ausgeführt wird.
Skript nach dem Fit LabTalk-Skripts, die nach Ende der Anpassung ausgeführt werden. Wenn Sie beispielsweise die Anpassungsergebnisse auf eine bestimmte Weise angezeigt oder gespeichert haben möchten, schreiben Sie ein Skript, dass diese Aufgabe durchführt, und geben Sie es in diesem Textfeld ein.
Konstanten Definieren Sie die Konstanten hier. Diese Konstanten können im Anpassungsfunktionskörper und Parameterinitialisierungscode verwendet werden. Trennen Sie diese Variablen durch Semikolon (;), wenn es mehrere Konstanten gibt. Zum Beispiel:
c1 = 20;
c2 = -3,5;

Hinweis: Parameternamen, abgeleitete Parameternamen und Systemvariablennamen sollten nicht als Namen von Konstanten verwendet werden.
