Automatisierungsserver

Automation-Server

Das sogenannte Component Object Model (COM) ist ein von Microsoft® entwickelter Schnittstellenstandard für Softwarekomponenten. Es wird für die Interprozesskommunikation und dynamische Objekterstellung mittels Programmiersprachen verwendet, die COM unterstützen. Der Begriff COM wird häufig in der Softwareentwicklungsindustrie als ein Überbegriff, der OLE, OLE Automation, ActiveX® , COM+ und DCOM umfasst, verwendet.

Origin kann jetzt als Automatisierungsserver für Client-Anwendungen eingesetzt werden, wie Microsoft® Excel®, Microsoft® Visual Basic® und National Instruments™ LabVIEW™, die allesamt die COM-Programmierung unterstützen. Daten können von der Client-Anwendung an Origin weitergegeben werden. Sie können grafisch dargestellt, analysiert und das Ergebnis anschließend exportiert und wieder an Ihre Anwendung zurückgesendet werden, ohne dass sie mit Origin in Berührung gekommen sind.

Sobald der Client die Verbindung schließt, kann der Anwender später Origin starten und das gespeicherte Projekt zur weiteren Bearbeitung erneut öffnen.

Eine Netzwerkinstallation (Server/Client) von Origin ist ideal, um aus dieser Funktion Nutzen zu ziehen. Origin wird als Server auf einem Computer und als Client auf mehreren Computern mittels eines FlexLM-Lizenzmanagers installiert. Danach kann jeder Techniker, der einen Client-Rechner verwendet, von dem Unternehmen mit einer einfachen Oberfläche werden, wie einem Excel-Arbeitsblatt oder einer firmeneigenen Anwendung von Visual Basic. Diese einfachen Oberflächen können die Installation des Origin-Clients auf ihrem lokalen Rechner dazu verwenden, den Origin-Server aufzurufen. Dazu muss der Techniker nicht lernen, wie Origin verwendet wird, da sich alle erforderlichen Bedienelemente auf der benutzerdefinierten Oberfläche befinden.

Themen, die in diesem Abschnitt behandelt werden:

  • Microsoft® Excel als Client-Anwendung verwenden
  • Microsoft® Visual Basic als Client-Anwendung verwenden
  • Mathworks MATLAB® als Client-Anwendung verwenden
  • National Instruments™ LabVIEW™ als Client-Anwendung verwenden
  • MSXML-Parser zum Konvertieren eines Origin-Arbeitsblatts in ein XML-Dokument verwenden
