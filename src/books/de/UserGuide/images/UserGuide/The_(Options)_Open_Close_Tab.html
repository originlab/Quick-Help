Optionen Fenster schließen

Arbeitsmappe, Excel-Arbeitsmappe, Grafik, Layout, Matrix und Notizen

Entscheiden Sie, ein Unterfenster ohne eine eingehende, zu bestätigende Warnmeldung zu Verbergen oder zu Löschen (um ein Fenster beispielsweise zu verbergen, anstatt es zu löschen, wählen Sie in der Auswahlliste Nur Verbergen aus). Abfrage ist das Standardverhalten.

Eingabeaufforderung beim Löschen eines Blatts

Aktivieren Sie dieses Kontrollkästchen, um beim Löschen eines Arbeitsblatts oder einer Matrix eine Eingabeaufforderung anzuzeigen.

Beim Schließen des Skriptfensters nach Speicherung fragen

Aktivieren Sie dieses Kontrollkästchen, um eine Abfrage beim Schließen des Skriptfensters anzuzeigen (, wenn die Inhalte verändert wurden). Diese Abfrage erscheint auch beim Schließen von Origin, wenn die Inhalte des Skriptfensters verändert wurden.

Zusätzliche Bedienelemente

Neues Projekt starten

Diese Auswahlliste ermöglicht es Ihnen, die Fenster oder Projekte auszuwählen, die angezeigt werden sollen, wenn Sie ein neues Projekt öffnen. Sie können aus Worksheet, Excel-Arbeitsmappe, Leeres Diagramm, Matrix, ORIGIN.OPJ oder Keine auswählen. Die Datei ORIGIN.OPJ beinhaltet ein Worksheet und ein Diagrammfenster. Der Layerinhalt des Diagrammfensters ist so konfiguriert, dass der Datensatz der Spalte B angezeigt wird.

Öffnen im Unterordner

Wenn dies ausgewählt ist, öffnen Sie die Auswahl Neues Projekt starten in einem Unterordner unter dem Hauptordner Projekt Explorer (PE). Andernfalls beginnen Sie das neue Projekt nur mit einer Auswahl im Hauptordner PE.

Vor dem Speichern Sicherungskopie erstellen

Aktivieren Sie dieses Kontrollkästchen, um automatisch eine Sicherungskopie der aktuell gespeicherten Datei vor dem erneuten Speichern zu erstellen. Origin sichert die gespeicherte Datei mit dem aktuellen Projektnamen im Backup-Ordner und speichert dann die modifizierte Datei mit einem benutzerspezifischen Dateinamen. Sie behalten bei Aktivierung von "Backup vor dem Speichern" eine zweite Wiederherstellungsoption.

Hinweise:

  • Das Backup ist für das Wiederherstellen von Daten am nützlichsten, wenn Sie eine Projektdatei haben, die sich nicht öffnen lässt. Sie können die Sicherungsdateien über eine der folgenden Methoden finden:
    • Windows Startmenü > Alle Programme > OriginLab > Origin 2018b > Backup
    • Origin-Menü Hilfe: Ordner öffnen: Projektsicherungsordner
  • Es wird nur die zuletzt gespeicherte Backup-Version aufbewahrt, wenn die gleiche Projektdatei mehrmals am gleichen Tag gespeichert wurde. Das bedeutet, wenn Sie ein Projekt mehrmals speichern, wird die frühere Backup-Version ersetzt.
  • Wenn es mehrere Projekte mit dem gleichen Name an unterschiedlichen Orten gibt, wird nur eine Sicherungsdatei gespeichert.
  • Die Backup-Dateien werden nicht automatisch gesäubert. Sie müssen sie ggf. manuell säubern.
  • Setzen Sie die Systemvariable @PBO auf 1, um nur ein Backup.opj bzw. Backup.opju für ALLE Dateien zu speichern. Das Format der Backupdatei hängt davon ab, in welchem Format, OPJ oder OPJU, das Projekt geöffnet wurde.
Projekt automatisch alle xx Minute(n) speichern

Aktivieren Sie das Kontrollkästchen Automatisches Speichern des Projekts alle x Minuten, um die automatische Speicherung Ihrer Origin-Projekte zu veranlassen. Sie können festlegen, wie oft das Projekt automatisch gespeichert werden soll. Standard ist alle 12 Minuten (das Projekt muss geändert worden sein). Ihr Projekt muss einmal gespeichert worden sein, damit die automatische Speicherung funktioniert. Wenn Sie ein neues Objekt-Projekt öffnen, das noch nicht auf der Festplatte gespeichert wurde (und daher auch keinen gültigen Pfad und Dateinamen besitzt), kann der Prozess zum automatischen Speichern nicht gestartet werden.

Hinweise:

  • Die automatische Speicherung ist nützlich zum Wiederherstellen nach Programmabstürzen und Stromausfällen.
  • Projektinhalte werden regelmäßig in einer temporären Datei im Unterordner \Autosave gespeichert.
  • Um auf den Ordner \Autosave zuzugreifen, klicken Sie auf das Menü Start auf dem Windows-Desktop, wählen Sie Programme: Originlab: Origin-Version: Autosave (ältere Windows-Betriebssysteme); oder durchsuchen Sie die Programme in Ihrem Start-Menü (bzw. die Windows "Apps") und klicken Sie auf das OriginLab-Ordnersymbol Autosave.
  • Bei großen Projektdateien (>20MB) kann es vorkommen, dass die automatische Speicherung Ihre Arbeit verlangsamt. In solchen Fällen werden Sie gefragt, ob die Funktion der automatischen Speicherung für aktuelle oder alle Projekte deaktiviert werden oder mit der automatischen Speicherung fortgefahren werden soll. Siehe die LabTalk-Systemvariable @ASC.
  • Wenn Sie das Projekt oder Origin selbst normal beenden, wird die temporäre Datei gelöscht.
Ungespeicherte Projekte automatisch speichern

Wenn Sie die Änderungen in der aktuellen Projektdatei verwerfen (indem Sie "Nein" im Dialog Projekt speichern oder die Option wählen, das Projekt beim Schließen des Programms nicht zu speichern), speichert Origin diese Änderungen automatisch in eine temporäre Datei unter dem Pfad C:\Users\<LoginAccount>\Dokumente\OriginLab\Unsaved\.

Anders als bei den Optionen Backup vor dem Speichern und Automatisches Speichern des Projekts alle "x Minuten" sichert die Option Ungespeicherte Projekte automatisch speichern jedes modifizierte Projekt, entweder ein bereits vorher gespeichertes Projekt oder ein neues Projekt, das noch nicht gespeichert wurde.

Die automatisch gespeicherte Datei hat den gleichen Namen wie das Quellprojekt (das neue Projekt, das zuvor noch nicht gespeichert wurde, wird als "UNTITLED.opju" gespeichert). Wenn Sie das Projekt modifizieren und die Änderungen dann mehrmals verwerfen, wird nur die letzte Änderung bewahrt.

Sie haben Zugriff auf die ungespeicherten Dateien über Origin-Navigator -> Projekte. Der Abschnitt Ungespeicherte Dateien listet alle temporär gespeicherten Dateien. Sie können sie auch im Unterordner \Unsaved finden. Sie bemerken vielleicht, dass der Dateipfad, der im Origin-Navigator als Pfad der Quelldatei gezeigt wird, nicht der reale Pfad der ungespeicherten Kopie ist. Wenn Sie eine ungespeicherte Kopie vom Origin-Navigator aus öffnen und diese manuell speichern, wird die Quelldatei ersetzt und verschwindet aus dem Unterordner \Unsaved.

Die automatisch gespeicherten Dateien werden per Standard 7 Tage aufbewahrt. Wenn Sie wählen, ein Projekt zu speichern, das über eine automatisch gespeicherte Version verfügt, räumt Origin auf, indem es die automatisch gespeicherte Datei löscht. Sie können den Dialog Hilfsmittel: Systemvariablen setzen öffnen und USKT auf einen neuen Wert setzen, um den Zeitraum, für den automatisch gespeicherte Dateien bewahrt werden, zu ändern.

Eine zweite LabTalk-Systemvariable steuert die maximale Speicherung von Dateien für das automatische Speichern von ungespeicherten Dateien (Standard ist 20 MB). Siehe die Systemvariable @USM.

Hinweise:

  • Die Funktion "Ungespeicherte Projekte automatisch speichern" ermöglicht, dass fälschlicherweise verworfene Änderungen wiederhergestellt werden können, was besonders nützlich für Projekte ist, die niemals vorher gespeichert wurden.
  • Wenn Sie sich beim Speichern eines großes Projekts (> 20 MB) dazu entscheiden, die Funktion "Automatisches Speichern des Projekts alle "x Minuten"" für das aktuelle oder alle Projekte auszuschalten, befolgt die Funktion "Ungespeicherte Projekte automatisch speichern" die gleiche Regel. Sie können die Systemvariable @ASC verwenden, um die Schwellenwertgröße zu ändern, die das automatische Speichern auslöst.
Beim Öffnen minimierter Fenster

Diese Auswahlliste ermöglicht Ihnen die Entscheidung, wie Fenster, die minimiert in Projekten einer Vorversion von Origin 6 gespeichert wurden, in Origin 6 geöffnet werden sollen. Das Öffnen von minimierten Fenstern als verborgene Fenster verbessert die Ladezeit Ihres Projekts. Verborgene Fenster können vom Projekt Explorer aus aufgerufen werden.

Beim Schließen des Optionen-Dialogs nach Speicherung fragen

Aktivieren Sie dieses Kontrollkästchen, um nach dem Schließen des Dialogs Optionen eine Warnmeldung zu öffnen, die Sie fragt, ob Sie die Änderungen als Origin Startup-Optionen speichern möchten.

Beim Schließen speichern

Wenn Beim Schließen des Optionen-Dialogs nach Speicherung fragen deaktiviert ist, ist das Feld Beim Schließen speichern verfügbar.

Aktivieren Sie Beim Schließen speichern, um die Änderungen im Origin-Dialog Optionen automatisch als Origins Startup-Verhalten zu speichern.

Deaktivieren Sie dieses Feld, um die Standardoptionen beim Beenden von Origin wiederherzustellen.
