ASCII-Importoptionen benutzerdefiniert anpassen

ImpASC-Options

Wenn Ihre ASCII-Dateistruktur einfach ist und keine komplexen Funktionen wie Headerzeilen mit zu extrahierenden Parametern besitzt, können Ihre Dateien mit dem Import für Einzelne ASCII oder Mehrere ASCII importiert werden. Sie haben noch immer die Option, die Importeinstellungen mit Hilfe der Bedienelemente im Dialog impASC benutzerdefiniert anzupassen. Sobald die benötigten Anpassungen in den ASCII-Importoptionen durchgeführt sind, können Sie diese auf zwei Arten abspeichern:

  • Als Design auf der Festplatte: Durch das Speichern als eine Designdatei können Sie dieses Design später aufrufen, um ähnliche Dateien mit identischer Struktur zu importieren, und dabei die gespeicherten Einstellungen verwenden.
  • Mit der Vorlage: Sobald Sie den Importvorgang starten, werden benutzerdefinierte Einstellungen standardmäßig in <Blatt> gespeichert und werden Teil des Arbeitsblatts. Diese Option eignet sich für die Stapelverarbeitung von mehreren Dateien mit Hilfe von Analysevorlagen.

Um den Dialog impASC zu öffnen (siehe Bild unten), wählen Sie Daten: Aus Datei importieren: Mehrere ASCII oder Daten: Aus Datei importieren: Einzelnes ASCII. Aktivieren Sie Optionendialog zeigen im Dialog ASCII, wählen Sie dann die zu importierende(n) Datei(en) aus und klicken Sie auf OK.

Hinweis: Wenn Sie eine Arbeitsmappenvorlage geöffnet haben und diese Einstellungen in der Arbeitsmappenvorlage speichern möchten, müssen Sie die Vorlage erneut speichern (Datei: Template speichern unter).

Importoptionen

Sparklines hinzufügen

Legt fest, ob die Sparklines für die importierten Daten hinzugefügt werden.

  • Nein: Fügt keine Sparklines für die importierten Daten hinzu.
  • Ja: Fügt immer Sparklines für die importierten Daten hinzu.
  • Ja (falls...): Wenn die importierten Daten weniger als 50 Spalten darstellen, werden Sparklines hinzugefügt.
Importmodus der ersten Datei

Diese Einstellung bestimmt, wo die importierten Daten hinzugefügt werden:

  • Bestehende Daten ersetzen
    Bei einer einzelnen Datei werden die bestehenden Daten im aktiven Arbeitsblatt ersetzt. Bei mehreren Dateien wird die 1. Datei im aktiven Arbeitsblatt ersetzt und weitere Dateien öffnen neue Arbeitsmappen (mit Vorlage, wenn in der Auswahlliste Vorlagenname festgelegt), um Daten zu importieren.
  • Neue Mappe öffnen
    Daten werden für jede Datei in eine neue Arbeitsmappe importiert.
  • Neues Blatt öffnen
    Daten werden in die vorhandenen leeren Blätter nach dem aktiven Blatt importiert. Sollte es keine geben, wird ein neues Blatt erstellt. Ein Maximum von 1024 Blättern kann zu einer Mappe hinzugefügt werden, bevor eine neue Mappe erstellt wird.
  • Neue Spalten öffnen
    Es werden neue Spalten für jede Datei von der 1. leeren Spalte an geöffnet. Die Header von jeder Datei werden an die entsprechenden Spalten Langname, Einheiten etc. angehängt.
  • Neue Zeilen
    Es werden neue Zeilen für jede Datei von der 1. leeren Zeile an geöffnet. Nur die Header von der letzten Datei werden an die entsprechenden Spalten Langname, Einheiten etc. angehängt.
Importmodus mehrerer Dateien (außer der ersten)

Importmodus für die Dateien Nr. 2 bis n

  • Neue Mappe öffnen
    Daten werden für jede Datei in eine neue Arbeitsmappe importiert.
  • Neues Blatt öffnen
    Daten werden in die vorhandenen leeren Blätter nach dem aktiven Blatt importiert. Sollte es keine geben, wird ein neues Blatt erstellt. Ein Maximum von 1024 Blättern kann zu einer Mappe hinzugefügt werden, bevor eine neue Mappe erstellt wird.
  • Neue Spalten öffnen
    Es werden neue Spalten für jede Datei von der 1. leeren Spalte an geöffnet. Die Header von jeder Datei werden an die entsprechenden Spalten Langname, Einheiten etc. angehängt.
  • Neue Zeilen
    Es werden neue Zeilen für jede Datei von der 1. leeren Zeile an geöffnet. Nur die Header von der letzten Datei werden an die entsprechenden Spalten Langname, Einheiten etc. angehängt.
Vorlagenname

Zum Importieren von Daten kann eine Arbeitsmappenvorlage (*.otw) oder eine Analysevorlage (*.ogw) verwendet werden. Sie können einen Vorlagenname mit seiner Dateierweiterung in dieses Feld eingeben oder auf die Schaltfläche Durchsuchen klicken, um eine Vorlage auszuwählen. Vorlagenname ist nur verfügbar, wenn für den Importmodus die Option Bestehende Daten ersetzen, Neue Mappe öffnen oder Neues Blatt öffnen ausgewählt ist.

  • Standard ist die Verwendung von Origins Prototyparbeitsmappenvorlage <Standard>.
  • Der <Klon> verwendet die aktive Arbeitsmappe als die Vorlage zum Importieren von Dateien in Duplikate des/r aktuellen Arbeitsblatts/-mappe.
  • Wenn Sie - ab Version 2018b - <Klon> wählen, müssen Analysen, die Operationen der Neuberechnung mit importierten Daten enthalten, nicht auf Auto gesetzt werden. Die Neuberechnung wird automatisch gestartet, unabhängig vom Modus Neu berechnen im Analysehilfsmittel. Diese Änderung gilt nicht für betitelte Vorlagen, die für den Datenimport verwendet werden. Beim Verwenden einer betitelten Vorlage müssen Sie sicherstellen, dass Ihre Operation auf die Automatisch Aktualisieren gesetzt ist.

Headerzeilen

Kopfzeilen automatisch bestimmen

Aktivieren Sie dieses Kontrollkästchen, wenn Sie wollen, dass Origin die Anzahl der Hauptheader- und Subheaderzeilen automatisch festlegt.

Headerzeilen sind normalerweise nicht in derselben Weise strukturiert wie die Daten. Subheaderzeilen werden normalerweise auf die gleiche Weise strukturiert wie die Daten (d.h. die gleichen Trennzeichen oder festgelegte Spaltenstruktur).

Deaktivieren Sie dieses Kontrollkästchen, wenn Sie diese beiden Werte selber eingeben möchten.
Zeilennummer beginnt von unten

Aktivieren Sie dieses Kontrollkästchen, wenn die Zeilennummern für Kurznamen, Langnamen, Einheiten, Kommentare von/bis, Systemparameter von/bis, Benutzerparameter von/bis die Nummern sein sollen, die vom unteren Teil des Subheaderbereichs aufwärts gezählt werden.

Deaktivieren Sie dieses Kontrollkästchen, um von oben nach unten zu zählen.
Hauptheaderzeilen

Einige Dateien -- besonders die durch Laborgeräte erzeugt wurden -- haben oben in der Datei einige Headerinformationen, die gewöhnlich das Instrument, den Mitarbeiter, das Datum, die Stichprobennummer usw. kennzeichnen. Diese sind die Hauptheaderzeilen. Sie können festlegen, wie die Anzahl der Hauptheaderzeilen mit diesem Bedienelement bestimmt wird. (Hinweis: Dieses Bedienelement ist nur verfügbar, wenn das Kontrollkästchen Kopfzeilen automatisch bestimmen deaktiviert ist.)

  • Festgelegte Anzahl
    Aktivieren Sie diese Option und geben Sie die Anzahl der Hauptheaderzeilen im folgenden Textfeld Festgelegte Anzahl ein.
  • Nach anführendem Zeichen
    Aktivieren Sie diese Option, um die Anfangszeichen einzugeben, mit denen die Headerzeilen erkannt werden, die mit den Zeichen im folgenden Textfeld Anführendes Zeichen beginnen. Dieses Bedienelement ist ab Origin 2018b verfügbar.
Anzahl der Subheaderzeilen

Dieses Bedienelement ist nur dann verfügbar, wenn das Kontrollkästchen Automatische Festlegung der Subheaderzeilen deaktiviert ist.

Sie können diese Liste verwenden, um die Anzahl der Subheaderzeilen Ihrer Datei festzulegen.

Dateistruktur

Datenstruktur

Es gibt zwei Typen von ASCII-Datenstrukturen:

  • Begrenzt - Einzelnes Zeichen, Begrenzt - Mehrere Zeichen: Dateien, in denen die Datenspalten durch ein oder mehrere Zeichen (normalerweise ein Komma, ein Tab oder ein Leerzeichen) getrennt sind.
  • Feste Spaltenbreite: Dateien mit einer festgelegten Spaltenbreite

Origin kann die richtige Datenstruktur beim Import einzelner ASCII-Dateien normalerweise "raten", kann aber bei einigen Dateien Schwierigkeiten haben. Wenn Dateien nicht korrekt importiert werden, verwenden Sie diese Bedienelemente.
Trennzeichen

Nur verfügbar, wenn Sie Getrennt - Einzelnes Zeichen wählen. Legen Sie eine der folgenden Optionen fest: Tab/Leerzeichen, Tab, Komma(,), Semikolon, Leerzeichen, Unbekannt oder Anderes.

  • Wenn Sie Unbekannt auswählen, durchsucht Origin Ihre Datei nach einem wiederkehrenden Trennzeichen.
  • Wenn Sie Anderes wählen, wird das Feld Andere Trennzeichen hinzugefügt (siehe nächsten Abschnitt).
Weitere Trennzeichen

Wenn die beiden folgenden Optionen wahr sind ...

  • Datenstruktur = Begrenzt - Einzelnes Zeichen
  • Trennzeichen = Anderes

... verwenden Sie das Kontrollkästchen Weitere Trennzeichen, um das Trennzeichen einzugeben (z. B. ':' oder '/'). Beachten Sie, dass Sie mehrere Trennzeichen eingeben können. Im diesem Fall beachten Sie das Feld Aufeinanderfolgende Trennzeichen als eins betrachten (nächster Abschnitt).
Aufeinanderfolgende Trennzeichen als eins betrachten

Wenn alle folgenden Optionen wahr sind ...

  • Datenstruktur = Begrenzt - Einzelnes Zeichen
  • Trennzeichen = Anderes
  • Sie mehrere Trennzeichen in Weitere Trennzeichen eingegeben haben

... aktivieren Sie dieses Kontrollkästchen, um diese weiteren Zeichen als eine Einheit zu behandeln. Wenn Sie das Kontrollkästchen NICHT aktivieren, werden mehrere Zeichen als eine Liste akzeptabler Trennzeichen interpretiert und jegliche übereinstimmenden Zeichen oder eine übereinstimmende Sequenz von Zeichen werden als ein Trennzeichen behandelt.
Mehrere Zeichen

Nur verfügbar, wenn Datenstruktur = Getrennt - Mehrere Zeichen. Geben Sie eine geordnete Sequenz von Trennzeichen ein (keine Leerzeichen und kein Komma, es sei denn, diese werden als Trennzeichen betrachtet). Eine übereinstimmende Abfolge wird als ein Trennzeichen betrachtet.

Um ein {Tab}-Zeichen in dieses Textfeld einzugeben, öffnen Sie Notepad und drücken Sie die Tab-Taste. Markieren und kopieren Sie den gesamten eingegebenen Raum und fügen Sie das Ergebnis in das Textfeld ein.
Spaltenbreite

Für eine Datei mit festgelegter Spaltenbreite setzen Sie die Datenstruktur auf Feste Spaltenbreite und bestimmen Sie die Breite in dem Feld Spaltenbreite. Beachten Sie, dass Sie unterschiedliche Spaltenbreiten angeben können, indem Sie jede durch ein Komma trennen. Wenn die Anzahl der Datenspalten die Anzahl der Werte der Spaltenbreite übersteigt, verwendet Origin den letzten Wert, um die Breite für alle verbleibenden Spalten zu setzen. Wird nur ein einzelner Wert für die Breite eingegeben, setzt Origin alle Spalten auf diese Breite.
Numerische Trennzeichen

Numerische Trennzeichen ändern sich mit der Sprache. Der Standard <Auto> in dieser Liste veranlasst Origin, die Einstellung Trennzeichen für ASCII-Import auf der Registerkarte Einstellungen: Optionen > Zahlenformat zu verwenden. Wenn Sie nichts anderes festgelegt haben, entspricht das Trennzeichen für ASCII-Import den Windows-Einstellungen - den Dezimal- und Gruppierungstrennzeichen Ihrer regionalen Windows-Einstellungen. Weitere Optionen geben Ihnen die Möglichkeit, ein alternatives Schema anzugeben, das verwendet werden soll, wenn einfache ASCII-Dateien in eine aktive Arbeitsmappe importiert werden.
Benutzerdefiniertes Datumsformat

Wählen Sie ein Datumsformat aus der Auswahlliste oder geben Sie eine benutzerdefinierte Zeichenkette für das Datumsformat ein. Weitere Einzelheiten finden Sie auf der Seite Benutzerdefiniertes Datumsformat.
Benutzerdefiniertes Zeitformat

Wählen Sie ein Zeitformat aus der Auswahlliste oder geben Sie eine benutzerdefinierte Zeichenkette für das Zeitformat ein. Weitere Einzelheiten finden Sie auf der Seite Benutzerdefiniertes Datumsformat.
Daten transponieren

Dieses Kontrollkästchen wird verwendet, um festzulegen, ob die Daten während des Imports transponiert werden sollen.

Spalten

Anzahl der Spalten

Anzahl n der Spalten zum Importieren bestimmen Wenn die Datei, die Sie importieren, mehr als n Spalten enthält, werden nur die ersten n Spalten importiert. Wenn die Datei weniger als n Spalten enthält, wird die gesamte Datei importiert, ohne dass zusätzliche (leere) Spalten eingefügt werden.

Wenn in der Liste 0 ausgewählt wird, importiert Origin alle Spalten in die Datei.
Automatische Festlegung des Spaltentyps

Ist das Kontrollkästchen aktiviert, bestimmt Origin automatisch, ob jede Spalte Daten mit Texten oder Zahlen enthält, und legt das Spaltenformat im Arbeitsblatt entsprechend fest.

Wenn das Kontrollkästchen deaktiviert ist, behält Origin die Formateinstellungen der Zielarbeitsblattspalten bei. In diesem Fall sollte das Eingabeformat mit dem Format der Arbeitsmappenspalte übereinstimmen, damit keine Daten verloren gehen.
Mindestanzahl von einheitlichen Zeilen, um die Datenstruktur festzulegen

Geben Sie eine Mindestanzahl von übereinstimmenden Zeilen an, um die Datenstruktur festzulegen. Wenn diese Anzahl von Zeilen mit der gleichen Struktur gefunden werden, wird diese Struktur als Datenstruktur für die gesamte Datei verwendet.
Maximale Anzahl von einheitlichen Zeilen, um die Datenstruktur festzulegen Geben Sie eine Zahl ein, die die Anzahl an Headerzeilen übersteigt, aber kleiner ist als die Anzahl der Zeilen in der Datei.
Spaltenzuordnung

Übernehmen Sie die Spaltenzuordnungen zum Zeichnen in Origin für importierte Arbeitsmappendaten. Wählen Sie aus der Auswahlliste aus oder geben Sie das Zuordnungsmuster direkt ein. Diese Option unterstützt die gleiche Syntax der Spaltenzuordnung (einschließlich wiederholte Zuordnungen) wie der Importassistent. Weitere Einzelheiten finden Sie auf dieser Hilfeseite.

Dateiname

Automatisches Umbenennen mit Hilfe des Dateinamens

Ist diese Option aktiviert, verwendet Origin die Standardeinstellungen zum Umbenennen der Namen des Arbeitsblatts und der Arbeitsmappe. Diese sind:

  • Verwendung des gesamten Dateinamens (ausschließlich Dateierweiterung) als Arbeitsblattname,
  • Verwendung des gesamten Dateinamens (einschließlich Dateierweiterung) als Arbeitsmappenlangname, falls die aktuelle Arbeitsmappe leer ist oder der Importmodus Bestehende Daten ersetzen gewählt wurde.
Blatt mit (partiellem) Dateiname umbenennen

Benennt das Zielarbeitsblatt nach dem Dateinamen. Falls bereits ein Datenblatt mit dem angegebenen Namen besteht, wird eine Zahl an den Namen angehängt. Dieses Kontrollkästchen ist nur verfügbar, wenn das Kontrollkästchen Automatisches Umbenennen mit Hilfe des Dateinamens aktiviert wurde.
Verkürzen des Dateinamens von/auf

Verwenden Sie diese beiden Textfelder, um den Teil des Dateinamens festzulegen, der zum Umbenennen des Arbeitsblatts verwendet werden soll.
Mappe mit (partiellem) Dateiname umbenennen

Benennt die Zielarbeitsmappe nach dem Dateinamen. Dieses Kontrollkästchen ist nur verfügbar, wenn das Kontrollkästchen Automatisches Umbenennen mit Hilfe des Dateinamens aktiviert wurde.
Verkürzen des Dateinamens von/auf

Verwenden Sie diese beiden Textfelder, um den Teil des Dateinamens festzulegen, der zum Umbenennen der Arbeitsmappe verwendet werden soll.
Langname nur für Mappe umbenennen

Aktivieren Sie dieses Kontrollkästchen, um nur den Langnamen für die Arbeitsmappe umzubenennen.
Dateipfad beim Umbenennen der Mappe einschließen

Ist dieses Kontrollkästchen aktiviert, wird der Dateipfad der importierten Datei an den Arbeitsmappennamen angehängt.

Der Dateiname wird an die Kommentare der Arbeitsmappe angehängt.
Dateiname an Arbeitsblattkommentar anhängen

Der Dateiname wird an die Kommentare des Arbeitsblatts angehängt.

Der Dateiname wird an die Kommentarheaderzeile des Arbeitsblatts angehängt. Falls die Kommentarzeile verborgen ist, wird sie automatisch angezeigt. Der Wert wird für jede importierte Spalte wiederholt und ist nützlich, wenn der Importmodus für mehrere Dateien Neue Spalten öffnen lautet.
Anwenderparameterzeile des Dateinamens hinzufügen

Wenn dieses Kontrollkästchen aktiviert ist, wird der Dateiname in der Anwenderparameterzeile Quelldatei gezeigt.
Den Dateipfad einbinden, wenn der Dateiname angehängt wird

Ist dieses Kontrollkästchen aktiviert, wird der Dateipfad der importierten Datei an den Dateinamen angehängt.

Partieller Import

Partieller Import

Diese Auswahlliste ist nur verfügbar, wenn Importmodus für mehrere Dateien (außer der 1. Datei) auf Neue Spalten öffnen gesetzt ist.

  • Alle Dateien: Die Einstellungen des partiellen Imports werden auf alle ausgewählten Dateien angewendet.
  • Ab der 2. Datei: Die Einstellungen des partiellen Imports werden auf alle ausgewählten Dateien außer die 1. Datei angewendet.
Partielle Spalten

Dieser Zweig steuert, wie Dateien teilweise nach Spalten importiert werden.

  • Von und Bis
Legen Sie den zu importierenden Anfangs- und Endspaltenindex fest.
  • Lesen und Überspringen
Wenn Sie eine Zahl l für Lesen und eine Zahl k für Überspringen festlegen, werden für alle l+k Spalten l Spalten importiert und k Spalten ausgelassen.
  • Benutzerdefiniert
Siehe die Notation für Benutzerdefiniert zum Festlegen der Spaltenauswahl unter Teilweiser Import und Erneuter Import.
Partielle Zeilen

Dieser Zweig steuert, wie Dateien teilweise nach Zeilen importiert werden.

  • Von und Bis
Legen Sie den zu importierenden Anfangs- und Endspaltenindex fest.
  • Lesen und Überspringen
Wenn Sie eine Zahl l für Lesen und ein Zahl k für Überspringen festlegen, werden für alle l+k Zeilen l Zeilen importiert und k Zeilen ausgelassen.
  • Benutzerdefiniert
Siehe die Notation für Benutzerdefiniert zum Festlegen der Zeilenauswahl unter Teilweiser Import und Erneuter Import.
