Appendix 5 - Wichtige Änderungen für Benutzer von älteren Versionen

notable-changes

Wenn Sie ein Upgrade von einer älteren Version von Origin auf die neueste Version durchführen, kann es hilfreich sein, sich die folgenden Änderungen anzusehen, die an den letzten Versionen von Origin gemacht wurden, um Ihren Arbeitsablauf entsprechend anzupassen.

Namen der Arbeitsblattspalte

Version bei Änderung: Origin 2017(9.4)

Das Benennen der Spaltenkurznamen in Arbeitsblättern wurde geändert, so dass der Standard nun A, B, C ... ist, ähnlich wie bei Excel. Diese Änderungen wurde gemacht, um das Schreiben von Spaltenformeln zu vereinfachen. Wenn Spalten verschoben werden, passen sich die Namen an, um die alphabetische Reihenfolge von links nach rechts beizubehalten. Existierende Spaltenformeln werden automatisch aktualisiert.

Diese neue Notation wird als vereinfachte Zellennotation (SCN) bezeichnet. Es ist eine Eigenschaft des Arbeitsblatts und kann bei Bedarf ausgeschaltet werden. Wir empfehlen jedoch, dass Sie statt der Kurznamen die Spaltenlangnamen für deskriptive Namen verwenden.

Wenn Sie alte Projekte oder Arbeitsmappe öffnen, deren Spaltenkurznamen auf spezielle Namen festgelegt waren, dann wird die vereinfachte Zellennotation in der neuesten Version automatisch ausgeschaltet. Wenn Sie die Eigenschaft einschalten, verlieren Sie die zugewiesenen Kurznamen.

Weitere Informationen finden Sie in dieser FAQ über SCN.

Dateiimport

Version bei Änderung: Origin 2019b(9.65)

Der Mechanismus der Datenkonnektoren zum Verbinden von lokalen und Webdateien wurde in Origin 2019b eingeführt. Ein neues Menü Daten wurde auch eingeführt. Die alten Importroutinen sind nach wie vor im Menü Daten verfügbar, aber wir empfehlen die Verwendung der Datenkonnektoren.

Die über Datenkonnektor importierten Daten sind standardmäßig von der Bearbeitung gesperrt. Datensätze über einer vordefinierten Größe werden beim Speichern des Projekts ebenfalls standardmäßig gelöscht, um kleinere Projektdateigrößen beizubehalten.

Weitere Informationen finden Sie auf dieser Seite zu Datenkonnektoren.

Projektdatei

Version bei Änderung: Origin 2018(9.5)

Origin 2018b führte ein auf Unicode basierendes OPJU-Dateiformat zum Speichern von Projektdateien ein. Ältere Dateien mt der Erweiterung OPJ können in den letzten Versionen geöffnet werden. Der Dialog Datei > Speichern unter ... erlaubt auch das Speichern von Projekten im ältere OPJ-Format. Die Systemvariable @OPJ kann verwendet werden, um das Standardformat zum Speichern von Projekten zu steuern.

64 vs. 32 Bit

Version bei Änderung: Origin 2020(9.7)

Origin 2019b war die letzte Version, die 32 Bit unterstützt hat. Seit Origin 2020 wird die Origin-Software nur in 64-Bit-Versionen angeboten.

Benutzerdefinierte Menüs

Version bei Änderung: Origin 2020(9.7)

Die Option Kurzmenüs wurde von der Bedienoberfläche entfernt und ist nur über die Einstellung @SCNFM = 1 verfügbar. Bei Vorhandensein von benutzerdefinierten Menüs funktionieren Kurzmenüs jedoch nicht ordnungsgemäß.

Benutzerdefinierte Menüs, die auf CNF-Dateien basieren, funktionieren noch korrekt, wurden aber durch Benutzerdefinierte Menüs verwalten (basierend auf OMC-Dateien) ersetzt. Wenn Benutzerdefinierte Menüs verwalten ausgeführt wurde, werden die zuvor erstellten Menüs, die auf CNF-Dateien basieren, deaktiviert, da Sie keine CNF- und OMC-Menüs haben können. Um die CNF-basierten Menüs wiederherzustellen, bearbeiten Sie origin.ini (im Anwenderdateiordner) und fügen Folgerndes hinzu:

[Config] MenuLevelByOMC=0

Origin-C-Dateien automatisch laden

Version bei Änderung: Origin 2015(9.3)

Seit Origin 2015 können Sie keine Origin-C-Anwenderdateien zum Systemordner im Code Builder hinzufügen. Sie fügen sie jetzt zum Ordner Autoload (Automatisch laden) hinzu. Dateien im Autoload-Ordner werden automatisch während des Origin-Starts kompiliert. Ebenfalls während des Origin-Starts: Wenn Anwenderdateien im Systemordner entdeckt werden, werden sie automatisch in den Autoload-Ordner verschoben und der Anwender wird benachrichtigt. Weitere Informationen zu diesem Thema finden Sie unter: Dateien zum Anwenderordner [AutoLoad] hinzufügen.
