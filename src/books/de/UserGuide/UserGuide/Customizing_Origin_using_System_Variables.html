Origin mit Hilfe von Systemvariablen benutzerdefiniert anpassen

Customization-with-SysVar

LabTalk besitzt einen Typ Systemvariable, der als "@"-Systemvariable bezeichnet wird. Diese (im Allgemeinen) Lese-/Schreibvariablen sind ein Bestandteil Ihrer Origin-Software. Ihr Zweck besteht darin, Ihnen die Steuerung bestimmter Origin-Verhaltensweisen zu ermöglichen, indem ein vordefinierter Wert einer Variable zugewiesen wird. Diese Zuweisung kann automatisch beim Start stattfinden oder kann jederzeit manuell während der Origin-Sitzung vorgenommen werden.

Eine "@"-Systemvariable hat Einfluss auf Einstellungen auf einer viel niedrigeren Ebene als zum Beispiel die Einstellungen im Dialog Einstellungen: Optionen.

Zum Beispiel:

  • @A: Steuert Standardwinkeleinheiten (0=Radiant, 1=Grad, 2=Gon).
  • @CCT: Steuert die Zellenbreite der Standardtextspalte.
  • @DAM: Steuert den Standardstatus der Auswahlliste Neuberechnen in den Analysedialogen (0=Kein, 1=Auto, 2=Manuell).
  • @EEE: Steuert, ob leere Blätter während des Drag&Drop-Imports von Excel-Dateien ausgeschlossen werden.
  • @FBM: Steuert, ob Datenfilter auf maskierte Daten angewendet werden sollten (0=Filter ignoriert maskierte Daten, 1=Filter hat Zugriff auf maskierte Daten).

Es gibt buchstäblich Hunderte von diesen "@"-Variablen. Sie können den Wert einer "@"-Variable jederzeit beim Öffnen des Skriptfensters (Fenster: Skriptfenster) überprüfen, indem Sie den Namen der Variable gefolgt von einem "="-Zeichen eingeben und Enter drücken, wie in diesem Beispiel:

@FBM= //type and press Enter @FBM=1 //Origin returns the variable's value

Den Wert einer Systemvariable ändern

Wie oben erwähnt sind fast alle "@"-Variablen zum Lesen/Schreiben. Sie können das Skriptfenster verwenden, um den Wert einer Variable zu ändern, wie beispielsweise im Folgenden:

@DAM= // type and press Enter @DAM=2 // Origin return's the variable's default value @DAM=1 // type and press Enter to change the default Recalculate mode in analysis dialogs to "Auto".

Wenn Sie jedoch den Wert einer Variable auf diese Weise ändern, wird er nur für die Sitzung geändert. Um Ihre Änderung sitzungsübergreifend anzuwenden (d.h., jedes Mal, wenn Origin geöffnet wird), sollten Sie die Variable und ihren zugewiesenen Wert zum Dialog Systemvariablen festlegen hinzufügen.
