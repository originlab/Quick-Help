Nutzungskontext Benutzerdefiniertes Anpassen des Nutzungskontexts der X-Funktion
  • LabTalk
    Legt fest, ob die X-Funktion von LabTalk aus aufgerufen werden kann. Wenn diese Option nicht ausgewählt ist, kann die X-Funktion nur vom Menü aus aufgerufen werden.
  • Arbeitsblattauswahl
    Momentan nicht verwendet.
  • Ereignisse Grafikobjekt
    Dies ist nur gültig, wenn die X-Funktion mit einem grafischen Objekt verbunden ist. Es wird festgelegt, welche grafischen Objektereignisse die X-Funktion auslösen. Verfügbare Ereignisse sind Initialisieren, Verschieben, Größe verändern, Löschen, Auswählen, Auswahl aufheben, auf Schaltflächen klicken, Drehen, Verzerren, Bearbeiten, Neue Auswahl, Teil löschen und Tastatur. Der grafische Objekttyp kann ein Rechteck oder eine Linie sein.
  • Design
    Legt fest, ob und wo (Dialog, Menü, Mappe oder Blatt) das Design für die X-Funktion unterstützt werden soll. Optionen sind: Unterstützung zum Speichern von Designs im Dialog, Designs im Menü als Popups anzeigen, Unterstützung zum Speichern in Mappe und Unterstützung zum Speichern In Blatt.
  • Menüs
    Legt die verfügbaren Menüs der X-Funktion fest. Optionen sind: Kein, Hauptmenüs, Kontextmenüs, Beide, Kontextmenü 1, Kontextmenü 2, Kontextmenü 3 und Kontextmenü 4. Der Zugriff auf die X-Funktion über das Menü kann benutzerdefiniert angepasst werden. Optionen sind: Als Hilfsmittel hinzufügen, Klassenoperation, Dialog N automatisch erhalten, Dialog vom Menü für Optionsindex aus öffnen und Menü prüfen. Verfügbare Optionen für Dialog N automatisch erhalten schließen Einfache GetNBox, GetNGraphBox mit Vorschau, GetNImageBox, Wizard und Benutzerdefiniert (durch Festlegen der benutzerdefinierten Dialogfunktion) ein.
  • Fenster
    Legt den Fenstertyp fest, der mit der X-Funktion verbunden ist. Optionen: Diagramm, Arbeitsblatt, Matrix, Bericht, Layout und Kein.
LabTalk-Baum bei Ausführung erzeugen Legt fest, ob der LabTalk-Baum bei Ausführung automatisch erzeugt wird. Der LabTalk-Baum ist eine Variable mit dem gleichen Namen wie die X-Funktion und wird im Ergebnisfenster ausgegeben.
Kontrollkästchen Ergebnisprotokollausgabe Legt fest, ob ein Kontrollkästchen mit der Beschriftung Ausgabe des Ergebnisfensters im Dialog X-Funktion hinzugefügt wird, so dass die Anwender sich entscheiden können, ob sie Meldungen an das Ergebnisfenster weitergeben. Optionen:
  • Kein
    Kontrollkästchen Ausgabe des Ergebnisfensters wird nicht hinzugefügt.
  • Aktiviert
    Kontrollkästchen Ausgabe des Ergebnisfensters wird hinzugefügt und standardmäßig aktiviert.
  • Deaktiviert
    Kontrollkästchen Ausgabe des Ergebnisfensters wird hinzugefügt und standardmäßig deaktiviert.
  • Kein und Aktivieren
    Kontrollkästchen Ausgabe des Ergebnisfensters wird nicht hinzugefügt, aber Nachrichten werden immer im Ergebnisfenster ausgegeben.
Ausgabe ins Ergebnisprotokoll bei Ausführung Legt fest, welcher Inhalt im Ergebnisfenster ausgegeben wird; Optionen können sein: Kein, Alles, Berichtsbaum, Erste Eingabe, Erste Ausgabe, Alle Ausgaben und Alle außer Bäume.
GetN Dialog Benutzerdefinierte Schaltflächen Legt die benutzerdefinierten Schaltflächen im Dialog fest.
Befehlszeilenverwendung Fügt die Notizen bei Verwendung der Befehlszeile der X-Funktion hinzu, die in der Dokumentation der X-Funktion angezeigt werden.

Datenfilter als Teil der Operation zulassen

Wenn dieses Kontrollkästchen aktiviert ist, gibt es drei Arbeitsblattfilteroptionen (Arbeitsblattfilter: Blockieren/ Auf Quellarbeitsblatt anwenden/ Löschen), die im Kontextmenü des Schlosssymbols für die Neuberechnung gezeigt werden. Sie können festlegen, ob die Ergebnisse durch weitere Filteränderungen beeinflusst werden. Standardmäßig ist dieses Element in den X-Funktionen Spalten kopieren und Pivot-Tabelle aktiviert.
Dokumentname Legt den Pfad und den Namen der entsprechenden zu öffnenden Hilfedatei fest. Beachten Sie, dass die Zeichenkettenverzeichnisse (string registers) wie %Y unterstützt werden.
Neuordnen von Variablen nach Code zulassen

Im Allgemeinen sind X-Funktions-Variablen in einer bestimmten Reihenfolge geordnet. Wenn dieses Kontrollkästchen aktiviert ist, können Sie das Origin-C-Skript verwenden, um die Variablen neu zu ordnen.
Leere Eingabe als gültige Operation zulassen

Im Allgemeinen hat eine X-Funktion einen tatsächlichen Eingabedatenbereich. Sie können aber dieses Kontrollkästchen aktivieren, um die Dummy-Eingabe für die X-Funktion zuzulassen und ein Ergebnis mit der Neuberechnung auszugeben.
Automatische Tabellenzellennotation in neuer Mappe Sie können steuern, ob die vereinfachte Tabellenzellennotation SCN (Speedsheet Cell Notation) für die neue Arbeitsmappe, erstellt mit Hilfe der Ausgabe einer X-Funktion, aktiviert werden soll. Wenn dieses Kontrollkästchen aktiviert ist und der Bit-Wert (4) von @RCN ausgewählt ist, wird für jede neue Arbeitsmappe, die mittels Ausgabe einer X-Funktion erzeugt wird, die SCN aktiviert sein (Automatisch Ein), unabhängig von dem Status der Zellennotation der aktiven Mappe.

Dieses Kontrollkästchen wird nicht angewendet auf wsplit, wsplit_book, merge_book, wmove_sheet.
Automatische Aktualisierung in Beschriftungszeile unterstützen

Legen Sie fest, ob die Ergebnisse mit Neuberechnung automatisch aktualisiert werden, wenn die Werte in der/n zugehörigen Spaltenbeschriftungszeile/n geändert werden.

Bei einigen X-Funktionen wie wstackcol ist dieses Kontrollkästchen standardmäßig aktiviert. Wenn Sie die Werte in den Spaltenbeschriftungszeilen, die Teil der Stapeloperation waren, ändern (setzen Sie die Zeile Kommentar zum Beispiel auf Gruppenidentifizierer), werden die gestapelten Ergebnisse automatisch aktualisiert.
