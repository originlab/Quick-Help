Hilfsmittel starten

Mathematica-LaunchingTool

Damit Origin mit eine einzelnen Instanz von Mathematica direkt kommuniziert, müssen Sie Mathematica konfigurieren, um Origins Kernel OSequencer.exe auszuführen. Ansonsten kann Origin nur mit einem ausgeführtem Mathematica-Kernel kommunizieren, der gestartet wird, wenn das Hilfsmittel Mathematica Link aufgerufen wird.
Wenn Sie keine Kommunikation zwischen Origin und einer gestarteten Mathematica-Instanz benötigen, ignorieren Sie bitte den ersten Teil der Konfiguration innerhalb von Mathematica.
Mathematica konfigurieren, damit Origins Kernel OSequencer.exe beim Start ausgeführt wird
  1. Starten Sie Mathematica und wählen Sie Evaluate: Kernel Configuration Options im Mathematica-Menü.
  2. Klicken Sie in dem aufgerufenen Dialog auf die Schaltfläche Hinzufügen, um den Dialog Kernel Properties zu öffnen.
  3. Geben Sie dem Kernel einen Namen. Setzen Sie Arguments to MLOpen auf -LinkName "<Origin-Installationsverzeichnispfad>\OSequencer -mathlink -noicon". Beachten Sie, dass der vollständige Pfad der OSequencer.exe erforderlich ist.

    Der Kernel mit dem Namen "OSequencer" wird jetzt der Liste hinzugefügt.

 

Starten Sie das Hilfsmittel Mathematica Link in Origin.
  1. Wenn Sie im Origin-Hauptmenü Konnektivität: Mathematica Link auswählen, fragt Origin beim ersten Mal nach Mathematicas Kernel.
  2. Suchen Sie nach Mathematicas Installationsverzeichnis und wählen Sie dann MathKernel.exe.
  3. Das Hilfsmittel Origin Link für Mathematica wird dann geöffnet.

Sobald Sie dies getan haben, können Sie zum Ordner C:\Users\<Anwendername>\AppData\Local\Temp gehen und die Datei OMAthLinkConfig.INI öffnen.

[MathKernelExecutable] Location=C:\Program Files\Wolfram Research\Mathematica\<MathVersion>\MathKernel.exe

Prüfen Sie, ob der Pfad zu dem Mathematica-Installationsverzeichnis führt, zwischen dem und Origin Sie eine Kommunikation herstellen möchten. Dies ist besonders wichtig, wenn Sie mehr als eine Version von Mathematica installiert haben. Sollte es nicht der richtige Pfad sein, können Sie:

  • Den Pfad direkt ändern und OMAthLinkConfig.INI neu speichern. Starten Sie dann Origin erneut.

Oder

  • Löschen Sie OMAthLinkConfig.INI. Starten Sie Origin erneut. Origin wieder nach dem Kernel fragen. Sie können dann den richtigen Kernel auswählen.
