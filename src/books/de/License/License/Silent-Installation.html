Installationseinstellungen mit Config.ini benutzerdefiniert anpassen

Customize-Install-Settings

Was ist Config.ini und was kann es?

Die Datei Config.ini ist eine Konfigurationsdatei, mit der die Installationseinstellungen vordefiniert bzw. benutzerdefiniert angepasst werden. Sie wird mit der Origin-Installationsdatei ausgeliefert. Hinweis: Falls Sie den Exe-Installer (Format exe) von unserer Webseite herunterladen, enthält dieser keine Config.ini-Datei. Bitte wählen Sie stattdessen das zip-Format.

Was beinhaltet die Datei Config.ini?

Die Datei Config.ini enthält standardmäßig die folgenden Konfigurationen. Sie können sie in einem Notepad-Editor zum Anzeigen und Modifizieren öffnen. Es sind ausführliche Kommentare enthalten, die beim Verstehen der Einstellungen unterstützen.

Installationseinstellungen

Sie können alle Installationsdaten im Abschnitt [Setup] vordefinieren:

- Seriennummer, Benutzername und Unternehmensinformationen, Installationspfad, 32- oder/und 64-Bit-Version, ob Python-Konsole und SPSS-Import aktiviert werden sollen
- ob Origin still installiert werden soll

Hinweis: Wenn Sie QuietMode = 1 setzen, um Origin im stillen Modus zu installieren, müssen Sie die folgenden Elemente vorkonfigurieren.

AcceptOriginLabLicenceAgreement=1 SN= UserName= Company=

Ob die 32-Bit- oder/und 64-Bit-Anwendung installiert werden soll, wird von Betriebssystem und 32-Bit-/64-Bit--Einstellungen bestimmt. Das heißt:

  • Falls Sie Benutzer eines 32-Bit-Betriebssystems sind, wird nur die 32-Bit-Version von Origin installiert, unabhängig von Ihren Einstellungen für 32 Bit und 64 Bit.
  • Falls Sie Benutzer eines 64-Bit-Betriebssystems sind, wird die zu installierende Anwendung von 32 Bit und 64 Bit bestimmt. Wenn Sie beides überspringen oder auskommentieren, wird nur die 64-Bit-Version von Origin installiert.

Weitere Elemente verwenden die folgenden Standardwerte, falls sie nicht festgelegt sind:

InstallPath=C:\Program Files\OriginLab\Origin2019 EmbeddedPython=1 SPSSImport=1 AllUsers=1 ProgramFolder=C:\ProgramData\Microsoft\Windows\Start Menu\Programs\OriginLab Origin 2019


Origin-Ordnerpfad

  • Anwenderdateiordner
- Anwenderdateiordner im Abschnitt [Locations] vordefinieren
- ob der Dialog Anwenderdateiordner mit dem voreingestellten Pfad gezeigt wird, wenn Origin zum ersten Mal gestartet wird
  • Projektordner AutoSave, Backup und Unsaved
- übergeordneten Ordner für die Projektordner AutoSave, Backup und Unsaved im Abschnitt [Locations] voreinstellen
  • Apps
- Apps-Ordner im Abschnitt [Apps] voreinstellen

Hinweis: Falls Config.ini über vordefinierte Ordnerpfade verfügt (zum Beispiel die Ordner Anwenderdateien oder AutoSave), werden diese verwendet anstatt die alten Einstellungen aus der vorherigen Version zu verwenden.

Lizenzverwaltung

Bei einer Concurrent-Netzwerklizenz definieren Sie den Hostname bzw. die IP-Adresse im Abschnitt [License] vor. Falls Origin zuvor installiert gewesen ist, können Sie auch entscheiden, ob die aktuellen Serverinformationen aktualisiert werden sollen oder nicht.

Stille Installation

Im Fall der Datei Config.ini haben wir seit Origin 2019 eine neue stille Installationsmethode.

Hinweis: Falls Sie Origin 2018b oder eine ältere Version besitzen und Origin mit stiller Installation verteilen möchten, lesen Sie bitte diese Seite.

Um das Setup-Programm von InstallShield zu verwenden, um Origin still auf Computern Ihrer Organisation zu installieren, führen Sie die folgenden Schritte durch.

  1. Kopieren Sie die Origin-Installationsdateien in einen Ordner auf Computer "A". Hinweis:
    • Computer "A" muss sich in einem Netzwerk befinden, zu dem alle potenziellen Origin-Anwender eine Verbindung herstellen können.
    • Stellen Sie sicher, dass im Ordner 14 Installationsdateien vorhanden sind, einschließlich Config.ini.
  2. Modifizieren Sie die Datei Config.ini mit allen gewünschten Konfigurationsvoreinstellungen.
  3. Falls erforderlich, nehmen Sie folgende Voreinstellungen für die Lizenzverwaltung vor:
    • Bei einer Concurrent-Netzwerklizenz definieren Sie den Hostname bzw. die IP-Adresse im Abschnitt [License] der Datei Config.ini vor. Ansonsten muss der Anwender sie im Dialog Lizenzinformation eingeben, wenn Origin zum ersten Mal gestartet wird.
    • Verwenden Sie im Fall einer rechnergebundenen Gruppenlizenz (Node-Locked) die Datei pk.txt, um den Produktschlüssel voreinzustellen. Origin aktiviert diese automatisch beim ersten Start von Origin. Andernfalls muss der Anwender über den Dialog Lizenzaktivierung eine Lizenz von der OriginLab-Webseite aktivieren.
  4. Stellen Sie sicher, dass die modifizierten Dateien Config.ini und pk.txt (ggf.) sich im Origin-Installationsordnder auf \\A befinden, wo sich die setup.exe abgelegt ist.
  5. Geben Sie diesen Ordner mit VOLLEN LESE-/SCHREIBrechten frei und notieren Sie sich seinen UNC-Freigabenamen. (z. B. \\A\Origin\)
  6. Schreiben Sie allen Anwendern eine E-Mail, in der Sie sie auffordern, die Datei setup.exe auszuführen. Anwender müssen an ihren Computern mit einem Konto angemeldet sein, das über Verwaltungsrechte verfügt. Außerdem müssen Sie sich im Netzwerk befinden und auf \\A zugreifen können.

Hinweis:

  • Diese Methode der stillen Installation installiert Origin nur auf den Zielcomputern.
  • Im Fall von Concurrent-Netzwerklizenzen wird ein Dialog zur Registrierung geöffnet, nachdem Origin 10-mal ausgeführt wurde. Der Anwender muss Origin selbst registrieren.
