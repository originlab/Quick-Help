Known Issues in 8.5.1 SR1

Worksheet

JIRA Summary Details
ORG-3006 Dynamic merging does not work on the Units row. Identical column labels and data can be dynamically merged horizontally or vertically by setting Dynamic Merge on the Format tab of the Worksheet Properties dialog. However, the Dynamic Merge control for Units is not working.
The workaround is:
  1. In Origin, open the script window by clicking Windows on the main menu, and then click Script Window.
  2. In the Script Window, type @wef=1 and press Enter. This will add Edit Formatting... context menu in worksheet.
  3. Right-click in the gray area in the workbook. In the context menu, click Edit Formatting...
  4. In the Worksheet Theme Editor, expand the Root folder, the Name Styles folder, the ogUnit folder, and the Style folder. In the Merge row, click None and select Horizontal from the dropdown menu.
  5. Click Apply and then close the dialog box.
  6. If you want workbook units to ALWAYS be merged: From Origin's main menu, click Worksheet, then click Clear Worksheet... Click OK to confirm. Then, from the File menu, click Save Template As... to save this as your default Origin.otw template
  7. Now go back to the Script Window. Type @wef=0 and press Enter to remove the Edit Formatting... context menu from worsheet.

Licensing

JIRA Summary Details
ORG-3039 License files with expiration dates were not working in German and Japanese Origin. This included renewable nodelocked Origin, student Origin, concurrent Origin, license keys, evaluation licenses, and temporary licenses.

For German Origin, if the expiration date in the license file was mar, may, oct, or dec, it was not accepted. For Japanese Origin, no license files with expiration dates were accepted. This is fixed in 8.5.1 SR2.
