Graphing

JIRA Summary Details
ORG-3382 Merge Graph Windows tool doesn't work unless run Origin as admin and use the tool once.
  • This only happens on some Win7/Vista PCs.
  • It also exists for Origin 8.0, 8.1, 8.5, 8.5.1.
  • User needs to run Origin as administrator and use the tool once.
  • After that no need to run Origin as admin everytime.

Import&Export in 64-bit

JIRA Summary Details
ORG-3881, SUP-948 pClamp import in 64-bit Origin not correct for some pclamp data. This is related to the Molecular Devices DLL. Digital Bitpattern may be incorrect with the 2.2.0.1 Molecular Devices DLL. We recommend using 32-bit Origin when importing pClamp data.
ORG-4193 Export Worksheet as Multipage PDF fails in 64-bit Origin. GIF image file is exported instead. No problem in 32-bit Origin.
ORG-3865 Some Import/Export Formats are not supported in 64-bit Origin due to non-availability of third-party plug-ins/DLLs: We recommend using 32-bit Origin for the following imports and exports:
  • Thermo (SPC, CGM) Import
  • *.dbf, *.WK *, *.DIF Import
  • Database Import: Query Builder tool not available. SQL Editor tool works fine.
  • Export Graph as Image: *.ai, *.cgm, *.dxf

Statistics

JIRA Summary Details
ORG-4370 Some results of Discriminant Analysis are wrong for quadratic discriminant analysis or unequal probabilities
  • Some observations will be misclassified in cross-validation when prior probabilities proportional to group size
  • Upper triangular elements in the group distance matrix are not correct for quadratic discriminant analysis(QDA)
  • Discriminant analysis does not support masked data in the group column
ORG-4389 Last three columns of Eigenvalues of the Covariance Matrix are empty When choosing Covariance Matrix as Analyze type, in Eigenvalues of the Covariance Matrix tab, the last 3 columns (Chisq, Degrees of Freedom, Significance Level) are empty.
ORG-4377 Long name of some Principal Components is wrong in PCA. After doing Principal Component Analysis, if user clicks the lock to change parameters in the analysis dialog, the long names of some Principal Components are wrong in Score Data sheet.
ORG-4376 Labels of means comparison plots in ANOVA report sheet are wrong or gibberish. The means comparison plots show as Data(1), Data(2), etc., instead of the method name.

File Access

JIRA Summary Details
ORG-449 Double-clicking .opj failed to open project in Origin on some PCs with Windows 7/Vista. Workaround in Origin 8.6 (not available in earlier versions):
  • Run Origin and select Window: Script Window.
  • Run this code: 
    doc -ddde 1
(Note, ddde stands for disable DDE)
  • As prompted, close Origin and then right-click program icon to run as administrator. Close and restart Origin as usual.
  • You should now be able to open an Origin project file by double-clicking.

Licensing

JIRA Summary Details
ORG-4353 Origin 32bit and 64bit don't share "borrowed license" If user borrows 32 bit Origin, he/she will not be able to use the 64 bit Origin. Vice versa.
On Windows Vista/7, must run as admin to return a borrowed license which hasn't expired yet. When borrowing license, there is a checkbox to return it when reconnected to network. If user doesn't check it, and then reconnects to the network, borrowed Origin will not automatically return. User must run as administrator to manually return it.

Group Folder Sharing

JIRA Summary Details
ORG-4362 Group folder sharing will not work for 8.6 sr1 patching. (Will only affect users with Group folder sharing.) For multi-user Origin installations, one Origin can be set as group leader and others can be set as group member. Group leader can get patch and publish it for others, but group members will not be able to get it. When 8.6 SR1 patch is released, the workaround is to have group leader instruct everyone to manually copy the patch to Origin exe folder\Updates\ folder and run the patch, or to have every member run Check for Updates.

Programming

JIRA Summary Details
We no longer include Flexgrid control access in Origin from 8.6 For customer who has created custom tools using Flexgrid, those tools will not work in 86 unless customer purchases and installs Flexgrid/distributes Flexgrid.
ORG-5477 Script using StringArray Append method produces Run Time Error This worked in versions previous to 8.6. 
stringarray sa;
sa.append("one|two|three|four","|");
loop(ii,1,sa.GetSize())
{
   sa.GetAt(ii)$=;
}
sa.append("oh, just five more things!");

It still works if you specify the delimiter: 

stringarray sa;
sa.append("one|two|three|four","|");
loop(ii,1,sa.GetSize())
{
    sa.GetAt(ii)$=; 
}
sa.append("one more thing","|");
sa.append("three more things"," ");
