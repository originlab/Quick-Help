マスターINIファイル


Originには、起動時にOriginの機能の設定を制御する多くのINIファイルがあります。例えば、Originのデフォルトの初期化ファイルであるORIGIN.INIは、Originのセッションが閉じるまで有効な数多くのOriginのオプションを事前に設定します。これらのオプションには、メニューの設定や表示上の設定が含まれます。標準的なサーバ・クライアントとしてインストールした場合、または、1台のコンピュータにOriginをインストールし、複数ユーザが使用する場合、それぞれのクライアントまたはユーザは、「ユーザファイル」フォルダに別々のINIファイルを保存します。つまり、それぞれのクライアントまたはユーザは、他のクライアントまたはユーザの設定を上書きすることなく、独自の設定でOriginをカスタマイズすることができます。

複数のINIファイルが存在していると、そのファイルの所有者がわからないため、ネットワーク管理者が複数ユーザ用のOriginを保守したり、更新する必要があります。 マスターINIファイルの特徴は、サーバ・クライアントでのインストールまたはマルチユーザワークステーションのインストールにおいて、複数のシステムファイルをコピーし、保存することにより生じる問題を軽減するように設計されています。

マスターINIファイルは、以下のように管理されます。

  • マスターINIファイルの名前は、「M」で始まる必要があります(例えば、ORIGIN.INIファイルはMORIGIN.INIファイル、NLSF.INIファイルは、MNLSF.INIファイルなど)。
  • マスターINIファイルは、Originのルートフォルダにある必要があります(Originのプログラムファイルのあるフォルダです)。

Originは、起動時にマスターINIファイルの存在を検出した場合、ローカルINIファイルとマスターINIファイルを統合します。もしマスターINIファイルにある設定が、ローカルINIファイルにない場合、ローカルINIファイルに設定されます。もしローカルINIファイルにある設定が、マスターINIファイルにある場合、ローカルINIファイルのその設定は上書きされます。

ファイルの統合が終了すると、マスターINIファイルの更新日時のタイムスタンプが、ローカルINIファイルに書かれます。次回、クライアントまたはユーザが、Originを起動すると、Originは、マスターINIファイルの更新日時と、ローカルINIファイルに書かれたタイムスタンプを比較します。2つの日付が同じであれは、INIファイルの統合は起こりません。そのため、マスターINIファイルの設定を無効にしたローカルINIファイルの変更が可能です。マスターINIファイルが変更されると、クライアントの起動時にローカルINIファイルは統合されます。

クライアントまたはユーザが、OPXツールを利用できるようにインストールする

システム管理者は、Originのルートフォルダを「ユーザファイル」フォルダにして、Originを実行する必要があります。そうすると、システム管理者がOPXファイルをインストールする時、Originのルートフォルダ直下にOPKツールがアンパックされます。

OPXファイルのインストールによって新しいツールバーが追加される場合、ルートフォルダにあるORIGIN.INIファイルに、ツールバーのエントリーが書かれます(今回の場合、ルートフォルダは、「ユーザファイル」フォルダと同じです)。

例えば、

[UserDefinedButtons] 
 
User Defined=Oubtn.ini 
 
Script Toolbar=OSCRIPTTOOLBAR.INI

すべてのクライアントまたはユーザがツールバーを利用できるようにするために、システム管理者はこのエントリーをMORIGIN.INIファイルに追加する必要があります。
