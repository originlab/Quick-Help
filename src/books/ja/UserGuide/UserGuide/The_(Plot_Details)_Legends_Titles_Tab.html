自動凡例

%(1), %(2)の解釈モード

凡例ボックス内の %(1) を %(1, @LM)、%(1, @L, 1) などに手動で変更する代わりに、凡例の %(1) を変換するためにどのワークシート列ヘッダ情報を使用するかを指定します。

@LM: コメントを使用 (第1行)

デフォルトの設定です。 コメント行の最初の行を使用します（複数ある場合）。 それ以外の場合、ロングネームを使用します。コメントもロングネームも内容がない場合、対応するショートネームを使用します。

@R: データ範囲

プロットのワークブックデータ範囲です。シンタックスは、[Workbook]Worksheet!Col(Index)[start index, end index]です。

@U: ロングネーム&単位

列ラベル行のロングネームと単位を組み合わせて使います。英語版と日本語版ではLong Name(Unit)、ドイツ語版ではLong Name[Unit]のフォーマットで表示されます。

ロングネームがない場合は、ショートネームが使用されます。

@LD: ユーザパラメータ 1

対応するデータ列の第一ユーザパラメータ行を使用します。

@LA: ロングネーム

対応するデータ列のロングネームを使用します。

@LS: ショートネーム

対応するデータ列のショートネームを使用します。

@LP: パラメータ 1

対応するデータ列の第一パラメータ行を使用します。

カスタム

凡例カスタムフォーマット 及び 追加フォーマット (for @L, @V, @WT) 編集ボックスで指定されたカスタムストリングを使います。使用可能な引数については@Optionの完全リストを参照してください。@(Letter) arguments のような組み合わせがサポートされています。

Notes: 列ヘッダ行ではリッチテキストフォーマットを有効にでき、これにより上付き/下付きの表示や特殊記号を設定できます。詳細はこのクイックヘルプを参照してください。

凡例オブジェクトを右クリックして、フォーマットのコピー：全てを選択することにより、あるグラフの%(1), %(2)の解釈モードを別のグラフに適用できます。（凡例のカスタムフォーマットを含む）目的の凡例上で右クリックし、フォーマットの貼り付け先：全てを選択します。

凡例カスタムフォーマット

この設定は、%(1), %(2)の解釈モードドロップダウンリストでカスタムが選択されているときのみ利用できます。 テキストを入力するか、頻繁に使用するフォーマットはドロップダウンリストからも選択できます。

コンビネーションボックスの隣にあるボタンをクリックすると@(Letter) argumentsのリストを確認できます。詳細は@オプションの完全リストを参照してください。

追加フォーマット

変換モード %(1) %(2)にカスタムが選択され、 凡例のカスタムフォーマットに@L (X値)、@V (Y値)、@WT (セル値) が選択されている場合、行インデックスまたは行と列のインデックスを指定するには、追加フォーマットを有効にする必要があります。例えば、

  • 凡例のカスタムフォーマットで@Lと入力し、追加フォーマット (for @L, @V, etc.)で1と入力すると、各プロットの1つめのX値を凡例テキストとして表示します。
  • 凡例のカスタムフォーマットで@Vと入力し、追加フォーマット (for @L, @V, etc.)で5と入力すると、各プロットの5つめのY値を凡例テキストとして表示します。
  • 凡例のカスタムフォーマットで@WTと入力し、追加フォーマット (for @L, @V, etc.)で2,3またはB,3と入力すると、各プロットの5つめのY値を凡例テキストとして表示します。プロットのソースワークシートの2列目であるBの3 行目の値を凡例テキストとして表示します。

詳細は@オプションの完全リストを参照してください。

自動軸タイトルに単位 %(?X)  %(?Y)を(可能なら)付ける

このオプションを選択すると、ワークシート列の「単位」セルを軸タイトルに追加します。

自動軸タイトルのプロットインデックス

このオプションは、1つのX列に対して複数のY列をプロットしているときに便利です。 ドロップダウンリストを使って、自動軸タイトルとしてどの列の情報を使用するかを指定します。

Note: システム変数@FPTで自動軸タイトルのプロットインデックスで関数グラフを除外するか設定できます。

凡例更新モード

これは新しいデータプロットがグラフに追加された時に凡例をどのように変更するか指定します。ドロップダウンリストで3つのオプションを利用できます。

追加時に更新 グラフ上の関連するレイヤにデータプロットが追加または削除されるたびに、凡例を再構築します。

Note: 既存の凡例の形式と構造は変更されません。

追加/削除時に再構成 グラフ上の関連するレイヤにデータプロットが追加または削除されるたびに、凡例を再構築します。

Note: 凡例の形式がデフォルト設定に戻ります。

なし グラフ上の関連するレイヤにデータプロットが追加または削除されても凡例を更新しません。

全てのレイヤのプロットを凡例に含める

このボックスのチェックを外すと、凡例オブジェクトが作成されたときにアクティブだったグラフレイヤに含まれるデータセットのみが凡例に含まれます。 一部のグラフ テンプレート（特に、2Y軸グラフテンプレートなどの複数のグラフレイヤを使用するもの) では、この機能がデフォルトで有効になっています。 有効にした場合、グラフウィンドウ内のすべてのデータセットが凡例に自動的に表示されます。

Note: このオプションはカテゴリー値の凡例と1ポイントずつの凡例では使用できません。

凡例シンボルの幅 (フォント幅の%)

この設定は、凡例のデータプロットタイプのアイコン幅を、凡例のフォントサイズの割合で決定します。

この設定はカラースケールやバブルスケールには影響しません。

アクティブデータセットの標識

アクティブデータセットの標識チェックボックスを選択すると、凡例内のアクティブデータセットに印を付けます。 このチェックボックスが選択されていると、凡例内でアクティブデータプロットのシンボルの周りを四角で囲みます。

さらに、このチェックボックスが選択されていると、凡例内のデータプロットのシンボルアイコンをクリックして、データプロットをアクティブにすることができます。

Note:

  1. このチェックボックスは、凡例を右クリックしてコンテキストメニューから凡例：アクティブデータセットの標識を選択するのと同じです。
  2. データメニューの一番下にあるデータリストやショートカットメニューからアクティブデータプロットを設定することもできます。(例えば、プロットを右クリックして、アクティブにするを選択します。)
