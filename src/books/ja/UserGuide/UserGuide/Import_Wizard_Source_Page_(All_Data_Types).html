インポートウィザード、データソースページ (すべてのデータタイプ)


このページのオプションを使って、インポートするファイルの名前、データタイプ、ターゲットテンプレート、インポートフィルタを指定します。このページは、すべてのデータタイプ(ASCII、バイナリ、ユーザ定義)に対して共通に使われます。 いくつかのオプションは、このページで行った別の設定によって変わります。そのため、このダイアログボックスの上から順に設定をしていくことをお勧めします。

データタイプ

ASCII

拡張ASCII文字を含むASCIIのファイル
バイナリ

任意のヘッダセクションの後にバイナリ構造のデータが続くようなファイル
ユーザ定義

ASCII または バイナリ インポートルーチンで読むことのできないファイル インポート手順を記述したOrigin Cコードを使う必要があります。

データソース

ファイル

ASCII、バイナリ、ユーザ定義のデータタイプからデータをインポートするには、「ファイル」ラジオボタンを選択します。ファイル名を入力します(または、参照ボタンを使ってデータファイルの場所を指定します。)。
クリップボード

クリッップボードのコピーされたデータをインポートするには、「クリップボード」ラジオボタンを選択します。

インポートフィルタ

「データタイプとファイル名の両方に適用可能なフィルタを表示」チェックボックス

このチェックボックスにチェックが付いていない場合、データタイプに適用可能なすべてのフィルタは、「現在のデータタイプのインポートフィルタ」ドロップダウンリストに表示されます。このチェックボックスにチェックが付いている場合、「現在のデータタイプのインポートフィルタ」ドロップダウンリストには、インポートウィザードの「フィルタの保存」ページの「このフィルタに対応するファイル名を...」に属するフィルタ(ワイルドカードで指定したフィルタも含む)を表示します。
「現在のデータタイプのインポートフィルタ」ドロップダウンリスト

「現在のデータタイプのインポートフィルタ」ドロップダウンリストは、次の場所にあるすべてのフィルタを表示します。

  • データファイルと同じサブフォルダ これらのフィルタ名の前に、「データフォルダ」という文字が付いています。
  • ユーザファイルパスの\Filters サブフォルダ これらのフィルタ名の前に、「ユーザフォルダ」という文字が付いています。
  • Originのプログラムフォルダ内の \Filters サブフォルダ これらのフィルタ名の前に、「Originフォルダ」という文字が付いています。
説明

この編集不可のテキストボックスは、インポートウィザードの「フィルタの保存」ページでの「フィルタの説明」テキストボックスに入力した説明文が表示されます。

ターゲットウィンドウ

ターゲットウィンドウ

データインポートのターゲットウィンドウ -- ワークシート、行列、無し -- を指定します。
テンプレート

「テンプレート」には新しいウィンドウを作成するのに使用するテンプレートを指定します。新しいウィンドウは次の場合に作成されます。

  • ドラッグ&ドロップの操作：
  • 「ファイル：開く」コマンドを使って、データファイルを開く場合
  • 「インポートモード」ドロップダウンリストでの設定 (次を参照)

テンプレート無し を選択すると、Originのデフォルトのワークブックまたは行列ブックのてんぷれーとが使われます(これは「標準」ツールバーから「新規」ボタンをクリックして開かれるワークブックまたは行列ブックと同じです。)

フィルタが名付けられたファイルタイプに対して存在していて、テンプレートの指定がある場合、新しいウィンドウが「現在のデータタイプのインポートフィルタ」ドロップダウンリストで指定されたテンプレートを使って作成されます。
インポートモード

「インポートモード」ドロッップダウンリストのオプションにより、データをインポートする場所が決まります。(ターゲットウィンドウとテンプレートの指定に一致)

「インポートモード」が「既存データを置き換える」「列を追加して読み込む」「行の末尾に追加する」「シートクを新たに作って読み込む」にセットされている場合、データはアクティブウィンドウにインポートされます。

インポートモードが「ブックを新たに作って読み込む」にセットされているか、アクティブウィンドウが空ではなかったり、ターゲットウィンドウタイプではない場合、指定した「テンプレート」を使って作成された新しいウィンドウにデータがインポートされます。
