クライアントアプリケーションとしてMicrosoft® Excelを利用する

Excel-client-app

Originのオートメーションサーバを使って、Microsoft® Excelツールをクライアントアプリケーションにします。サンプルでは、補間するために、Excelを使ってデータをOriginに送ることができます。

上記のサンプルで、補間用の素データ（青色の列ヘッダ）と新規のX値を確認することができます。「分析テンプレートを開く」ボタンをクリックします。この情報は、ExcelからOriginに見えない形で送信され、分析テンプレートの最初のシートに入力されます。この分析テンプレートは自動更新に設定され、補間の結果列（下のスクリーンショットのD列） は元のデータ（A～C列）に変更があるたびに更新されます。

元データは3次補間法により補間されます。補間した結果は自動的にOriginでグラフ化されます。

Excelクライアントは補間結果のデータを読み取り、Excelワークシートの　"Interpolated Y" の列に表示します。Excelクライアントは、Originで作成したグラフのEMF画像を取得し、同様に、Excelワークシートに表示します。全ての操作はOriginを画面に表示することなく行われます！

ExcelファイルInterpolation.xls はOriginをインストールしたパソコンの次のサブフォルダに保存されています：\Samples\COM Server and Client\MS Office\Excel.このサンプルの分析テンプレートはInterpolation.opjです。同じ場所に保存されています。

このサンプルはどのようにプログラムされたものでしょうか？

Excelクライアント側では、Excelワークシートに埋め込まれたVisual Basicマクロを使用してこのプログラムは実行されています。サーバ側では、分析テンプレートを使用して、補間処理とグラフの更新が自動的に行われています。
