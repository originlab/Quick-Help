オートメーションサーバ

Automation-Server

コンポーネントオブジェクトモデル (COM) はソフトウェアコンポーネント上のMicrosoft® centric interface standard です。COMでサポートしているプログラミング言語での、内部コミュニケーションとダイナミックオブジェクト作成を可能にします。COMは、OLE、OLEオートメーションのActiveX®、COM+およびDCOMを含む包括的な用語としてソフトウェア開発業界で広く使用されています。

現在Originは、マイクロソフトExcel®、マイクロソフトのVisual Basic®、およびナショナルインスツルメンツ™のLabVIEW™、そのサポートCOMプログラミングなどのクライアントアプリケーションのためのオートメーションサーバーとして機能します。クライアントアプリケーションからOriginにデータを渡すことが出来、Originをパソコン上で表示することなく、グラフ化や分析した結果をアプリケーションに戻すことが出来ます。

一度クライアント側がOriginとの通信を閉じた場合は、ユーザはOriginに再度接続して、保存したロジェクトを以後の処理のために再度開きます。

Originのネットワーク（サーバ＆クライアント）は、この通信機能に最適です。FLEXnetライセンスマネージャを使って、サーバとして1台のPC、クライアントとして他の複数のPCにインストールして利用します。クライアントPCを利用しているユーザは、ExcelワークシートまたはVisual Basicアプリケーションのような、一般的なアプリケーションを組織から提供されていると思います。これらの一般的なインターフェイスは、ローカルなPCからOriginサーバと通信するためのインターフェイスとして利用出来ます。その場合、クライアントのユーザはOrigin自体の操作知識が無くても、Originをカスタマイズしたインターフェイス上で全て実行することが出来ます。

このセクションで説明している項目

  • クライアントアプリケーションとしてMicrosoft® Excelを利用する
  • クライアントアプリケーションとしてMicrosoft® Visual Basicを利用する
  • クライアントアプリケーションとしてMathworks MATLAB® を利用する
  • クライアントアプリケーションとしてNational Instruments™ LabVIEW™を利用する
  • OriginのドキュメントをXMLドキュメントに変換するためにMSXML Parserを利用する
