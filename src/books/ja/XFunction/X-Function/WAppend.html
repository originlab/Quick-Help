メニュー情報

データ整形: ワークシートの追加

概要

1つのワークシートに複数のワークシートを追加します

追加の情報

必要なOriginのバージョン: Origin 2015 SR0以降

コマンドラインでの使用法

wAppend irng:=([Book1]Sheet1!, [Book1]Sheet2!, [Book2]Sheet1!) method:=row ow:=[<new>]<new>;

wAppend irng:=[Book1] method:=column ow:=[<new>]<new> -r 2;

Xファンクションの実行オプション

スクリプトからXファンクションにアクセスする場合、追加のオプションスイッチについてのページを参照してください。

ワークシート wks_list

入力

int

0

この変数は内部的な使用のみでスクリプトからはアクセスできません。

入力 irng

入力

Range

<unassigned>

これは、入力するワークシート範囲を指定するのに使用します。

シンタックス：　([BookName]SheetName!,[BookName2]SheetName2!)[BookName] のみを指定すると1つの出力シートにブックにあるすべてのシートが追加されることに注意してください。

追加方向 method

入力

int

0

ワークシートの追加方向を指定します。 オプションリスト：

  • column:列{0}
    ワークシートは新しい列として追加されます。
  • row:行{1}
    ワークシートは新しい行として追加されます。
各列の最後に追加 end

入力

int

0

追加方向を行 にした場合に利用できます。ワークシートの最大境界を考慮せずに、各列の最後に行方向で追加されます。

データセット識別子 id

入力

int

3

出力ワークシートに表示するソースデータセットの識別子を指定します。 オプションリスト：

  • none:なし{0}
    ソースデータセット識別子を表示しません。
  • range:範囲{1}
    ソースワークシートの範囲を表示します。
  • book:ブック名{2}
    ソースワークブックの名前を表示します。
  • sheet:シート名{3}
    ソースワークシートの名前を表示します。
  • index:インデックス{4}
    インデックスで指定されたソースワークシートのインデックスを表示します。
  • sheetLabel:シートラベル{5}
    ソースワークシートのシートラベルを表示します。
インデックス index

入力

string

<自動>

データセット識別子がインデックスのとき使用できます。出力で表示するソースワークシートのインデックスを指定します。

シンタックス：開始：増分、（例えば0:0.5(0, 0.5, 1, 1.5...)）、デフォルトは<自動>で 1:1と等しい。

データの前に識別子を挿入する idpos

入力

int

0

追加方向が行でデータセット識別子がなしでない場合に利用できます。データセット識別子をデータの前に置くか後に置くかを指定します。
