FAQ-976 分析テンプレートのファイルサイズが大きくなってしまうのはなぜですか？

Analysis-template-file-size-growing-bigger

最終更新日：2018/12/07

Origin 2018、Origin 2018b、Origin 2019での問題により（問題はこれらのバージョンにのみ適用されます ）、分析テンプレートファイル（.OGWまたは.OGWU）のサイズが、バッチ処理で使用するたびに大きくなる場合があります。

この問題を解決するには、システム変数の設定ダイアログ、コマンドウィンドウ、またはスクリプトウィンドウでシステム変数@BST=0 を設定します。

分析テンプレートファイルのサイズを元のサイズに戻すには、次の手順を実行します。

  1. ファイル：開くメニューから分析テンプレートファイル(.OGWまたは.OGWU)を開きます。
  2. インポートデータのあるワークシートをアクティブにします。
  3. ウィンドウメニューからワークシートのクリア...を選択し、注意ダイアログでOKをクリックします。
  4. コマンドウィンドウまたはスクリプトウィンドウで、次のコマンドを実行します。 del -r 16;
  5. ファイルメニューから、ワークブックを分析テンプレートとして保存...を選択し、テンプレートファイルを再保存します。

@BST=0を設定し、分析テンプレートファイルを元のサイズに戻した後は、自身で変更しない限り、サイズが再び大きくなることはありません。
