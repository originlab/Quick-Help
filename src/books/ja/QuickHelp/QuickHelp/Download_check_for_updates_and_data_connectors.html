FAQ-1003 Originがインターネットに接続してアップデートのチェックなどができないのはなぜですか？

Download-updates-data-connectors

最終更新日：2023/1/30

次のいずれかで問題が発生した場合…

  • ヘルプファイルを取得/アップデートのチェック
  • アプリセンターからアプリをダウンロードまたはアップデート
  • ラーニングセンターをアップデート
  • データコネクタを使用してインターネット上からデータをインポートできます。


ネットワーク上のプロキシサーバーの使用が関係している可能性があります。多くの組織ではプロキシサーバーを使用し、制御されたポータルを介して特定のネットワークをルーティングします。デフォルトでは、Originはプロキシサーバーの設定に従わないことがあり、それによりさまざまな機能に必要なインターネット関連のアクティビティの実行がブロックされる場合があります。


Origin 2019からは、Originのプロキシサーバーの設定を変更して、問題が解決するかどうかを確認することができます。

  1. 環境設定：オプションを選択します（Origin 2019ではツール：オプションメニュー）。
  2. オプションダイアログでその他のオプションタブに移動します。
  3. プロキシのコントロールには、3つのオプションがあります。
    • 直接接続はプロキシサーバーをバイパスします。つまりOriginはターゲットに直接接続しようとします。
    • システム設定を使用するは、Webブラウザやシステム上のその他多くのネットワーク指向ソフトウェアで通常使用されるものと同じプロキシサーバー設定を使用します。
    • HTTPプロキシサーバーを使うでは、ハードコードディングされたプロキシサーバーのIPアドレスとポートを指定できます。
      認証が必要なプロキシサーバーを使用する場合は、次の構文を使用します。 
      [username]:[password]@[server name|server ip]:[port]

      ユーザー名またはパスワードに「パーセントエンコード予約文字」が含まれている場合は、それらをパーセントエンコードのものに置き換えます（Note：パーセントエンコードはOrigin 2021bからサポートされています）。 例えば、 username = user@a.com, password = pwd, server name = proxy and port = 3129 この場合、次のように記述します。 

      user%40a.com:pwd@proxy:3129


まず初めは、システム設定を使用するのオプションを選択して問題が解決するかどうかを確認することを推奨します(新しい設定を有効にするにはOriginを再起動する必要があります)。それでも問題が解決しない場合は、ハードコーディングされたプロキシサーバーのアドレスとポートをHTTPプロキシサーバーを使うのオプションで入力する必要があります(その後再起動を行います)。使用する適切なアドレスとポートについては、IT部署のご担当者に相談をしてください。

Originでこれらの変更を行っても問題が解決しない場合、WindowsまたはアンチウイルスファイアウォールがOriginをブロックしている可能性があります。Originのトラフィックの通過を許可する方法については、関連するファイアウォールのドキュメントを参照してください。


キーワード：インターネット、http、https、アップデートのチェック、データコネクタ、ヘルプファイル、プロキシ、ウェブ、ネットワーク
