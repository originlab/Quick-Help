FAQ-1005 エクスポートされたグラフのサイズがOriginで設定されたページサイズよりはるかに小さいのはなぜですか?

export-graph-size-small

最終更新日: 2019/5/10

emf、pdf、eps、wmfといったベクター形式としてグラフをエクスポートすると、エクスポートされた画像サイズが予想より小さいことがあります。

方法1：

グラフエクスポートダイアログのエクスポート設定: 詳細を開き、サイズを保存を選択します。

方法2：

上記の解決策ではうまくいかない場合があります。

考えられる問題の原因

グラフが、Microsoft WordやPowerPointなどのサードパーティ製ソフトウェアの埋め込みグラフから保存されたものである場合があります。つまり、ある時点でグラフがOriginからWord/PPTにコピーされ、その後Word/PPTから再起動され、Originの新しいセッションに再度保存されたということです。

このページサイズをリセットするには

ウィンドウ: スクリプトウィンドウを開き、「問題の起こっている」グラフをアクティブにして、次のLabTalkスクリプトを実行します。

doc -e p { page.info.system.export.auto=1;}

キーワード:グラフエクスポート, ページサイズ, 大きさ維持, 小さいサイズ
