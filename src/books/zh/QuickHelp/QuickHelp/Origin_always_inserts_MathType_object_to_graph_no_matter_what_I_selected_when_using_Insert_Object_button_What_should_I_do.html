1.279 FAQ-1146 当点击插入对象按钮时，尽管选择了 AxMath，Origin 却总是在图上插入 MathType 对象，如何解决此问题？

Fail-to-Insert-AxMath-Object-MathType

Last Update: 10/26/2022

如果同一台电脑上同时安装了 MathType 7.x 和 AxMath， 那么您会发现，在其它的一些应用中，比如 Origin 或者 Word， "插入OLE对象" 工具会被 MathType 劫持。例如，在 Origin 中，使用 插入对象 工具栏按钮在图形窗口上插入OLE对象，尽管在 插入对象 对话框中选择了 AxMath，Origin却总是插入 MathType 对象。

这个问题的产生是由于 MathType 7 及之后的版本在安装的时候在注册表中设置了AutoConvertTo键。

解决方法：

  1. 点击 Windows 的开始按钮，在运行输入框中输入： 
    regedit

    然后会打开 注册表编辑器。

  2. 在 注册表编辑器 中，定位到 AxMath 的键。 
    Computer\HKEY_CLASSES_ROOT\WOW6432Node\CLSID\{B18C2BCC-4E79-436A-A2A5-A7F8D25A9A28}

    在此下面有一个名为 AutoConvertTo的键，您会发现它的值被设置成了 MathType。

  3. 删除 AutoConvertTo 键之后再尝试插入对象，劫持的问题应该得到解决。

Keywords:Microsoft, AutoConvertTo, OLE, Insert Object, MathType, AxMath，微软，插入对象
