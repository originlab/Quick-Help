5.64 FAQ-926 是否可以控制将哪个窗口作为文件资源管理器中的缩略图以及显示在预览窗格中？

Control-Origin-Window-Display-File-Explorer-Preview

Last Update: 7/19/2018

当使用 Origin 2018b 或更高的版本保存 OPJU 文件时，其中的一个项目窗口将被用来在 Windows 文件资源管理器中作为缩略图显示 (在文件资源管理器中，在项目文件所在文件夹的空白处右击，然后选择一种查看方式，例如，超大图标)。此外，当启用文件预览窗格 (ALT + P) 时，同一窗口将显示在预览窗口列表的顶部。

在最简单的情况下，激活的图形窗口中的图形将作为项目文件图标的缩略图。在更复杂的项目文件中，或者在需要使用特定的窗口类型作为项目图标的情况下，可以通过设置 "@ 系统变量" @SPM 来施加某些控制，这与以下窗口的优先级排名有关： 

Matrix = 1
Worksheet = 2,
HolderImage = 3 //book with MatrixSheet in Image View
MatImage = 4  //matrix in Image View
HolderGraph = 5 
GraphNotOpen = 6 //in another folder, hidden or iconized
Graph = 7

... 其中， "Holder" 表示的是嵌入在工作簿或者工作表中的图形或矩阵，如果是矩阵的话，则视图必须是图形模式 (View: Image Mode)。

系统变量 @SPM 用来确定一个给定的窗口类型是否将提升为"如果激活就必须使用"。 默认情况下 @SPM = 3 ，但是可以通过脚本窗口 (Window: Script Window) 将其修改为以上列表中的任一值。系统变量 @SPM 的值是存储在 Windows 注册表中的，并且会在以后的 Origin 操作中保持不变，直到再次修改。

例如，如果设置变量的值为 @SPM=5，则仅使用图形，既包括独立的图形窗口，也包括 "HolderGraphs" (将图形作为工作簿嵌入工作簿中的嵌入图)。如果当前激活的窗口不是图形窗口，将使用另一个可见的且可用的图形窗口 (不是最小化的) 。如果当前文件夹中有多个图形窗口，则使用最后被激活的那一个。

如果担心预览图会影响文件的大小，则可以使用系统变量 @PESS 阻止项目资源管理器将预览保存到文件中。
