2.137 FAQ-1182 如何导入东欧/中欧字符？

Import-Eastern-Central-European-Characters

Last Update: 10/16/2023

要导入包含东欧/中欧字符（ANSI 编码）的数据文件，请尝试以下三种方法之一：

解决方案1：暂时将@cps设置为1250
  1. 选择Origin菜单窗口：脚本窗口。
  2. 在打开的脚本窗口中输入
    @cps=1250
    并按Enter键临时设置[1] php?title=LabTalk:System_Variable_List#C @cps] 为1250。
  3. 导入您的数据。
注意：导入后记得将 @cps 设置回 -1（默认）。
解决方案 2：将操作系统代码页更改为捷克语
  1. 打开 Windows 控制面板 。
  2. 选择时钟和区域 > 区域链接以打开区域对话框。
  3. 转到管理选项卡。 在非 Unicode 程序的语言部分下，单击更改系统区域设置按钮。
  4. 选择系统区域设置捷克语（捷克）。
  5. 单击确定。 根据提示重启电脑。
  6. 重新启动后，尝试导入您的数据。
注意：此更改将影响整个操作系统，Micro软件（Word、Excel、PowerPoit 等）、记事本也可以正确显示东欧/中欧字符。
解决方案 3：将文件重新保存为 UTF-8 编码
  1. 按照上面解决方案 2 中的步骤将操作系统代码页更改为捷克语（捷克）。
  2. 在记事本中打开数据文件。 它应该会被正确显示。
  3. 从菜单中选择文件：另存为，然后为编码选择UTF-8，以编码为 UTF-8 保存。
  4. 现在您可以将操作系统代码页更改回原来的状态，然后尝试导入您的数据。 应该正确导入。
