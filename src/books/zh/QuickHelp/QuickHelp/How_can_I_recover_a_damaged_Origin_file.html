2.32 FAQ-464 怎样恢复损坏的 Origin 文件？

Recover-Damaged-OPJ

Last Update: 10/19/2020

当你保存 Origin 项目文件时，Origin 会保存一份拷贝来备份。如果你发现你的项目文件损坏了，可以打开备份目录来恢复文件先前保存的版本：

  • 在 Origin 2018b 以及之后版本中, 每个项目文件都有单独的备份，这些文件存储在 \Backup 文件夹中(比如 C:\Users\user\Documents\OriginLab\Backup) 并以如 myProject-1234567.opju 这样的的文件名命名。）
  • 在先前的 Origin 版本中, 在 User Files 文件夹下 (如 C:\Users\user\Documents\OriginLab\2017\User Files) 只有一个以 Backup.OPJ 命名的备份文件。

1. 找到并打开你的备份项目文件

  • 在 Origin 2018b 以及之后版本中,选择菜单 Help: Open Folder: Project Backup Folder 或 选择菜单 Preference: Options,转到 System Path 选项卡 将 \Backup 路径复制粘贴到 Windows 的资源管理器的地址栏中。
  • 在 Origin 2018 以及之前版本中,你会发现在 User Files 文件夹 下只有一个文件名为 Backup.OPJ 的备份文件。 选择菜单Preference: Options, 单击 System Path 选项卡，复制 User Files Folder 路径并粘贴到 Windows 的资源管理器的地址栏中。


找到所需要的备份文件后，用 Origin 打开：

  • 对于Origin 2018b 和更高版本，将文件另存到新的位置。
  • 对于Origin 2018 和更高版本，将文件保存到新的位置并重命名。 ( 除了 "Backup.OPJ" 这个名称之外的文件名).

2. 从损坏的项目中恢复数据

如果方法 (1) 没用， 在备份文件中找不到所需数据。可以尝试下列方法：打开损坏的 Project 文件，再打开 Script 窗口 ( Window:Script Window ) 并输入： 

list s

按 Enter 键以确认。如果文件损坏不是非常严重，将返回工作簿列表，它看起来与下面的示例相似: 

 1	MyData_Time
 2*	MyData_Sample1
 3*	MyData_Sample2
 4	Data2_A
 5*	Data2_B
 6*	Data2_C

列表中列出了两种不同的工作表 : MyData 和 Data2。要恢复这些工作簿，需要输入以下命令依次恢复工作表： 

Edit MyData_Sample1;

按 Enter 键确认。这将打开包含三个工作簿的 "MyData" 的工作表。 Origin 2021之后，你可以输入以下命令一次性恢复所有工作簿。 

run -oc EditAllSeries;

但不幸的是，图形窗口无法被恢复。

在 "打开" 每个工作表之后，将已损坏 Origin 文件中的数据复制并粘贴到未损坏 Origin 文件中。或者，你可以 (选择菜单 File: Export ASCII）来将工作表导出为 ASCII 文件，然后将 ASCII 文件导入到新建的 Origin 项目文件中。 </ol>
更多详情请参阅： Origin 文件的打开，关闭和备份。
