5.9 FAQ-346 如何解决 Origin 在高分辨率屏幕中的显示问题？

Mac-Retina-Display-of-Origin

Last Update: 5/10/2022

现在有越来越多的显示器具有比普通的显示器更高的分辨率(高达 227 ppi vs 96 ppi)。虽然 Origin 能在原始的分辨率下运行，但是经测试表明，在使用分辨率为 1920 X 1200 和中等的系统文字缩放(125%)的环境下运行 Origin 可以获得更好的用户体验。

如果您的显示器具有更高的分辨率，例如分辨率为 3200 X 1800 的 QHD 显示器，您将会发现按钮和菜单文字或者对话框都会变得十分地小。若要解决这个问题，可以尝试以下的方法：

从 Origin 2022b 开始，窗口大小和位置会根据屏幕的分辨率和DPI在启动时进行调整，这就更容易在不同笔记本和高分辨率桌面显示器之间共享项目。此功能是由系统变量@SRWS控制/微调。

方法 1

  1. 选择 View: Toolbars... 打开 Customize 对话框。
  2. 选择 Options 标签页，勾选 Use Large Icons 和 Scale Toolbars for Large System Font Settings。
注: Scale Toolbars for Larger System Font Settings 可能会对工具栏按钮的分隔功能产生干扰（多个工具按钮将会堆叠在上一次使用的工具按钮后）。

如果方法1效果不好，那么可以尝试方法2。

方法 2

您可以如下述方法调整计算机的分辨率和 DPI 缩放：

  • 降低计算机的分辨率为 1900 X 1200
  • 提高 DPI 缩放使其具有较大的图标
注: 在调整屏幕分辨率或者系统文字缩放后，您需要注销 Windows 然后再次登录来确保新的设置生效。

Origin 在 Apple 的 MacBook 视网膜屏的显示问题

因为 Apple 的 MacBook 视网膜屏具有较高的分辨率，所以可能会导致 Origin 出现一些显示的问题。您可以尝试上述的方案来解决这个问题。在 MAC 上运行 Origin 的最佳屏幕设置也是分辨率为 1920 X 1200 和中等的系统文字缩放(125%)。

测试配置
硬件: MacBook Pro, 2.0GHz 四核 Intel i7, 15.4" 视网膜屏 (2880 X 1800, 220 ppi)。
软件: OS X Version 10.9.5, BootCamp 5.1, Windows 8.1, OriginPro 2015。

若想了解更多关于如何在 Mac 上运行 Origin 请参阅此页。

注: Parallels 和 VMware Fusion 的最新版本均支持在视网膜屏中运行 Windows。有关更多的详细信息请参阅相应的软件文档。
