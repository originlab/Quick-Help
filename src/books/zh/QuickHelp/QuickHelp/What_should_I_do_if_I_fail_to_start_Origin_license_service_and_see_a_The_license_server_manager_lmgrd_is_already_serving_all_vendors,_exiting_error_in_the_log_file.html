6.97 FAQ-1124 Origin授权服务启动失败，日志文件显示"The license server manager (lmgrd) is already serving all vendors, exiting."错误信息，该如何处理?

Lic-Server-Manager-Serving-AllVendors

Last Update: 5/10/2022

如果您试着点击 LMTools 的 Start/Stop/Reread 标签卡上的 Start Server 按钮来启动 Origin 授权服务器，但在状态栏看到以下的错误：

VD is starting, please check vendor daemon's status in debug log

当您查看日志文件，您可能会看到类似于下面的信息：

18:13:21 (lmgrd) orglab already running 27000@ServerName

18:13:21 (lmgrd) The license server manager (lmgrd) is already serving all vendors, exiting.


这有可能是因为您在同一台服务器上配置了两个 OriginLab 授权服务实例，所有第二个不可启动。

要解决这个问题，

  1. 去到 Config Serivces 标签卡，您会看到所有存在的服务都列在了 Service Name 下拉框。检查是否设置了两个或以上的服务到相同的 lmgrd.exe/Origin 许可文件。如果是这样，保留一个，停用另一个。
  2. 从 Service Name 下拉框选择不要的服务，然后去到 Start/Stop/Reread 标签卡，点 Stop Server 来停止这个服务。
  3. 现在去到 Service/License File 标签卡，选择您想要的服务保留下来。
  4. 再到 Start/Stop/Reread 标签卡，点 Start Server 按钮，看服务是否已经启动。

如果现在工作正常，您就可以安心的把不要的服务从 LMTools 上移除：

  1. 去到 Config Services 标签卡，从 Service Name 下拉框选择不要的服务，点 Remove Service 按钮把它移除。


如果您在 LMTools 只看到一个服务，但仍然有 "VD is already starting" 的错误，请试一下下面的方法：

  1. 打开 任务管理器 对话框。
  2. 去到 Services 标签卡（您可能需要点 "More details" 链接才能看到 Services 标签卡）。这台机器上所有的服务都列在这里。搜索您建立的 Origin 授权服务。
  3. 如果找到，右击并选 Stop 来停用。
  4. 回到 LMTools，试着重启服务。
