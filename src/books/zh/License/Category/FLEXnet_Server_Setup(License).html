安装必读！

  1. 本篇文档是主要面向新用户首次设置 FLEXnet 服务器的说明指南。 您也可以观看此英文视频：如何设置您的网络版 .
  2. 只要您的金牌服务没有过期，您就可以使用最新版本的 Origin。您只需要在客户端电脑上安装最新版的 Origin 即可。
  3. 如果您是要在 Linux 系统上安装F LEXnet 服务器，请看这篇文档。

选择一台计算机作为您的FLEXnet 服务器

服务器要求

  • 一台不需要频繁重启的稳定的机器，它可以是一台服务器或是普通电脑。
  • 系统内存不低于64 MB。
  • Origin 的 FlexNet 服务器支持的系统有：Windows Server 2022, 2019, 2016, 2012, 2008 和 Windows 11, 10, 8/8.1, 7, Vista (32 和 64-位)。
  • 畅通的局域网络。所有可能会安装 Origin 的计算机(即客户端)都可以连接到服务器。 FLEXnet 服务器与 Origin 客户端之间通过 TCP / IP 网络协议进行交互。

安装 FLEXnet 软件

  • Flexera Software (FLEXnet 软件的制造商) 建议，为避免安全风险，请升级 FLEXnet 服务器至最新版本。现最新版本为 11.19.4.1。
  • 必须安装 Microsoft Visual C++ 2015 Redistributable Package 组件包 FLEXnet 服务器才能正常运行。如尚未安装，请先下载并安装。


在新的机器上安装FlexNet服务器

从下列官网地址下载 FLEXlmServerSetup.exe 并安装即可。如果您有 Origin DVD，您也可以从DVD中安装。请不要将它安装到 C:\Program Files\ 或电脑桌面上。

  • FLEXlmServerSetup32Bit.exe(32位 FLEXlm 也可以安装在 64 位 Windows 系统上)
  • FLEXlmServerSetup64Bit.exe

如果您选择的服务器已经安装了 FLEXnet 并且不安装在 C:\Program Files\ 文件夹或桌面上，您不想重复安装，可以下载并解压缩下面的文件，用其中的 lmgrd.exe, orglab.exe 和 lmtools.exe 来替换原有文件。

  • FLEXnetServer11.19_Files_32Bit.zip
  • FLEXnetServer11.19_Files_64Bit.zip

获取FLEXnet 服务器的许可证文件

找到作为 FLEXnet 服务器的计算机的 Host Name 和 Host ID

  1. 安装了 FLEXNet 服务器后，打开安装目录。 在 \Tools\ 文件夹下，双击运行 LMTools.exe。
  2. 记下 System Settings 选项卡上的 Computer/Hostname 和 Ethernet Address

    或，
    打开命令提示符并输入命令： 
    ipconfig /all


    记下 Host Name 和 Physical Address （也称为 MAC Address 或 Ethernet Address）

从 Originlab.com 获取许可证文件

  1. 打开 Originlab.com 网页，并登陆。如果您还没有账号，请先自行创建一个。
  2. 登陆后，在页面上点击 "Manage My License(s)" 链接。
  3. 点击Register New Serial Number 按钮。
  4. 选择 "I want to register my Origin software and obtain a license file for a computer" 并点击 Next.
  5. 按要求输入您的 Origin Serial Number(序列号) 和 Version(版本) ，然后点击Continue。
  6. 输入 FLEXnet 服务器的 Host Name 和 Host ID (Ethernet Address/Physical Address)，然后点击 Submit 按钮以获取许可证文件。
  7. 网页上将生成许可证文件的文本。从网页上复制许可证文件文本，粘贴到文本编辑器（例如记事本Notepad）中。
  8. 在文本编辑器中，选择菜单 文件：另存为，设置 保存类型 为 所有文件(*.*)， 将文件命名为 orglab.lic，并保存在F LEXnet 安装目录的 \Licenses\ 文件夹下（当然，您也可以把它保存到其他路径下）。请注意，该文件是保存为 orglab.lic 而不是orglab.lic.txt (后缀名为.lic)。

设置并启动 FLEXnet 服务器

  1. 打开 FLEXNet 安装目录下的 \Tools\ 文件夹，运行 LMTOOLS.exe。如果是 Windows Server 2012/2008, Windows Vista/7/8 系统， 需要右击 LMTools.exe 并选择 "以管理员身份运行" 。
  2. 在 Service/License File 选项卡下，选择 Configuration Using Services。
  3. 在 Config Services 选项卡下，
    • 在 Service Name 文本框中输入 OriginLab 或其他您喜欢的服务名称。
    • 点击 Path to the lmgrd.exe file 右边的 Browse 按钮，到 FLEXNet 的安装目录下的 \Servers\ 文件夹中选择 lmgrd.exe。
    • 点击 Path to the license file 右边的 Browse 按钮，到 FLEXNet 服务器的安装目录下的 \Licenses\ 文件夹中选择 orglab.lic。
    • Path to the debug log file 文本框中已经预设好一个默认路径（C:\ProgramData\FLEXlm\lmgrd\debug.log）。日志文件debug log 此时还未生成 - 当您启动服务后，服务器会根据 Path to the debug log file 文本框中的路径来自动创建此文件。
    • 勾选 Use Services。
    • 如果您希望在服务器启动时服务自动启动，请选择 Start Server at Power Up。
    • 点击 Save Service.
  4. 在 Start/Stop/Reread 选项卡下， 勾选 Force Server Shutdown。
  5. 点击 Start Server 按钮。在LMTOOLS状态栏中，您会看到 "Server Start Successful" 的消息。
  6. 在 Server Status 选项卡下，点击 ‘’‘Perform Status Enquiry’‘’ 按钮。在输出的运行状态报告中查看服务器是否正常运行。


故障自我排查

如果遇到 Origin 无法与 FLEXnet 服务器连接的情况，

  1. 首先，在 FLEXnet 服务器上打开 LMTools.exe。在 Server Status 选项卡上点击 Perform Status Enquiry 按钮，查看输出结果，判断服务器是否正常运行。
  2. 在确保服务器正常运行的情况下仍然无法连接，很有可能是因为防火墙仍然阻止了服务器和客户端之间的交互，请参阅这一小节进行进一步设置。
  3. 如果 Origin 能通过防火墙却还是连接不上服务器，在 Config Services 选项卡上查看日志文件（debug log file）。
  4. 将序列号与日志文件发送到技术支持的邮箱: support@originlab.com.cn。
