3 多用户单机版（Node-Locked）许可证

Multi-User-Node-Locked-Lic

如果您的序列号如下，那么您正在使用的是多用户单机版许可证：

xFxxx-9xxx-76xxxxx, 或者
xFxxx-6xxx-76xxxxx.

如果您的序列号是:

xxxxx-2xxx-79xxxxx, 或者
xxxxx-3xxx-79xxxxx,

那么您使用的是网络版(Concurrent Network) 许可证，以下信息则不适用于您的情况。

根据您购买的许可证的数量，多用户单机版可以授权安装Origin在同等数量的计算机上。每台计算机都需要从OriginLab网页上获取一个许可证，并且所获取的许可证是该计算机专用的。如果您想更换已获得Origin许可证的计算机，那么您需要先停用此计算机上的许可证，然后才能在新的计算机上获取新的许可证。

管理员请注意：在您分配Origin 给您的用户前...

对于多用户单机版, 我们提供了一个机制，可以控制哪些OriginLab网页账号有权使用您的序列号来安装Origin并获得许可证。此机制旨在防止其他人未经授权使用您的序列号，以确保您所购买的许可证均为贵司的授权成员所使用。

如何在OriginLab网站上建立产品密钥分配机制

作为Origin管理员，请完成以下步骤以注册序列号，生成产品密钥并确定将产品密钥分配给用户的方法。请在告知用户Origin软件可供使用之前完成此操作。

  1. 购买Origin后，您将收到一封包含您的Origin序列号和注册链接的电子邮件。
  2. 点击注册链接进入OriginLab网站。按要求登录。如果您还没有帐户，请先自行创建帐户。
    如果您收到的是注册码，请登录OriginLab网站（www.originlab.com），并点击Welcome页面上的 Register my product -version 2017 (9.4) 链接，然后继续第3步的操作。
  3. 登录后，您的序列号，注册码，和3个分配产品密钥的方法将显示在网页上。选择下列其中一个选项。请仔细考虑后再选择，因为一旦点击Register 按钮，您将无法更换选项:
    • I will obtain all license Product Keys myself
      如果您选择此选项，所有的产品密匙都会列在您的账户名下。您将负责在指定电脑上安装Origin，运行Origin并输入产品密钥激活，或者将产品密钥分发给您的用户，让他们自行激活。
      您将会:
      1. 使用您的帐户登录到OriginLab网站。
      2. 点击 "Manage My License(s)" 链接.
      3. 在表格中找到相应的序列号，点击链接。您将会看到所有的产品密钥。
      4. 您可以选择任一未使用的产品密匙，在一台电脑上运行Origin，并使用密钥激活许可证。
    • I will approve user requests for license Product Keys
      如果您选择此选项，您将获得一个URL，用户可以通过这个URL向您申请产品密钥。 在将URL发送给用户之前，您需要添加用户电子邮箱后缀。
      您将会:
      1. 通过电子邮件将URL发送给用户。
      2. 当他们通过URL请求获取产品密钥时，您将收到一封电子邮件。
      3. 然后您可以到"Handle Product Keys request"的页面，在表格中将产品密钥分配给选定用户。这将会自动生成一封给用户的邮件，其中包含了产品密钥。
      4. 用户将可以安装并运行Origin，然后使用产品密钥来完成许可证激活。
    • Each user will obtain a license Product Key
      如果您选择此选项，您将获得一个URL，用户可以通过这个URL自行申请产品密匙。在将URL发送给用户之前，您需要添加用户电子邮箱后缀。
      您将会:
      1. 通过电子邮件将URL发送给用户。
      2. 当用户点击URL时，会生成一个产品密钥并通过电子邮件发送给用户。
      3. 用户将可以安装并运行Origin，然后使用产品密钥来完成许可证激活。
  4. 根据需要输入电子邮箱后缀
    • 如果您在上一步中选择了 I will obtain all license Product Keys myself，则无需设置邮箱后缀。
    • 如果您选择了 I will approve user requests for license Product Keys，
      • 如果您希望只有指定的用户能够申请许可证。
        输入您的登录帐户所用的完整的电子邮箱地址。此后，您可以将有权申请许可证的用户的完整电子邮箱地址逐个添加到“邮箱白名单”列表（具体步骤请参考下面的提示）。 您可以随时回到OriginLab网站添加这些邮箱地址，只要在用户申请许可证前，均可添加他们的邮箱地址。
      • 如果您希望限定凡是贵司的成员均能申请许可证，
        您可输入您的机构的电子邮件地址后缀，以确保只有有贵司邮箱的人员才能获得许可证。
    • 如果您选择了 Each user will obtain a license Product Key， 您可以输入您的完整的电子邮件地址，稍后再添加其他完整的电子邮箱地址，或输入您的机构的电子邮箱地址后缀。具体步骤请参考下面的详细描述。

    在成功输入一个电子邮箱地址后，添加其他完整的电子邮箱地址或邮箱后缀，

    1. 点击位于OriginLab网站右上角的User Profile链接（如果您未登录，请登录）。
    2. 在"Welcome"页面上，点击License Access Restrictions链接。
    3. 按照打开的页面上的说明，添加其他的电子邮箱地址或后缀。
  5. 点击Register 按钮完成注册。

如果您所在的机构人数众多，例如您是一个校园版的许可证的管理员，我们提供了一个特别的Group Product Key，所有的用户都可以使用这个Group Product Key来激活自己的Origin，这便于您快速将Origin分配给大量用户。要生成Group Product Key：

  1. 登录到 OriginLab 网站。
  2. 点击 "Manage my Licenses" 链接，然后点击相应的序列号。
  3. 点击 "you can see all product keys here" 链接。 您可以在打开的页面上找到Request Group Product Key 按钮。

Group Product Key的好处在于，Group Product Key可以预设到安装文件当中，安装好的Origin可以自动激活，无需手动输入密钥：

  1. 将Group Product Key 保存在文本文件中并命名PK.txt.
  2. 将文件 PK.txt 放在Origin安装文件所在的文件夹下。
  3. 运行Origin安装文件进行安装。
  4. 安装完成后，启动Origin。 请注意，Group Product Key已预先填入 许可证激活对话框，并且许可证自动激活。

如果您需要批量安装Origin到多台电脑上，Group Product Key 与静态安装模式 相结合，可以大大的提高效率。

向您的用户提供Origin时的说明

当您向最终用户分配Origin时，除了向他们提供安装程序和序列号之外，还必须提供URL以申请产品密钥，因为在激活时必须输入产品密钥。此外，您必须告知您的用户，在激活时他们必须登录到OriginLab网站，如果他们还没有账号，请到OriginLab网站上注册。如果他们已经有账号，但使用的并不是机构的邮箱地址(例如，他们使用Gmail邮箱注册)，需要告知他们，在申请产品密匙之前，必须要登录OriginLab网页，点击"View/Update my Profile Details"链接，使用机构信息更新他们的账号。


批量安装Origin和用户间的资源共享

许可证激活

重要提示！
  • Origin许可证系统从2017版开始全新升级，单机版许可证的激活不再需要计算机的管理员权限。
  • 一旦成功激活许可证，这个许可证将保存到当前计算机中，此后不再需要访问互联网。

最终用户激活许可证激活的说明

  1. 从您的Origin管理员除获取您的序列号和相关的产品密匙。
  2. 打开Origin，许可证激活对话框将自动弹出。
  3. 选择在下面输入产品密钥，然后单击激活，输入您的产品密匙，然后点击激活。
  4. 激活过程将在后台完成。如果成功，您将收到成功激活的信息。

如果在后台的激活过程失败，请继续以步骤：

  1. 如果激活过程因为一些原因失败（例如，没有网络连接，originlab.com被防火墙阻止访问等），一个带有链接的记事本窗口会被打开。
  2. 在网络链接正常的情况下，将此链接复制到网络浏览器。该链接会打开OriginLab的网站。
  3. 许可证文本将生成并显示在网页上。
  4. 复制许可证文本，并粘贴到Origin的许可证激活对话框。点击确定。

管理员为用户的计算机激活许可证说明

如果您是多用户单机版的管理员，您可以让用户自行激活自己的Origin，也可以亲自为用户激活他们的Origin。在为序列号建立产品密钥分配机制时，如果您选择的是I will obtain all license Product Keys myself，您需要按照上述说明，为每一个用户的计算机激活许可证。

FAQ

  • 当我尝试申请产品密钥时，网页显示我无权获取许可证。如何解决这个问题呢？
  • 在升级新版本之前是否必须停用旧版本许可证呢？

停用许可证

如果您想将激活了的Origin软件从一台计算机转移到另一台计算机上，您需要先停用原来计算机上的许可证，然后才能在新计算机上使用相同的产品密匙来激活。

如果原来的计算机上的Origin仍能正常运行

  1. 启动Origin。选择帮助: 停用许可证，打开停用许可证对话框。
  2. 点击立即停用按钮。
  3. 许可证停用过程将在后台完成。 如果成功，您会收到成功停用的消息

如果后台停用许可证失败，请继续以下步骤：

  1. 如果停用许可证进程因为一些原因失败（例如，没有网络连接，originlab.com被防火墙阻止访问等），一个带有链接的记事本窗口会被打开。
  2. 在网络链接正常的情况下，将此链接复制到网络浏览器。该链接会打开OriginLab的网站。
  3. 该网页上会显示您的许可证信息。记下停用编码 Deactivation Code (DC)。
  4. 返回Origin。在许可证停用对话框底部的停用编码文本框中输入停用编码。点击‘’‘停用许可证按钮。
  5. 一个带有链接的记事本窗口会被打开。在网络链接正常的情况下，将此链接复制到网络浏览器。
  6. 在网页上点击Continue按钮。当您看到成功的消息时，则当前的许可证已经被释放，可以使用在其他电脑上了。

如果激活了Origin的计算机已经无法使用

如果您是管理员,

在OriginLab网站上完成以下步骤，以停用许可证。

  1. 打开OriginLab网页 (www.OriginLab.com)，登录。
  2. 在Welcome页面上，点击"Manage my License(s)" 的链接。
  3. 在 Manage my License(s) 页面上，点击目标序列号链接。
  4. 在打开的页面上，点击“you can see all product keys here” 的链接。
  5. 在 Origin Product Registration | License request history 页面上， 点击"go" 链接。
  6. 在 Deactivate Origin from a device 页面上，确认计算机的信息并点击Continue. 若操作成功，该计算机上的许可证将被释放，产品密匙可以使用在其他计算机上了。
  7. 若操作失败，网页会自动跳转到System Transfer Request Form页面。按要求填写表格并点击Submit提交移机申请。

如果您是用户，

请联系您的管理员，帮助您在OriginLab的网站上停用许可证。

FAQ

  • 升级操作系统后，Origin要求我提供重新许可证。怎么办？
  • 在升级新版本之前是否必须停用旧版本许可证呢？

金牌服务续约后更新您的许可证

从Origin2018开始，不同版本共享同一许可证，这意味着新版本会沿用之前版本所有的许可证信息，包括产品密匙及其使用情况。这使得升级更简单。

如果您是管理员，

当您的金牌服务续费完成后，您的序列号将会自动升级为最新的版本。上一版本的所有许可证相关的设定（包括分配密钥的机制和邮箱后缀）将应用到最新的版本中。准备就绪后，您可以通知你的用户进行升级。以下是一些提示：

  1. 打开OriginLab网站(www.OriginLab.com)。登录。
  2. 在Welcome页面，点击"Manage my License(s)"链接。
  3. 在Manage my License(s)页面，查看序列号表格中的Version列，看是否可升级到更新的版本。
    • 如果有新版本,请通知您的用户安装新版本，首次打开新版本时会自动激活。请参考以下如果您是用户 > 如果有可升级的新版本 小节的详细步骤。
    • 如果没有新版本, 请通知您的用户使用相同的产品密钥来重新激活当前版本。新的许可证会更新金牌服务期。请参考以下如果您是用户 > 如果没有可升级的新版本 小节的详细步骤。
    • 如果您想重置一组产品密钥, 请参考 这一小节.

如果您是用户，

  1. 从您的管理员处收到金牌服务续费的信息。
  2. 如果有可升级的新版本
    1. 在激活了当前Origin版本的计算机上安装新的版本。
    2. 打开新版本。激活过程将在后台自动完成。 如果成功，您将会成功激活的信息。 如果失败，将会打开一个带有说明的记事本窗口。请按照说明手动激活。

    如果没有可升级的新版本

    1. 打开Origin。选择 帮助：激活许可证。
    2. 在 许可证激活 对话框， 产品密钥已预先填写。点击激活 按钮。如果成功，您将会收到成功激活的信息。如果失败，将会打开一个带有说明的记事本窗口。请按照说明手动激活。
  3. 在激活过程完成后，选择帮助：关于 Origin，检查金牌服务截止日期是否已经更新。

给管理员的提示：如果您想重置一组产品密钥

  1. 打开OriginLab网站 (www.OriginLab.com)。登录。
  2. 在Welcome 页面中, 点击"Manage my License(s)" 链接。
  3. 在 Manage my License(s) 页面中，点击目标序列号链接。
  4. 在打开的页面上，点击“you can see all product keys here”的链接。
  5. 在Origin Product Registration | License request history 页面上，勾选出不再使用的产品密钥。
  6. 在Comments编辑框中填写您的重置理由。
  7. 点击Mark New Product Keys needed 按钮。
  8. 如果您的申请通过，您将收到一封电子邮件，通知您产品密钥已被重置。 然后您可以在OriginLab网站上看到新的密钥。
