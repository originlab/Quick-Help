6.104 FAQ-1160 What should I do if I see a "tcp_s is bad" error when I start server?

Tcp-s-Is-Bad

Last Update: 11/15/2022

If Origin license server cannot start and you see following error in the debug log:

tcp_s is bad!!! Exiting

It is caused by that you have specified a port number for the vendor daemon orglab in the license file but that port is already used by another process.

To solve this problem,

If Origin is not on the other side of the firewall, there is no need to specify a port number for orglab. Otherwise, ensure you use a free, unoccupied port. Refer to this page for more information of how to allow Origin access through a firewall.
