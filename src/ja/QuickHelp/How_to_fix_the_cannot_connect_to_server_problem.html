FAQ-670 クライアントPCで Origin を起動し、FLEXlmサーバのホスト名を入力すると、情報が拒否されてライセンス情報ダイアログが再表示されます。どのようにしたらよいでしょうか？

troubleshoot-server-connection-problem

最終更新日:2015/02/04

  1. OriginのFLEXlmライセンスサービスが正常に動作しているか確認してください。以下のように操作します。
    a. FLEXlm ライセンスサーバのPCでLMTOOLSを実行します。
    b. Server Status タブで、Perform Status Enquiryをクリックします。
    license server、vendor deamonのステータスとライセンス数が表示されれば問題ありません。ない場合、システム管理者に問い合わせるか、Originのテクニカルサポートに問い合わせてください。
  2. Originネットワークメンバーのコンピュータが、FLEXlmサーバPCに接続しているかを確認してください。そのためには、Windows Explorerを開き、OriginのFLEXlmライセンスサービスにブラウズできるか確認します。
  3. 問題がある場合、FLEXlmライセンスサーバがインストールされているコンピュータで「インターネット接続のファイアウォール」が「例外」なしでアクティベートされてないかを確認します。
    Windows XP, 7, 8 (あるいは他のWindows OS)が固有の「例外」なしの「インターネット接続のファイアウォール」を持つ場合、他のPCにインストールされたOriginのFLEXlmサービスへの接続を防いでいることがあります。これを確認するには、FLEXlmサーバPCのWindowsファイアウォールダイアログを開き、Windowsファイアウォールの左側のリンク「Windows ファイアウォールを介したプログラムまたは機能を許可する」をクリックし、LMGRD.EXE と ORGLAB.EXEをリストから選択します。これらのプログラムを例外として追加すれば、ファイアウォールを介することができます。
  4. まだ問題がある場合、ネットワークシステムの管理者に、ネットワークメンバーのPCとFLEXlmライセンスサービスとの間に上述以外のファイアウォールがないか確認してください。ファイアウォールがある場合、システム管理者にポートを開けてもらい、ポート番号を教えてもらいます。
    完了し、ポート番号がわかったら、FLEXlmライセンスサーバのライセンスファイルにこのポートの情報を追加します。詳細は、こちらを確認してください。
  5. 1 - 4 の項目で問題が解決しない場合、FLEXlm で管理するソフトウェアが同じコンピュータにあるかを確認し、そのソフトウェアがLM_LICENSE_FILE環境変数を作成しているか確認します。この環境変数の存在を調べるには、クライアントPCのコントロールパネル | システム | 設定の変更を開きます。高度な設定タブを開き、環境変数ボタンをクリックします。LM_LICENSE_FILE変数があるか確認します。ある場合、問題解決のためにOriginのパッチを適用する必要があります。
