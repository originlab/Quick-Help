FAQ-917 Origin 2018が、ネットワークロケーションからユーザファイルフォルダを移動した後も正しく動作しなくなる。

Origin-2018-Moving-User-Files-Folder-from-Network-Location

最終更新：2018/07/14

Note: このFAQはOrigin 2018および2018SR1のみ（2018bまたはそれ以前/それ以後のバージョンではありません）に関するものです。

OriginのUser Filesフォルダをネットワーク上の場所からPC上のローカルの場所に移動した後にOriginで問題が発生した場合は、Origin 2018が使用する特定のフォルダをいくつか変更する必要があります。たとえば、変更したプロジェクトファイルを保存しようとしたときに準備完了と報告されることがあります。

Origin 2018は、User Filesフォルダに加えて4つのフォルダを作成します。これらのフォルダは、User Filesフォルダへの兄弟フォルダであり、Autosave、Unsaved、Backup、およびTempSaveという名前が付けられています。Originが正しく動作していない場合は、それらのフォルダの1つ以上がまだネットワーク上に存在する可能性があり、ローカルの場所に移動する必要があります。可能であれば、新しいUser Filesフォルダの兄弟フォルダとして移動してください。

フォルダの場所を変更する

ユーザーファイルのフォルダの場所を変更したら、以下に説明する手順を実行します。

Autosave フォルダ

  1. Originのヘルプメニューから、フォルダを開く：ユーザファイルフォルダを選択します。
  2. Windowsファイルエクスプローラウィンドウで、フォルダ階層の1つ上のレベルに移動します。
  3. Autosaveという名前のフォルダがUser Filesフォルダの兄弟フォルダとして既に存在しますか？
  4. そうでなければ、その場所にAutosaveという名前の空のフォルダを作成します。
  5. Originに戻り、ツール：オプションを選択します。
  6. オプションダイアログで、システムパスタブを選択します。
  7. 自動保存の項目を選択し、変更...ボタンをクリックします。
  8. フォルダをブラウズするダイアログで、上記の手順4で作成した自動保存フォルダに移動し、OKをクリックします。
  9. オプションダイアログに戻り、自動保存のリストが正しいことを確認し、OKボタンをクリックします。
  10. Originを閉じます。

Unsavedフォルダ、Backupフォルダ、およびTempSaveフォルダ

Part I

  1. Windowsファイルエクスプローラで、新しく作成したローカルユーザーファイルフォルダの場所に移動します。
  2. フォルダ階層内の1つのレベルまで移動しましょう。
  3. Unsaved、Backup、およびTempSaveフォルダのいずれかまたはすべてがUser Filesフォルダの兄弟フォルダとして既に存在していますか？
  4. そうでない場合は、欠落しているフォルダのために空のフォルダを作成し、必要に応じてUnsaved、Backup、またはTempSaveの名前を付けます。それぞれのフォルダへの完全なパスを書き留めます。

Part II

  1. レジストリエディタを開く
    Windows 10の場合は、検索ボックスにregeditと入力してEnterキーを押します。
    以前のバージョンのWindowsでは、スタートボタンをクリックし、ファイル名を指定して実行ボックスにregeditと入力してEnterキーを押します。
  2. レジストリエディタで次の場所に移動します。 Computer\HKEY_CURRENT_USER\Software\OriginLab\Origin 9.5\International
  3. レジストリエディタダイアログの右側にあるパネルで、Autosaveのエントリが表示されます。Unsaved、Backup、およびTempSave（または既に存在する場合は変更）に対しても同様のエントリを作成します。
  4. これらのエントリを作成するには、右側のパネルで右クリックし、新規：文字列値を選択します。
  5. 新しい値は新しい値＃1と命名されます。それをダブルクリックして名前をUnsaved、Backup、またはTempSaveに変更します（大文字小文字は重要です）。
  6. エントリを右クリックし、変更...を選択します。
  7. 新しい文字列ダイアログで、関連するローカルフォルダの完全パスを入力します。パスの最後に\（バックスラッシュ）文字を追加してください。
  8. Unsaved、Backup、およびTempSaveの各フォルダについて、手順4〜7を繰り返します。
  9. レジストリエディタを閉じます。

最後に

Originを起動し、以前の不適切な動作が修正されているかどうかを確認します。そうでない場合は、OriginLabテクニカルサポートに連絡して、追加のヘルプとガイダンスを得ることができます。
