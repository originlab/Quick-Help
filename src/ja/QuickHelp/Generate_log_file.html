FAQ-1105 どうやってログファイルを作成したらいいですか?

generate-log-file

最終更新日：2021/5/26

Originは、ソフトウェアとライセンスの問題の診断に役立つログファイルを生成します。OriginLabのカスタマーケアまたはテクニカルサポートの担当者からログファイルの生成を求められた場合は、次の手順で生成できます。（日本のユーザ様のサポートは、株式会社ライトストーンが提供いたします）

  1. Originを起動します。
  2. スクリプトウィンドウ( ウィンドウ：スクリプトウィンドウ )にて、以下のように入力してEnterキーを押します。 
    @log=1
  3. Originを閉じてから再起動し、ログファイルの生成を開始します。
  4. 問題が発生した際に行った手順を繰り返して、エラーメッセージを生成します。
  5. ロギングは、3セッション後に自動的にオフになります。ログファイルを見つける -- ORIGIN_LOG_###.TXTという名前で、ユーザファイルフォルダに保存されています（ヘルプ：フォルダを開く：ユーザファイルフォルダ）。
  6. ログファイルを既存のsupport ticketに添付します。support ticketをお持ちでない場合は、こちらで作成してください。

Origin 2019以前のバージョンでは

  1. スクリプトウィンドウを開き(ウィンドウ： スクリプトウィンドウ)、以下のように入力してEnterキーを押して実行します。 
    debug_log 2
    これにより、ロギングが開始され、ログファイルの場所がスクリプトウィンドウに返されます。
  2. 次のスクリプトを実行して、ロギングをオフにします。 
    debug_log 0
  3. ログステータスを確認し、ログファイルの名前と場所をデバッグするには、次のスクリプトを実行します。 
    debug_log
  4. 上記の手順6の手順に従って、ログファイルを送信してください。

キーワード：ライセンスエラー, 3分間モード, 3分限定, 期限切れ, 失効, expired
