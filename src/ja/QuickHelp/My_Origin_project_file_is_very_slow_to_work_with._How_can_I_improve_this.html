FAQ-468 私のOriginプロジェクトファイルは操作するのに時間が掛かります。どのように解決したらよいでしょうか。

ProjectFile-Very-Slow

最終更新日：2022/02/20

プロジェクトの読み込みが遅い場合、またはプロジェクトを開いている間にOriginの実行速度が遅い場合は、次の手順を試してください。

プロジェクトに多数のグラフウィンドウがある

1つのプロジェクトエクスプローラフォルダ内に多数のウィンドウ（特にグラフウィンドウ）がある場合は、それらをリロードするには多くの時間がかかります。

サブフォルダ

  • それらのいくつかをサブフォルダに移動する。

または

  • このページのウィンドウを表示しないための他のオプションを試してください。

加えて、

  • 現在のプロジェクトフォルダに多数のグラフウィンドウがあり、ウィンドウを非表示にしたくない場合は、システム変数@NPP の値を小さくしてみることができます。そうすることで、部分的に覆われたグラフウィンドウが再描画されないようにして、システムリソースの需要を減らすことができます。システム変数の値を変更する手順については、この FAQを参照してください。

スパークラインを削除する

スパークラインは、「サムネイル」グラフで、ワークシート列に表示されます。複数のワークブックでできている大きなプロジェクトファイル、もしくは操作/分析により、多数のワークシート/ワークブックが生成される場合（例：ワークシートの分割）で、これらのワークブックに（表示や非表示の）スパークラインがあると、システムが遅くなります。

解決法

  • プロジェクトファイルからスパークラインを削除するには、このFAQを参考にしてください。

自動保存を削除する

定期的にフリーズする大きなプロジェクトファイルがある場合、これは自動保存機能 が原因である可能性があります。

解決法

  • 自動保存機能をオフにするには、環境設定:オプションを選択し、オープン/クローズタブの自動セーブする間隔xx分をオフにします。

結果ログをクリアする

Origin 8.6より前のバージョンでは、インポート時に多くの関数のロギングがデフォルトで有効になっていたため（表示するには表示：結果ログを選択するもしくはALT+2を押します）、これらのエントリは時間の経過とともに蓄積される可能性がありました。Origin 8.6では、そのようなエントリをログに記録しないようにデフォルトが変更されましたが、古いOPJファイルの場合はログがすでに非常に大きい可能性があります。大きな結果ログをクリアすると、OPJの読み込みと保存の速度が大幅に向上します。

解決法

  • 結果ログをクリアするには、このページを参考にしてください。
    Note：結果ログの内容が失われることが心配な場合は、内容をノートウィンドウにコピーしてプロジェクトファイルと一緒に保存します。
  • Origin 2020b以降、Originは30,000エントリに達すると結果ログをクリアするように要求します（システム変数@CRLをご参照ください）。

古いバージョンのOriginで作成されたプロジェクト

  • 古いバージョンのOriginで作成されたOPJファイルには、余分な内部オブジェクト（空のオペレーション）が含まれていることがあります。新しいバージョンのOriginではこの問題は修正されましたが、特にOrigin 8.5.1で作成されたファイルには、これらの余分なオブジェクトが依然として含まれている場合があります。
    解決法
    OPJファイルで、空のオペレーションの削除を試してみてください。ウィンドウ：スクリプトウィンドウをクリックしてスクリプトウィンドウを開き、次のように入力してEnterキーを押します。 
    del -op
  • Origin 2018は、新しいUnicode準拠のプロジェクトファイルタイプOPJUを導入しました。Originの最近のバージョンにアップグレードし、OPJファイルタイプの古いプロジェクトファイルを使用している場合は、プロジェクトファイルサイズが大幅に小さくなり、読み込みと保存にかかる時間が短くなるため、ファイルを新しいOPJUフォーマットに保存し直してください。

ルースデータセットを削除する

ルースデータセットは時間の経過とともにいくつかのプロジェクト内に蓄積され、プロジェクトを大きくしてしまうことが分かりました。

解決法

  • ワークシート内になく、かつグラフでも使用されていないすべてのデータセットを削除するには、「ウィンドウ：スクリプトウィンドウ」をクリックし、次のように入力して実行します。
del -as
