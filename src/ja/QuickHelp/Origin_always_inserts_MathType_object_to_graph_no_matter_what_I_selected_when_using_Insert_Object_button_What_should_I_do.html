FAQ-1146 オブジェクトを挿入ボタンでAxMathを選択しても、常にMathTypeオブジェクトがグラフに挿入されます。どうしたらよいでしょうか？

Fail-to-Insert-AxMath-Object-MathType

最終更新日：2022/8/16

MathType 7.x と AxMath の両方が同じPCにインストールされている場合、OriginやWordなどのほかのアプリケーションでの「OLEオブジェクトを挿入」がMathTypeのものになります。例えば、Originでオブジェクトを挿入ボタンを使てグラフウィンドウにOLEオブジェクトを挿入する場合、オブジェクトの挿入ダイアログでAxMathを選択してもMathTypeオブジェクトが常に挿入されます。

MathType 7以降のバージョンでは、インストール時にAutoConvertToキーをレジストリに設定するため、この問題が発生します。

この問題を解決するには、

  1. Windowsスタートボタンをクリックします。検索テキストボックス に次のように入力します。 
    regedit

    レジストリエディタを開きます。

  2. レジストリエディタで、AxMathのキーを参照します 
    Computer\HKEY_CLASSES_ROOT\WOW6432Node\CLSID\{B18C2BCC-4E79-436A-A2A5-A7F8D25A9A28}

    その下にAutoConvertToキーがあります。値がMathTypeに設定されているのが確認できるでしょう。

  3. AutoConvertToキーを削除して、再度オブジェクトの挿入をお試し下さい。オブジェクトが問題なく挿入されるはずです。

キーワード:Microsoft, AutoConvertTo, OLE, オブジェクトを挿入, MathType, AxMath
