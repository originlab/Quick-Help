FAQ-1124 Originのライセンスサービスを開始できず、ログファイルに"The license server manager (lmgrd) is already serving all vendors, exiting."というエラーメッセージが表示されます。

Lic-Server-Manager-Serving-AllVendors

最終更新日：2022/1/14

LMToolsのStart/Stop/RereadタブのStart ServerボタンをクリックしてOriginライセンスサーバーを起動しようとすると、ステータスバーに次のエラーが表示されます。

VD is starting, please check vendor daemon's status in debug log

デバッグログファイルを調べると、次のように表示される場合があります。

18:13:21 (lmgrd) orglab already running 27000@ServerName

18:13:21 (lmgrd) The license server manager (lmgrd) is already serving all vendors, exiting.


同じサーバ上にOriginLabライセンスサービスの2つのインスタンスを構成したために、2番目のインスタンスを開始できなかった可能性があります。

この問題をトラブルシューティングするには、

  1. Config Servicesタブ開きます。Service Nameドロップダウンリストで既存のサービスが表示されます。複数のサービスが同じlmgrd.exe/Originライセンスファイルに設定されているかどうかを確認してください。複数あった場合、1つのサービスを残して他は停止します。
  2. Service Nameドロップダウンリストから停止するサービスを選択します。Start/Stop/Rereadタブを開きます。Stop Serverをクリックしてサービスをクリックします。
  3. Service/License Fileタブを開きます。残すサービスを選択します。
  4. Start/Stop/RereadタブをクリックしてStart Serverボタンをクリックします。サービスが開始されたか確認します。

これで動作する場合は、LMToolsから不要なサービスを安全に削除できます。

  1. Config Servicesタブを開きます。Service Nameドロップダウンリストから停止するサービスを選択します。Remove Serviceボタンをクリックして削除します。


LMToolsにサービスが1つしか表示されないのに、"VD is already starting"というエラーが表示される場合は、以下をお試しください。

  1. タスクマネージャダイアログを開きます
  2. サービスタブを開きます（簡易表示の場合は、詳細ボタンをクリックするとタブを表示できます）。そのマシンでの全サービスが表示されます。設定したOriginライセンスサービスを検索します。
  3. 見つかった場合、右クリックして停止を選択します。
  4. LMToolsに戻ります。再度サーバを開始します。

キーワード:ライセンス, VD already start, serving all vendors, サーバを開始できない
